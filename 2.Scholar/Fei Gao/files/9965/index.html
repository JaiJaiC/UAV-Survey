2. Insight & Novelty

2.1 Core Insight

🎯 核心洞察
问：为什么现有的高速视觉飞行方法在密集杂乱环境中难以同时保证速度和安全？
答：因为现有方法陷入了两个极端——纯学习型方法（端到端 RL）为了追求更高奖励而做出危险动作，在 OOD 场景下安全性完全无保证；纯规划型方法（A*/RRT* + 轨迹优化 + 控制器）每条管线的级联延迟累加（建图 → 规划 → 优化 → 跟踪），在 7.5 m/s 的高速下总延迟已经超过了可用的反应时间窗口。本文的核心洞察在于：RL 负责"敏捷性"——学会在杂乱环境中的高速飞行策略；HOCBF 负责"安全性"——作为最后一道形式化防线，将 RL 的每个输出实时投影到可证明安全的动作集上。这相当于给 RL 策略加了一个数学保证的"安全护栏"——策略可以在护栏内自由发挥性能，但永远不会跨越安全边界。

问：本文的关键技术洞察是什么？
答：(1) 非对称 Actor-Critic 是解决"贫乏观测下训练困难"的零代价方案——Critic 访问仿真真值状态获得准确价值估计，Actor 经域随机化训练学会从噪声/延迟/丢包的深度图中鲁棒推理。部署时仅需 Actor，Critic 不参与——训练时的信息不对称在部署时自动消解；(2) HOCBF 的安全投影本质上是求解一个 4 维单约束二次规划——在 RL 输出动作的"邻域"内寻找满足可证明安全约束的最近可行动作，计算开销微秒级，对实时性零影响；(3) 七项奖励的精心逐项设计不是简单的线性加权——导航奖励使用指数衰减形式（鼓励快速接近目标），安全奖励使用二次形式（形成安全势场），平滑性奖励防止高频抖动——每项奖励的数学形式都对应一个具体的物理意图。

2.3 Novelty Breakdown

Novelty 1: 非对称 PPO——噪声/延迟/丢包容忍的视觉飞行策略学习

【解决的问题】真实无人机上的深度相机存在噪声、运动模糊、帧率不稳定（丢帧）、通信延迟等问题——如果在仿真中用"完美深度图"训练，迁移到真机时性能会雪崩式退化。另一方面，如果 Actor 和 Critic 都用噪声观测，价值估计的质量被感知噪声污染，训练收敛极慢甚至完全不收敛。
【灵感来源】域随机化 + 非对称 Actor-Critic 的组合——域随机化让 Actor 学会适应真实传感器的退化特性，非对称 Critic 让训练保持高效（价值估计不受噪声污染）。该模式在腿足机器人（ANYmal）中已被验证，但本文是首次将其适配至高速无人机视觉飞行场景。
【具体设计】Actor 观测（经过域随机化处理）：深度图 100×60（随机像素级 dropout 20%、高斯噪声 σ=0.05m、随机帧延迟 0-20ms）+ 线速度 + 旋转矩阵 3×3 + 上一帧动作（4 维）+ 目标相对位置（3 维）。每个分量有独立的随机化强度参数。Critic 观测（仅在训练时存在）：上述全部内容 + 真值位姿 + 真值速度 + 障碍物精确 3D 距离。这种设计让 Actor 学会了从严重退化的视觉输入中"推断"环境 3D 结构——类似于人类驾驶员在浓雾中依靠有限的视觉片段判断安全行车路线。

Novelty 2: 物理启发的七项复合奖励——从零学会完整高速飞行行为的隐式课程

【解决的问题】标准 RL 避障任务的奖励通常是稀疏的（"到达目标=+1，碰撞=-1"）——这种奖励信号几乎不可能训练出高速飞行策略（信用分配链过长，探索成本极高）。需要设计一个密集的、物理启发的奖励函数，逐步引导 RL 从零学会"快速、安全、平滑地穿越密集杂乱环境"。
【灵感来源】行为克隆中的"辅助损失"设计——在主要目标之外增加辅助目标（平滑性、能耗）来约束策略的行为空间，防止策略学到不可取的捷径行为。传统运动规划中全局引导 + 局部避障的分层思想。
【具体设计】七项奖励，从高到低的优先级层次：(1) 导航奖励 $r_{\text{nav}}$：目标距离减少量的指数奖励 $\exp(-\beta \cdot \Delta d_t)$——引导策略向目标前进，且"快速靠近"比"缓慢靠近"获得几何级更高的奖励；(2) 安全奖励 $r_{\text{safe}}$：与最近障碍物距离的二次惩罚 $\min(0, d_{\text{obs}}-d_{\text{thresh}})^2$——距离越近惩罚越大，形成"安全势场"；(3) 平滑性奖励 $r_{\text{smooth}}$：连续动作变化量 $\|a_t - a_{t-1}\|^2$ 的 L2 惩罚——防止策略输出高频抖动的姿态指令，这在高速飞行中特别关键（抖动 → 振动 → 传感器噪声加剧 → 恶性循环）；(4) 碰撞惩罚 $r_{\text{col}}$：一旦碰撞终止并给予 -100 的大额惩罚；(5) 成功奖励 $r_{\text{succ}}$：到达目标区域时 +100 的大额奖励；(6) 速度奖励 $r_{\text{speed}}$：速度向量模长的线性奖励 $w_{\text{speed}} \cdot \|v_t\|$——鼓励高速飞行；(7) 高度奖励 $r_{\text{height}}$：偏离目标高度的二次惩罚 $-(z_t - z_{\text{target}})^2$——维持安全飞行高度。

Novelty 3: HOCBF 安全过滤器——策略输出的实时可证明安全投影

【解决的问题】即使在仿真中训练得再好，RL 策略在真实环境中仍是一个黑箱——在 OOD 状态下（如比训练环境密集 3 倍的障碍物布局）可能输出致命动作。尤其在 7.5 m/s 的高速下，一个错误动作的后果在约 100ms 内就会发生——没有"尝试-纠正"的机会。传统解决方案要么在训练时加安全约束（如 constrained MDP）——但约束在训练分布外仍然空洞；要么在部署时靠人工干预——完全不可行。
【灵感来源】控制屏障函数（CBF）在自动驾驶中的成功应用——CBF 将"安全"定义为系统状态保持在一个前向不变的安全集合内，通过将控制输入限制在"安全许可集"中来提供可证明的安全保证。问题：标准 CBF 要求系统的相对阶为 1（即控制输入 $u$ 直接影响安全函数 $h(x)$ 的变化率），但四旋翼的加速度级动力学使得位置约束的相对阶为 2。
【具体设计】HOCBF（High-Order CBF）通过逐级构造中间约束来处理相对阶 ≥ 2 的情况。(1) 定义安全函数 $h(x) = d_{\text{obs}}(x) - r_{\text{safe}}$（无人机到最近障碍物的距离减去安全半径 1.0m）；(2) 构造一阶扩展约束 $\psi_1 = \dot{h} + \alpha_1 h \geq 0$——限制 $h$ 向零靠近的速率；(3) 构造二阶约束 $\psi_2 = \ddot{h} + (\alpha_1+\alpha_2)\dot{h} + \alpha_1\alpha_2 h \geq 0$——这个约束是控制输入 $a$ 的线性函数，可作为 QP 约束；(4) 每个控制周期（100Hz）求解 QP：$\min_{a} \|a - a_{RL}\|_W^2$ s.t. $\psi_2(a) \geq 0$——找到离 RL 输出最近且满足形式化安全约束的动作。QP 仅需微秒级求解，对实时性零影响。

4. Task — What Problem Does It Solve?

4.1 Problem Formulation

核心问题（约束马尔可夫决策过程 CMDP） $$ \max_{\pi} \mathbb{E}_{\tau \sim \pi} \left[ \sum_{t=0}^{T} \gamma^t R(s_t, a_t) \right] \quad \text{s.t.} \quad \forall t: h(x_t) \geq 0 $$ $$ h(x) = d_{\text{obs}}(x) - r_{\text{safe}}, \quad a_t^{\text{exec}} = \text{HOCBF}\big( \pi(o_t^{\text{noisy}}) \big) $$
$\pi$ = 视觉飞行策略（从噪声深度图 + 本体感知到姿态指令的端到端映射） |  $o_t^{\text{noisy}}$ = 经域随机化处理的观测（丢包/噪声/延迟——模拟真实传感器退化） |  $h(x_t)$ = 安全函数——四旋翼位置到最近障碍物距离减去安全半径 1.0m  |  HOCBF = 高阶控制屏障函数——将 $\pi$ 的输出投影到满足 $h(x) \geq 0$ 的可证明安全动作集  |  $R$ = 七项复合奖励：导航 + 安全 + 平滑 + 碰撞 + 成功 + 速度 + 高度。

4.3 Task Type

这是一个安全关键的视觉高速自主导航任务——处于"感知-规划-控制"的交汇地带。核心特征：(1) 感知层面——从 100×60 极低分辨率深度图中隐式理解 3D 空间结构，无需显式建图、SLAM 或占据栅格；(2) 决策层面——在部分可观测 + 感知退化的条件下实时输出安全飞行动作（100Hz 闭环）；(3) 安全层面——通过 HOCBF 提供可证明的数学安全保证（前提：障碍物检测正确）。任务的关键难点：100×60 = 6000 像素的深度图是极其有限的信息瓶颈，而 7.5 m/s 的高速意味着"看到障碍物 → 做出反应"的可用时间窗口约 500ms——策略必须在信息极度贫乏的条件下做出零容错的安全决策。

5. Challenge — Why Is It Hard?

5.1 Essence (First Principles)

💡 第一性原理诊断
核心矛盾：高速视觉飞行中的"敏捷性-安全性-感知可靠性"不可能三角。

（1）敏捷性 vs 安全性：速度越高，可用的安全反应时间越短（7.5 m/s 时，10m 外障碍物的反应窗口仅 1.3 秒，减去感知延迟约 50ms 和机动执行时间约 200ms，纯决策时间不足 1 秒）。RL 策略为了最大化速度奖励，天然倾向于选择紧贴障碍物的"擦边"路径——这在完美的仿真感知中可以成功，但在真实传感器噪声下极其危险。HOCBF 通过在每个控制周期强制执行形式化安全约束，从根本上解决了这个矛盾——RL 可以自由追求速度，但 HOCBF 在输出端将其"拉回"安全区域。

（2）感知可靠性 vs 速度：高速飞行中深度相机面临三类退化：(a) 运动模糊——7.5 m/s 线速度 + 角速度导致的图像模糊使深度估计算法失效；(b) 帧率不稳定——30Hz 相机在高动态下可能出现帧丢失，导致 GRU 时序链断裂；(c) 固有噪声——消费级深度相机（如 RealSense D435i）在 5m 处的深度噪声约 σ=0.05m。100×60 的超低分辨率进一步压缩了可用信息——CNN 必须仅从 6000 个像素中推断完整的 3D 避障策略。域随机化（丢包 20% + 噪声 σ=0.05m + 延迟 0-20ms）迫使策略学会在"感知严重退化"的条件下仍做出安全决策。

（3）Sim-to-Real 动力学 mismatch：仿真中的四旋翼动力学模型（推力响应曲线、气动阻尼系数、电池电压-推力映射、螺旋桨惯量效应）与真机存在系统性差异。RL 策略可能无意识地学会利用仿真的特定动力学特性（如"瞬间全推力加速"）——这在真机上由于电机电流限制和电池内阻而无法复现。解决策略：(a) 输出姿态角而非电机指令——将底层执行器动力学的 mismatch 委托给 PX4 成熟的级联控制律吸收；(b) 仿真中加入动力学域随机化（质量 ±15%、转动惯量 ±20%、推力系数 ±10%）——覆盖真机参数的制造公差范围。

三者交织：这不是三个独立问题的简单叠加——它们相互放大。感知噪声导致 RL 策略误判障碍物位置 → HOCBF 基于错误输入触发不当安全干预 → 过度减速导致真机动力学进入高气动阻尼区域 → 策略的输出与实际执行偏差增大 → 进一步加剧感知-控制的 mismatch。打破这个恶性循环需要三层设计同时到位：非对称训练（解决感知噪声下的学习）、域随机化（解决分布漂移）、HOCBF（解决安全底线）。

5.3 Key Challenges

  • 挑战 1 — 部分可观测性下的高速信用分配：深度相机 100×60 分辨率 + 仅前视 FoV——策略需要在信息极度不完整的条件下，对 100Hz 频率的动作输出进行正确的信用分配。经典 RL 的信用分配问题（"1 秒前的某个动作导致了现在的碰撞吗？"）在部分可观测 + 高速的场景下被极度放大——因为观测噪声使因果关系更加模糊。非对称 Critic 通过访问真值状态来切断这种模糊性。
  • 挑战 2 — 安全性的可证明性与感知错误的张力：HOCBF 提供的形式化安全保证以"障碍物距离 $d_{\text{obs}}$ 已知且准确"为前提——但在真实深度相机中，细薄障碍物（电线、细树枝）、透明障碍物（玻璃）、黑色表面（吸收红外）均存在系统性漏检。一旦 $d_{\text{obs}}$ 被低估，HOCBF 的可证明安全性立即失效——而策略对此完全无感知。这个"感知错误 → 安全保证失效"的链式故障是本文方法最根本的脆弱性。
  • 挑战 3 — CBF 保守性 vs 通过性的最优权衡：HOCBF 的安全半径 $r_{\text{safe}}$ 是一个关键设计参数——太大（如 1.5m）会过度约束飞行空间，在狭窄通道中 QP 无可行解（策略被"锁死"）；太小（如 0.5m）则安全冗余不足。且最优值取决于当前飞行条件（速度、环境密度、传感器噪声水平）——目前是固定值，无法自适应调节。这个参数的标定是经验和理论之间的灰色地带。

6. Method & Key Formulas

6.1 Core Algorithm

整体算法分为三个解耦的阶段：(1) 训练阶段——在仿真中运行非对称 PPO，Actor 接收经过域随机化（丢包/噪声/延迟）的观测，Critic 接收真值状态，通过最大化七项复合奖励学习从 100×60 深度图到归一化推力 + 姿态角指令的端到端映射；(2) 闭环验证阶段——将训练好的 Actor 与 HOCBF 结合，在仿真中验证碰撞率（必须为 0%）和 HOCBF 触发率（必须 < 5%）；(3) 部署阶段——将 Actor 权重（仅 Actor，不含 Critic）零样本迁移到真机，每个控制周期（100Hz）运行：Actor 推理（< 5ms）→ HOCBF QP（< 2ms）→ PX4 姿态跟踪。

6.2 Key Formulas

Group A: 非对称 PPO 目标函数

Eq.(1) — 非对称 Actor-Critic PPO 损失函数 $$ L_{\text{PPO}}(\theta) = \mathbb{E}_t \left[ \min\left( r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right] - \beta \cdot \mathcal{H}(\pi_\theta) + \lambda \cdot L_{\text{value}}(\phi) $$ $$ r_t(\theta) = \frac{\pi_\theta(a_t | o_t^{\text{noisy}})}{\pi_{\theta_{\text{old}}}(a_t | o_t^{\text{noisy}})}, \quad \hat{A}_t = \sum_{k=0}^{\infty} (\gamma \lambda_{\text{GAE}})^k \delta_{t+k} $$ $$ \delta_t = r_t + \gamma V_\phi(s_t^{\text{priv}}) - V_\phi(s_{t+1}^{\text{priv}}), \quad \text{其中 } V_\phi \text{ 的输入为 Critic 可见的真值状态} $$
$o_t^{\text{noisy}}$ = Actor 的噪声观测（深度图丢包 20% + 高斯噪声 σ=0.05m + 随机延迟 0-20ms） |  $s_t^{\text{priv}}$ = Critic 的特权状态（真值位姿/速度/障碍物精确 3D 距离） |  $\lambda_{\text{GAE}}$ = GAE trace-decay 参数 0.95——平衡优势估计的偏差和方差  |  $\beta \cdot \mathcal{H}$ = 熵正则化——在高维动作空间中防止策略过早确定性坍缩到局部最优  |  $L_{\text{value}}$ = 价值网络 MSE 损失——Critic 学习准确的价值估计以提供高质量策略梯度。

Group B: HOCBF 安全过滤器——QP 形式的实时可证明安全投影

Eq.(2) — HOCBF 安全投影 QP 及其约束构造 $$ \min_{a \in \mathbb{R}^4} \|a - a_{RL}\|_W^2 \quad \text{s.t.} \quad L_f^2 h(x) + L_g L_f h(x) \cdot a + (\alpha_1 + \alpha_2) L_f h(x) + \alpha_1 \alpha_2 h(x) \geq 0 $$ $$ h(x) = d_{\text{obs}}(x) - r_{\text{safe}} = d_{\text{obs}}(x) - 1.0, \quad \dot{h} \text{ 由有限差分法从连续深度帧中在线估计} $$
$h(x)$ = 安全函数——无人机位置到最近障碍物的距离减去安全半径 1.0m  |  $L_f, L_g$ = 四旋翼动力学函数 $f$（无控制项）和 $g$（控制系数矩阵）的 Lie 导数  |  $L_f^2 h(x)$ = 安全函数的二阶 Lie 导数（由无控制动力学决定的固有加速度项） |  $L_g L_f h(x)$ = 控制输入 $a$ 对安全的影响通道——这是 QP 约束中唯一与决策变量相关的项（线性） |  $\alpha_1, \alpha_2$ = 安全衰减系数——控制向安全边界靠近的速率上限，值越大越保守（默认 3.0） |  $W$ = 对角加权矩阵——$W_{ii}$ 大表示第 $i$ 维动作修改代价高（推力通常比姿态更不应被修改）。

Group C: 七项物理启发奖励函数

Eq.(3) — 七项复合奖励函数的完整设计（优先级递减） $$ R_t = \underbrace{w_{\text{nav}} \cdot \exp(-\beta_{\text{nav}} \cdot \Delta d_t)}_{\text{① 导航奖励: 指数衰减}} + \underbrace{w_{\text{safe}} \cdot \min\big(0, d_{\text{obs}} - d_{\text{thresh}}\big)^2}_{\text{② 安全奖励: 二次势场}} + \underbrace{w_{\text{smooth}} \cdot \|a_t - a_{t-1}\|_2^2}_{\text{③ 平滑性奖励: L2惩罚}} $$ $$ + \underbrace{w_{\text{col}} \cdot \mathbf{1}_{\text{collision}}}_{\text{④ 碰撞惩罚: -100}} + \underbrace{w_{\text{succ}} \cdot \mathbf{1}_{\text{success}}}_{\text{⑤ 成功奖励: +100}} + \underbrace{w_{\text{speed}} \cdot \|v_t\|_2}_{\text{⑥ 速度奖励: 线性}} + \underbrace{w_{\text{height}} \cdot (z_t - z_{\text{target}})^2}_{\text{⑦ 高度奖励: 二次惩罚}} $$
$\Delta d_t = d_t - d_{t-1}$ = 到目标距离的变化量——负值（靠近目标）获得正奖励，指数形式使"快速靠近"比"缓慢靠近"获得几何级更高的奖励  |  $d_{\text{obs}}$ = 到最近障碍物距离，$d_{\text{thresh}}$ = 安全阈值——距离 < 阈值时二次惩罚，形成安全势场（越近惩罚梯度越大） |  权重层次（经验调参）：$w_{\text{nav}} \gg w_{\text{speed}} > w_{\text{safe}} > w_{\text{smooth}} > w_{\text{height}}$——导航（到达目标）是最高优先级，速度次之，其余为辅助约束。

Group D: CNN + GRU 特征提取——从 100×60 深度图到 128 维时空隐状态

Eq.(4) — 视觉-时序特征提取完整流程 $$ \mathbf{f}_{\text{depth}} = \text{CNN}(I_{\text{depth}}) \in \mathbb{R}^{256}, \quad I_{\text{depth}} \in \mathbb{R}^{100 \times 60 \times 1} $$ $$ \mathbf{f}_{\text{state}} = \text{MLP}\big([v_x, v_y, v_z,\; \text{flatten}(R_{3 \times 3}),\; \Delta x_g, \Delta y_g, \Delta z_g]\big) \in \mathbb{R}^{128} $$ $$ \mathbf{h}_t = \text{GRU}\big(\mathbf{h}_{t-1},\; [\mathbf{f}_{\text{depth}},\; \mathbf{f}_{\text{state}},\; \mathbf{a}_{t-1}]\big) \in \mathbb{R}^{128} $$ $$ a_t = \text{MLP}_{\text{actor}}(\mathbf{h}_t) \in \mathbb{R}^4, \quad V_t = \text{MLP}_{\text{critic}}([\mathbf{h}_t,\; s_t^{\text{priv}}]) \in \mathbb{R} $$
CNN 详细架构：3 层 Conv2d(3×3 kernel, stride=2, pad=1) + BatchNorm + ReLU，通道数 64 → 128 → 256 → AdaptiveAvgPool2d(1×1) → Flatten → Linear(256)  |  GRU 隐状态 128 维——在时序记忆容量和推理延迟（< 5ms on Jetson）之间的工程折中  |  $\mathbf{a}_{t-1}$ = 上一帧动作——为 GRU 提供"当前处于何种飞行模式"的上下文（加速/转向/悬停），类似于 Kalman Filter 的控制输入项  |  关键工程细节：GRU 的隐状态充当"短期视觉记忆"——当深度图因丢包（20% 概率）而缺失时，策略可基于 $\mathbf{h}_{t-1}$ 进行"预测性推断"，而非输出随机动作。

6.4 有 HOCBF vs 无 HOCBF 的系统性能对比

🛡️ 有 HOCBF 安全过滤器（本文完整方案）

  • 每个动作都经过 100Hz QP 安全验证——可证明不碰撞（前提：障碍物检测正确）
  • QP 最小化动作修改量 $\|a - a_{RL}\|_W^2$——策略意图最大程度保留
  • 仿真碰撞率：0%（HOCBF 数学保证）
  • 真机碰撞率：~2%（仅由感知漏检导致——电线/细树枝未被深度相机检测到）
  • 平均飞行速度：比无 HOCBF 低约 5%（正常飞行中 HOCBF 极少触发修改）
  • HOCBF 平均触发频率：3.7%（每 100 个控制周期约 4 次触发安全投影）
  • OOD 泛化（3 倍训练密度）：成功率 > 90%

⚡ 无 HOCBF（纯 RL 端到端，无安全层）

  • RL 输出直接执行——无中间验证步骤，无安全保证
  • 无修正机制——RL 的 OOD 错误输出直接作用于无人机
  • 仿真碰撞率：~8%（RL 探索中自然发生的碰撞）
  • 真机碰撞率：~15%（sim-to-real gap 放大 RL 的危险动作倾向）
  • 平均飞行速度：理论更快（无 CBF 减速干预），但碰撞后速度为 0
  • 无触发频率概念——唯一的反馈是一次碰撞后的任务终止
  • OOD 泛化（3 倍训练密度）：成功率仅 ~61%

7. Fundamental Principles — 核心技术原理详解

7.1 非对称 Actor-Critic —— 用特权信息"教"策略在贫乏观测下做出最优决策

原理 1 — 非对称训练的信息理论与梯度正确性 $$ \text{信息不等式（Actor 能获取的信息始终少于 Critic）: } I(a_t; s_t^{\text{priv}} \mid o_t^{\text{noisy}}) < I(a_t; s_t^{\text{priv}} \mid s_t^{\text{priv}}) $$ $$ \text{但策略梯度无偏（因为 } o_t^{\text{noisy}} \text{ 是 } s_t^{\text{priv}} \text{ 的因果函数）: } \nabla_\theta J = \mathbb{E} \left[ \nabla_\theta \log \pi_\theta(a_t \mid o_t^{\text{noisy}}) \cdot Q^\pi(s_t^{\text{priv}}, a_t) \right] $$
关键洞察：虽然 Actor 无法访问特权状态（信息不等式约束了策略的最高可达性能——这是无法突破的上限），但 Critic 基于真值状态估计的 $Q^\pi$ 是正确的价值估计——因为 $o_t^{\text{noisy}}$ 在因果关系上是 $s_t^{\text{priv}}$ 的（噪声）函数。策略梯度定理保证了梯度方向的正确性——非对称 Critic 为 Actor 提供的梯度信号是无偏的。

直觉类比：驾校教练（Critic）可以看到整个路况（特权状态——所有障碍物的精确位置和速度），随时告诉学员（Actor）："刚才那个转弯可以更快"或"刚才那个距离太近了下次注意"。学员自己只能看到前方有限的模糊视野（噪声深度图），但通过教练的持续反馈，学员学会了从有限视野中推断出完整的路况信息。这就是非对称训练的"知识传递"效应——Critic 的"上帝视角"通过策略梯度桥梁完整地传递给了只有"局部视角"的 Actor。

在本文的具体场景中，Critic 访问的障碍物精确距离信息具有不可替代的作用——它让价值函数在"虽然 Actor 视野中看不到，但确实距离障碍物很近"的状态上赋予极低的价值（危险预警），即使该障碍物在当前深度图中因丢包或处于视野边缘而不可见。这种"隐藏危险的预先警告"通过优势函数 $\hat{A}_t$ 编码为梯度信号，引导 Actor 学会对视野边缘和丢包区域保持天然的谨慎。

7.2 HOCBF —— 为什么高阶控制屏障函数对四旋翼飞行安全是必要的

原理 2 — 相对阶的概念与 HOCBF 的构造逻辑 $$ \text{四旋翼动力学（简化）: } \dot{x} = f(x) + g(x) u, \quad x = [\mathbf{p}, \dot{\mathbf{p}}, \mathbf{q}, \boldsymbol{\omega}]^T, \quad u = [f_{\text{thrust}}, \phi_{\text{des}}, \theta_{\text{des}}, \psi_{\text{des}}]^T $$ $$ h(x) = \|\mathbf{p} - \mathbf{p}_{\text{obs}}\| - r_{\text{safe}} \text{（位置函数）} \Rightarrow L_g h(x) = 0 \text{（} u \text{ 是加速度级，不直接影响位置）} $$ $$ L_g L_f h(x) \neq 0 \Rightarrow \text{相对阶 } = 2 \Rightarrow \text{需要 HOCBF 而非标准 CBF} $$
$L_g h(x) = 0$ = 控制输入 $u$（推力 + 姿态角）不直接影响安全函数 $h$ 的变化率——因为 $u$ 直接影响的是加速度 $\ddot{\mathbf{p}}$，而 $h$ 是位置 $\mathbf{p}$ 的函数。必须经过两次时间积分（$\ddot{\mathbf{p}} \rightarrow \dot{\mathbf{p}} \rightarrow \mathbf{p}$）才能使 $u$ 的影响传播到 $h$。

标准 CBF 的局限：标准 CBF 约束为 $\dot{h}(x) + \alpha h(x) \geq 0$——这要求 $u$ 能瞬间影响 $\dot{h}$（即 $L_g h(x) \neq 0$，相对阶 = 1）。这对四旋翼的碰撞避免约束不成立——推力不能瞬间改变位置（只能改变加速度），因此标准 CBF 在数学上不适用于该问题。

HOCBF 的核心技巧：将原始的 1 个安全约束 $h(x) \geq 0$（相对阶 = 2）替换为 2 个级联的中间约束：$\psi_1 = \dot{h} + \alpha_1 h \geq 0$（相对阶 = 1）和 $\psi_2 = \dot{\psi}_1 + \alpha_2 \psi_1 \geq 0$（相对阶 = 0，即直接约束 $u$）。这个两级构造将高相对阶问题降阶为可直接求解的一阶问题。

7.3 域随机化 —— 从统计学习角度理解 sim-to-real 的泛化

原理 3 — 域随机化的泛化误差界与"包裹策略" $$ \mathbb{E}_{\text{real}}[R(\pi)] \geq \mathbb{E}_{\text{sim}}^{\text{rand}}[R(\pi)] - D_{TV}(p_{\text{sim}}^{\text{rand}}, p_{\text{real}}) \cdot R_{\max} $$ $$ p_{\text{sim}}^{\text{rand}} = \text{Mixture}\big(p_{\text{sim}}(\theta_1), p_{\text{sim}}(\theta_2), ..., p_{\text{sim}}(\theta_K)\big) $$ $$ \text{目标: } p_{\text{sim}}^{\text{rand}} \supseteq p_{\text{real}} \text{（仿真分布"包裹"真实分布）} $$
$D_{TV}$ = 仿真观测分布 $p_{\text{sim}}^{\text{rand}}$ 与真机观测分布 $p_{\text{real}}$ 之间的全变差距离（Total Variation distance）。距离越小，sim-to-real 的性能退化越少——域随机化的目标就是最小化这个距离。

$p_{\text{sim}}^{\text{rand}}$ = 经过域随机化后的仿真观测分布——是 $K$ 个不同仿真参数 $\theta_k$（对应不同的噪声水平、延迟、动力学参数）下观测分布的均匀混合。混合分布的支撑集远大于任何单个分布。

"包裹"策略：如果 $p_{\text{sim}}^{\text{rand}}$ 的支撑集覆盖了 $p_{\text{real}}$ 的支撑集（即真实世界中可能出现的任何传感器/动力学状态都在仿真中出现过），则策略在真实世界中的性能退化在理论上受限于覆盖的"紧密程度"。域随机化的艺术在于选择正确的随机化维度和范围——太少则覆盖不足（gap 大），太多则训练过难（策略学不会）。

本文的域随机化分为三组共 7 个维度：(a) 感知随机化 3 项——深度图丢包（20%）、高斯噪声（σ=0.05m）、通信延迟（0-20ms uniform）；(b) 动力学随机化 3 项——质量 ±15%、转动惯量 ±20%、推力系数 ±10%；(c) 环境随机化 1 项——障碍物密度/尺寸/位置/光照条件随机。消融实验表明：全部保留时真机成功率为 93.8%，移除全部域随机化后真机成功率暴跌至 58.3%——降幅高达 35.5 个百分点，是所有消融项中影响最大的单一模块。

💡 三个原理的系统级协同
非对称 Actor-Critic 提供训练效率（解决"贫乏观测下信用分配困难"——价值估计方差降低约 3 倍）→ 域随机化 提供鲁棒性桥接（解决"仿真到真实的分布漂移"——真机成功率从 58% 提升至 93%）→ HOCBF 提供部署安全底线（解决"RL 黑盒在 OOD 状态下的不可信性"——OOD 密度 ×3 时成功率从 61% 提升至 90%）→ 三者形成一个完整的"高效训练-鲁棒迁移-安全部署"闭环系统。缺少任何一环：无非对称训练 → 策略不够鲁棒（训练不收敛）；无域随机化 → 真机迁移失败（成功率 < 60%）；无 HOCBF → OOD 场景不安全（成功率 < 62%）。三个模块各自独立贡献，叠加后产生超线性的协同效果——这正是系统工程中"1+1+1 > 3"的典型案例。

9. Potential Flaw & Extensions

9.1 Limitations

  • HOCBF 对完美障碍物检测的根本性依赖：HOCBF 提供的"可证明安全保证"以障碍物距离 $d_{\text{obs}}$ 准确为前提——这是一个很强的假设。在实际部署中，以下障碍物类别存在系统性的检测失败风险：(a) 细薄物体——电线、细树枝（直径 < 5mm）在 100×60 深度图中可能占据 0-1 个像素，被亚像素采样遗漏；(b) 透明/反射表面——玻璃幕墙、水面反射导致深度值为 0（空洞）或错误值；(c) 动态障碍物——行人、其他无人机、飘落树叶的相对速度使有限差分估计失效。一旦 $d_{\text{obs}}$ 被低估或未检测，HOCBF 的数学保证立即失效——而系统对此完全没有感知（没有不确定性估计或故障检测机制）。
  • 全局导航能力的缺失：策略的输入仅包含"目标相对位置"（局部导航信号），没有全局地图或高层语义规划能力。这意味着：(a) 如果目标点在大型障碍物（如建筑物）后方，策略无法自动规划绕行路径——可能陷入"直冲障碍物→HOCBF 减速→徘徊→再冲"的死循环；(b) 在 U 型或环形障碍物布局中，策略可能被"困住"——因为局部梯度永远指向目标方向（穿过障碍物），而绕行需要暂时的"远离目标"——这在纯局部奖励函数中是不可学习的。
  • 100×60 深度图的信息瓶颈：虽然低分辨率有助于计算效率和泛化，但在某些场景中 6000 个像素根本不足以区分安全空隙和危险空隙。例如两个相距 0.8m 的杆子和一个 1.2m 空隙——在 100×60 分辨率 + 10m 距离的投影下，这些可能仅相差 1-2 像素——CNN 需要从亚像素模式中推断可通过性，这在不完美的深度值下几乎是随机猜测。

9.2 Extension Directions

  • 不确定感知的安全屏障（Probabilistic CBF）：将深度图的逐像素不确定性估计（如从深度相机的噪声模型中导出的方差图）纳入 HOCBF 约束——将确定性约束 $h(x) \geq 0$ 升级为概率约束 $\Pr[h(x) \geq 0] \geq 1-\delta$。当深度不确定性高时，CBF 自动变得更加保守——可从根本上解决"感知错误→安全保证失效"的链式故障。
  • 分层规划：拓扑地图 + RL 飞行：在 RL 策略之上增加一个轻量级拓扑规划器——高层负责"选择全局通行的子目标序列"（如基于可见性图的稀疏路径搜索，> 1Hz 更新），低层 RL 负责"在子目标之间以最高速度安全飞行"（100Hz 推理）。高层规划处理"绕行大型障碍物"的全局需求，低层 RL 保持端到端的敏捷性优势。
  • 多模态感知融合：深度 + 事件相机：事件相机（event camera）的高时间分辨率（微秒级）和低延迟是深度相机的理想补充——事件流可以捕获高速运动中的障碍物边缘信息（深度相机可能因运动模糊丢失），也可在强光/弱光下提供可靠的几何线索。将事件流作为一个额外通道馈入 CNN 或作为 GRU 的辅助输入，可大幅提升极端光照条件下的鲁棒性。
  • 在线自适应 CBF 参数：当前 HOCBF 的 $\alpha_1, \alpha_2$ 和 $r_{\text{safe}}$ 是手动标定的固定值——但在实际部署中，最优值依赖于飞行条件（速度、环境密度、电池电压、风速）。设计一个在线自适应机制——根据实时飞行状态动态调节 CBF 保守度：环境密度高 → 提高 $r_{\text{safe}}$ 和 $\alpha$ → 更安全（减速）；环境开阔 → 降低 $r_{\text{safe}}$ → 更敏捷（提速）。

9.4 Paper-Worthy Deep-Dive

🔬 值得深挖的方向
从"安全屏蔽"到"安全协同训练"——CBF-Guided Reinforcement Learning。当前架构中 HOCBF 和 RL 是单向关系：RL 输出 → HOCBF 修正。但如果将 HOCBF 的激活信号反馈到 RL 的训练损失函数中——例如，将"HOCBF 触发的频率"作为额外的正则化惩罚项加入 PPO 目标（惩罚那些频繁触发 CBF 的策略），RL 策略会主动学习"不需要 CBF 修正的内在安全行为"。HOCBF 的角色从"部署时的被动安全网"升级为"训练时的主动安全教师"——两者的关系从单向变为双向协同。预期效果：CBF 触发频率从 3.7% 降至 < 1%，飞行速度进一步提高（因为策略自身更安全，CBF 减少减速干预）。这代表了从"安全作为附加层"到"安全作为训练信号"的范式升级——将安全从系统外部的补丁变成系统内部的目标。

10. Motivation — First-Principles Derivation

10.1 Idea Derivation Chain

  1. 问题的本质是什么？
    无人机在杂乱环境中的飞行速度存在一个由系统架构决定的"天花板"——传统"感知→建图→规划→控制"的串行 pipeline 中，每一级的延迟都在累加：深度图获取 33ms + 建图 30ms + 规划 50ms + 优化 30ms + 通信 10ms = 总延迟约 150ms。在 7.5 m/s 下，这意味着从"看到"障碍物到"开始反应"，无人机已经飞了 1.1 米。这不是"算法不够优化"的问题——这是"架构性延迟"——只要 pipeline 是串行的，这个天花板就无法突破。
  2. 前人如何处理？各自的根本局限是什么？
    (i) 经典规划方法——被架构性延迟锁死在 3-4 m/s；(ii) 端到端 RL——消除了 pipeline 延迟（"深度图→动作"一步到位），速度可达 5+ m/s，但缺乏安全保证——黑箱策略在 OOD 场景可能输出致命动作；(iii) 可微分物理——离线优化、开环执行、不鲁棒。三者的共同问题：都在追求"一个系统同时解决性能和安全"，但这是两个正交的目标——最好由不同机制分别处理。
  3. 为什么 RL + HOCBF 的混合架构是正确方向？
    因为性能和安全性对系统设计的要求是不同的——(a) 性能（速度、敏捷、平滑路径）需要数据驱动的灵活性和泛化——这是 RL 的强项；(b) 安全性（不碰撞）需要可证明的数学保证——这是 CBF 的强项。混合架构让两者各自发挥所长：RL 在安全集内自由追求最优飞行策略，HOCBF 保证策略永远不会越出安全集。接口设计（RL 输出作为 QP 目标，QP 以最小修正投影到安全集）确保了"安全干预"的轻量化和策略意图的最大保留。
  4. 为什么非对称训练是必要的？
    在部分可观测 + 强噪声条件下，标准 RL 的信用分配面临"双重模糊"——(a) 感知模糊：观测噪声使状态识别不准确；(b) 时间模糊：高速下动作和其后果之间有显著的物理延迟（推力→加速度→速度→位置）。如果 Critic 也只看到噪声观测，它无法区分"动作本身是坏的"和"观测噪声让动作看起来像坏的"——价值估计质量极差。非对称 Critic（访问真值）切断了感知模糊对价值估计的污染——它准确地知道"在真实状态下，这个动作是好是坏"，然后通过策略梯度将这个判断无偏地传递给 Actor。
  5. 为什么域随机化是 sim-to-real 的基石？
    Sim-to-real gap 的本质是 $p_{\text{sim}} \neq p_{\text{real}}$（仿真和真机的观测/动力学分布不同）。域随机化通过将 $p_{\text{sim}}$ 扩展为多个分布的混合 $p_{\text{sim}}^{\text{rand}} = \text{Mixture}(p_{\text{sim}}(\theta_1), ..., p_{\text{sim}}(\theta_K))$，使其覆盖 $p_{\text{real}}$ 的支撑集。策略在混合分布上训练，等价于在所有可能的"真实世界"中同时学习——因此迁移到任何一个具体的真实世界时都已经"见过类似的情况"。消融实验中，域随机化的真机成功率贡献高达 35.5 个百分点——在所有单一模块中最高。

10.2 One-Sentence Motivation

💡 一句话动机
无人机在杂乱环境中的飞行速度被人为限制在约 3 m/s——这不是空气动力学极限，而是传统"感知-建图-规划-控制"串行 pipeline 的架构性延迟导致的——为什么不换一种思路：用 RL 跳过所有中间环节（深度图直接到飞行动作）突破速度瓶颈到 7.5 m/s，同时叠加一个数学上可证明"永不碰撞"的 HOCBF 安全层确保无论 RL 输出什么，无人机都不会撞？

10.3 How You'd Arrive at This Idea

  • Step 1 — 识别根本瓶颈（不是算法问题，是架构问题）：观察现有无人机自主飞行系统——最快的系统速度约 5 m/s（Loquercio et al., 2021）。深入分析 pipeline 每个环节的延迟，发现延迟不是任何单一模块的优化问题——是"串行多模块"架构本身的设计缺陷。结论：要突破速度天花板，必须消除串行 pipeline——端到端是唯一出路。
  • Step 2 — 对标人类 FPV 飞手：人类 FPV 飞手在森林中可以达到 100+ km/h（~28 m/s）——远快于任何自主系统。人是怎么做到的？答案是"端到端感知-动作闭环"——眼睛看到树木 → 大脑直接输出手指动作，中间没有显式的 3D 重建、路径规划、轨迹优化。这本质上就是端到端学习。但人类也会撞——尤其在疲劳或注意力不集中时。"人类水平的敏捷 + 超人类的安全"应该是设计目标。
  • Step 3 — 寻找安全兜底机制：CBF 提供了可证明的安全性——如果障碍物检测正确。CBF 的天然优势：(a) 轻量（QP 微秒级求解）、(b) 模块化（不侵入 RL 策略内部——仅在输出端修正）、(c) 可证明（数学保证安全集的前向不变性）。但标准 CBF 不适配四旋翼（相对阶问题）→ HOCBF 解决。
  • Step 4 — 解决训练和迁移：(a) 非对称 Actor-Critic（加速部分可观测下的训练）；(b) 域随机化 7 维（保证 sim-to-real 零样本迁移——消融实验证明这是贡献最大的单一模块）；(c) 输出姿态角而非电机指令（将底层执行器动力学 mismatch 委托给 PX4）。完整方案 → 仿真 96.3% + 真机 93.8% 成功率 → 证明了"端到端 RL（性能）+ HOCBF（安全）+ 域随机化（迁移）"三合一的范式有效性。

11. Key Takeaways

11.1 Lessons Learned

  • 方法论层面：这是无人机自主飞行领域一个标志性的混合范式案例——"学习负责性能上限，经典安全方法负责性能下限"。不是让 RL 解决所有问题，也不是完全依赖基于模型的方法，而是让各自在"比较优势"维度上发挥作用：(a) RL 擅长端到端的感知-决策映射（灵活性、泛化），(b) CBF 擅长基于模型的形式化安全保证（可靠性、可证明性），(c) 域随机化擅长 sim-to-real 分布桥接（鲁棒性）。三者的接口设计（RL 输出作为 QP 目标 → QP 以最小修正投影 → 安全动作 → PX4 跟踪）简洁且高效。类似架构正在被推广到自动驾驶（RL + 责任敏感安全 RSS）、机器人操作（RL + 力约束 CBF）、空中交通管理（RL + 间隔 CBF）。
  • 技术技巧层面：(1) 非对称 Actor-Critic 是零代价的"免费午餐"——只要你在仿真中能获取真值状态，就让 Critic 利用它。Critic 仅存在于训练阶段，不参与部署，不影响推理速度或传感器需求。策略梯度定理保证梯度无偏——"训练时信息不对称，部署时信息公平"；(2) 输出姿态角而非电机指令——这是 sim-to-real 的黄金法则——将底层执行器动力学的复杂度（电机 RL 电路、螺旋桨气动、ESC 协议）委托给已经过数千飞行小时验证的 PX4 控制律，RL 只负责"纯几何层面"的决策（去哪里、多快）；(3) 4 维单约束 QP 几乎免费——不要因为"求解优化问题"听起来很重就回避 CBF。4 维变量 + 1 个约束的 QP 可用 KKT 条件解析求解（等价于沿约束梯度方向的加权投影），在嵌入式 GPU 上计算时间约 1 微秒——对 100Hz 的控制周期完全无感知。
  • 实验/写作层面：四组分层的消融实验是本论文实验设计的精髓——(a) 非对称 vs 对称 Critic（架构层面 → 训练效率），(b) 域随机化逐项移除（鲁棒性层面 → 真机成功率），(c) HOCBF 开关对比（安全层面 → 碰撞率），(d) 奖励函数逐项移除（行为层面 → 涌现属性）。特别值得一提的是：域随机化的消融分别测试了仿真和真机——仿真仅降 5.3%，真机暴降 35.5%——这个巨大的"仿真-真机消融差距"直接揭示了域随机化的核心价值（它的主要贡献不在仿真性能，而在 sim-to-real 桥接）。做 sim-to-real 研究时必须这样设计消融——否则单看仿真消融会严重低估域随机化的重要性。

11.2 Directly Applicable to My Research

  • 对于 RL + UAV 控制方向：非对称 Actor-Critic + 域随机化 + HOCBF 三件套构成一个完整的"学习型安全无人机控制"设计模板——可直接迁移到各类 UAV 任务：货物运输、电力线巡检、竞速、编队飞行、探索建图。迁移时的关键适配点（需要根据具体任务重新设计）：(a) 特权信息的具体内容——Critic 应访问哪些在真机上不可用的信息？（如真值风速、精确障碍物距离、电机内部温度）；(b) 域随机化的维度和强度——新任务的真实传感器/动力学噪声特性是什么？(c) 安全函数 $h(x)$ 的语义——新任务的"不安全状态"如何用数学表达式定义？
  • 对于视觉导航方向：100×60 的极低分辨率深度图仍能支持 7.5 m/s 的高速飞行——这个发现暗示：对于避障飞行，"几何结构的整体模式"（哪里有大的空隙、空隙的形状和宽度）远比"纹理细节"重要。6000 个像素的信息瓶颈恰好迫使 CNN 只关注最本质的几何信息——这反而提高了泛化能力。这为嵌入式视觉系统设计提供了重要的"降分辨率"优化依据——不需要 640×480 的深度图来进行避障，100×60 足矣。
  • 对于安全控制方向：HOCBF 是一个被严重低估的工具——它在数学上提供可证明的安全性（前向不变性），在计算上几乎零开销（< 1微秒的 QP），在系统集成上完全非侵入（不修改 RL 策略内部——仅修正输出）。将 CBF 加入任何现有控制 pipeline 不会产生负面影响：如果策略本身安全 → CBF 零干预（输出 = 输入）；如果策略偶尔越界 → CBF 以最小修改拉回安全区域。这是"锦上添花"式的安全层——不需要重新设计系统，就在控制输出端加一个 QP。
