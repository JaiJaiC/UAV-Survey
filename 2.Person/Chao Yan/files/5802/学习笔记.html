作者: Chao Yan (闫超) 等 · 领域: 多智能体强化学习 + 3D 无人机导航

Distance-Attention Multi-UAV 3D Navigation MADRL Urban Environments POMDP

1. 系统架构全景图

DA2RL 构建了一个三层协同架构，在密集城市环境中实现多无人机的安全协同导航：多层圆周感知模型 (Multilayer Circumferential Perception) → 距离注意力Actor网络 (Distance-Attention-Based Actor) → 历史特征流Critic网络 (Historical Feature Flow-Based Critic)。

1.1 整体框架：POMDP + CTDE

系统将多无人机 3D 协同导航建模为部分可观测马尔可夫决策过程 (POMDP)，采用集中训练-分散执行 (CTDE) 范式。每架无人机独立感知局部环境，通过共享的策略网络输出动作，训练时 Critic 获取全局信息进行价值评估。

系统架构三核心模块
模块功能核心创新
多层圆周感知模型 3D 空间的环境特征提取——多圈层射线检测障碍物/友机/目标 将连续 3D 空间离散化为多层圆周网格，同时捕获水平和垂直方向的空间信息
距离注意力Actor网络 基于注意力机制精炼观测空间，捕捉时序依赖 距离注意力——根据障碍物/友机的距离远近来加权信息重要性（近处=注意力高）
历史特征流Critic网络 利用历史观测序列做更准确的动作价值评估 历史队列机制——存储历史特征队列，让Critic懂得"过去到现在的趋势"

1.2 历史队列MADRL框架

多无人机协同训练的独特挑战是非平稳性 (non-stationarity)——其他无人机也在学习，环境从单个无人机的视角来看是持续变化的。DA2RL 通过历史队列 (Historical Queue) 来维护过去若干步的特征序列，帮助网络理解动态变化趋势，稳定协同训练。

核心设计思想：空间注意力 + 时序记忆
传统 DRl 导航方法只考虑当前观测。DA2RL 的两大创新——距离注意力（空间维度选择）和历史特征流（时间维度连贯）——分别解决"哪些信息重要"和"信息如何演化"两个关键问题，模拟了人类飞行员"盯着近处障碍 + 记住刚才看到的"这一天然行为模式。

2. 核心方法深度解析

2.1 多层圆周感知模型 (Multilayer Circumferential Perception)

在密集城市环境中，无人机面临的 3D 空间极其复杂——高楼林立、狭窄通道、动态障碍物。传统的栅格化 (voxelization) 方法计算量巨大，而简单的距离传感器容易遗漏关键信息。

传统方法的局限

  • Voxel网格：O(n³) 计算复杂度，绝大多数格子为空
  • 单层射线：只检测水平面障碍，忽略垂直方向（如阳台、桥梁）
  • 2D LiDAR：无法感知 3D 空间结构

多层圆周感知的设计

  • 多圈层：在球面上沿俯仰角方向划分多个"纬度圈"，每圈沿方位角方向等距发射射线
  • 每层射线：检测该方向到最近障碍物/友机/边界的距离
  • 编码：距离值 + 物体类型（障碍物=0，友机=1，目标=2）

2.2 距离注意力机制 (Distance-Attention)

这是 DA2RL 最核心的创新——不是所有感知到的物体都同等重要。距离注意力根据物体到自身的距离计算注意力权重：近处物体的注意力权重高（高分辨率处理），远处物体的注意力权重低（粗略感知即可）。

Distance-Attention
Attentioni = softmax( −di / τ )
其中 di 是第 i 个感知目标的距离，τ 是温度参数控制注意力集中度
距离越近 → −d/τ 越大 → softmax 后权重越高 → 网络更"关注"近处物体
参数含义效果
τ (温度)注意力分布的"锐度"τ 小 → 注意力极度集中在最近物体；τ 大 → 注意力更均匀分布
di到第 i 个物体的欧氏距离基础度量——距离近=威胁大=需要更多注意力
设计启发：这模仿了生物视觉系统中的中心凹 (fovea)机制——视网膜中心分辨率高，周边分辨率低。无人机也应该"盯着近处"，同时"余光扫远处"。

2.3 历史特征流 (Historical Feature Flow)

Critic 网络需要评估动作的长期价值。DA2RL 在 Critic 中引入历史特征队列——一个固定长度的 FIFO 缓冲区存储过去 K 步的特征向量。Critic 同时处理当前特征和历史序列，从而理解"状态变化趋势"——是正在接近障碍物还是正在远离？

无历史队列的Critic

V( s_t )  # 只看当前状态

# 问题：两架无人机在同一位置
# ——但一架在靠近障碍物，一架在远离
# Critic 无法区分！

有历史队列的Critic

V( s_t, h_t )
where h_t = [f_{t-K}, ..., f_{t-1}, f_t]

# Critic 能看到趋势：
# 序列[f远离, f接近, f更近] → 危险
# 序列[f近, f远, f更远] → 安全

3. 论文精读

3.1 核心贡献

  1. 首次将距离注意力机制引入多无人机 3D 协同导航，实现空间选择性感知
  2. 提出多层圆周感知模型，以较低计算开销捕获 3D 密集城市场景的空间结构
  3. 设计历史特征流 Critic 网络，利用时序信息提升动作价值估计的准确性
  4. 构建历史队列 MADRL 框架，在非平稳多智能体环境中实现稳定协同训练
  5. 在密集城市狭窄通道场景中验证多无人机协同避碰能力

3.2 POMDP 形式化

POMDP
POMDP = ⟨ N, S, {Ai}, T, {Ri}, {Ωi}, {Oi}, γ ⟩
N 架无人机，各自拥有局部观测和独立动作空间，共享环境动力学和奖励结构
符号含义DA2RL 中的具体对应
N无人机数量3~5 架，可扩展
S全局状态所有无人机位姿 + 障碍物 + 目标位置
Ai第 i 架无人机的动作空间3D 速度指令 [vx, vy, vz] 或加速度指令
Ωi第 i 架无人机的观测多层圆周感知向量 + 自身状态 + 目标信息
Ri奖励函数5 项非稀疏奖励（见第 4 节）

3.3 5 项非稀疏奖励函数

3D 导航的稀疏奖励问题是核心挑战之一——目标远在几百米外，随机探索几乎不可能碰巧到达。DA2RL 设计了 5 项密集奖励，每一项都提供即时的学习信号：

奖励项类型公式含义设计意图
Rtarget目标趋近Δdtarget（目标距离变化）鼓励向目标靠近——"每一步走近了多少"
Robstacle障碍规避−Σ exp(−dobs/σ)（距离惩罚）靠近障碍物=负奖励，且距离越近惩罚指数增长
Ranti-collision防撞−Σ exp(−duav/σ)（友机距离惩罚）防止无人机之间的碰撞——编队安全间距
Rconnectivity通信保持max(0, dcomm − dmax)（超距惩罚）确保无人机不飞出通信范围——协同的前提
Rtime时间步每个时间步固定小负值鼓励尽快到达目标，避免绕远路
非稀疏奖励的关键：每一项都提供每步的信号——不需要等到达终点才知道做得好不好。这使得无人机可以在训练早期就获得有意义的梯度，大幅加速收敛。

3.4 UAV 动力学模型

论文使用简化的 3D 质点模型（而非完整 6-DOF 动力学），侧重于导航决策而非底层飞行控制：

# 简化动力学：加速度指令 → 位置更新
v_{t+1} = v_t + a_t · Δt
p_{t+1} = p_t + v_t · Δt

# 约束
v_min ≤ |v| ≤ v_max      # 速度限制
a_min ≤ |a| ≤ a_max      # 加速度限制
h_min ≤ z ≤ h_max        # 飞行高度限制

3.5 实验场景

实验在两种典型城市环境中进行：

  1. 城市峡谷 (Urban Canyon)：两侧高楼，中间狭窄通道——模拟典型的城市街道飞行
  2. 密集建筑群 (Dense Buildings)：随机分布的高层+低层建筑，需要复杂的 3D 避障路径

两种场景均包含动态障碍物（移动的无人机/飞行器），测试算法的实时避碰能力。

4. 算法公式全解

4.1 多层圆周感知模型

MCP
oray = { (dk, typek) | k = 1, 2, ..., K }
K 条射线，每条射线返回最近碰撞点的距离和类型标签。多层：L 个俯仰圈 × M 个方位角 = L×M 条射线。
参数含义典型值
L (俯仰层数)垂直方向的感知层数3~5 层（上、平、下）
M (方位角分辨率)水平方向的射线密度16~32 条/层
typek目标类型编码0=障碍物, 1=友机, 2=目标点, 3=边界
dmax传感器最大量程50~100m

4.2 距离注意力机制

Eq.(1)
wi = exp(−di / τ) / Σj exp(−dj / τ)
softmax 归一化的负距离指数权重——距离越小权重越大。τ 控制"注意力范围"。
Eq.(2)
fatt = Σi wi · fi
加权求和：将所有感知特征按距离注意力权重聚合为统一的注意力特征向量

距离注意力的效果可视化

场景最近障碍物距离注意力分布网络行为
开阔空域80m均匀分布在多个远距离物体自由导航，主要追踪目标方向
接近建筑15m高度集中在近处建筑精确避障，小心调整航向
狭窄通道3~5m极度集中在两侧墙壁精细控制，保持居中通过
友机靠近8m集中在友机上主动避让，保持安全间距

4.3 历史特征流 Critic

Eq.(3)
ht = Queue( ft-K, ft-K+1, ..., ft )
FIFO 队列，固定长度 K。每当新特征 ft 加入，最老的 ft-K 被移出。
Eq.(4)
V( st, ht ) = MLPcritic( [ st ; Flatten(ht) ; at ] )
Critic 输入：当前状态 + 历史特征队列的扁平化拼接 + 当前动作 → 输出 Q 值估计

4.4 完整奖励函数

Eq.(5)
Rtotal = w1·Rtarget + w2·Robstacle + w3·Ranti-collision + w4·Rconnectivity + w5·Rtime
奖励项公式权重贡献特征
Rtargetdtarget(t−1) − dtarget(t)w1正奖励鼓励靠近目标
Robstacle−Σi exp(−dobs,i / σobs)w2指数衰减惩罚——只有在真正接近障碍物时才有显著惩罚
Ranti-collision−Σj exp(−duav,j / σuav)w3友机间距太小 → 指数级惩罚
Rconnectivitymax(0, dcomm − dmax)w4超出通信范围 → 线性惩罚
Rtime−c（常数）w5每步固定负值，促进快速到达
指数衰减惩罚的巧妙：exp(−d/σ) 在距离远时几乎为 0（≈0），在距离近时急剧增大。这意味着无人机可以自由飞过开阔区域，但在接近障碍物时会收到强烈警告——这是一种"软约束"，比硬终止条件（撞到就结束）更有利于训练初期的探索。

4.5 Actor-Critic 策略优化

Actor Loss
Lactor = −E[ Q(s, a) ] + α · H(π)
最大化 Critic 评估的动作价值 + 熵正则化（鼓励探索）
Critic Loss
Lcritic = E[ ( Q(s, a) − (r + γ·Q'(s', a')) )² ]
TD 误差最小化：让 Critic 的估计逼近真实回报

6. RL 训练详解 —— 多智能体协同训练的工程挑战

6.1 CTDE 训练框架

DA2RL 采用集中训练-分散执行 (CTDE) 框架：

训练阶段 (Centralized Training)

  • Critic 可以访问所有无人机的状态+观测
  • Critic 基于全局信息评估每个动作的价值
  • 历史特征队列在训练时维护——提供时序上下文
  • 多无人机共享同一 Critic 网络

执行阶段 (Decentralized Execution)

  • 每架无人机只用自己的局部观测
  • Actor 网络独立运行，不需要全局信息
  • 不需要与友机通信（虽然系统假设通信存在）
  • 多无人机共享同一 Actor 网络（参数共享）

6.2 历史队列训练机制

历史特征队列是 DA2RL 训练稳定性的关键工程组件：

# 历史队列的工作流程
queue = FIFO_Queue(maxlen=K)

for episode in range(num_episodes):
    queue.clear()  # 每个 episode 开始清空历史
    for step in range(max_steps):
        obs = env.get_obs(uav_id)
        f_att = distance_attention_encoder(obs)
        queue.push(f_att)  # 入队
        h_t = queue.get_all()  # 获取整个历史窗口
        a = actor(obs)  # Actor 只用当前观测
        Q = critic(obs, h_t, a)  # Critic 用当前+历史

6.3 训练参数

核心训练超参数
学习率 (Actor)1×10⁻⁴
学习率 (Critic)3×10⁻⁴（比 Actor 高——价值函数更新更快）
折扣因子 γ0.99
软更新系数 τ0.005 (Target 网络缓慢跟随)
历史队列长度 K10~20 步（平衡时序信息与计算开销）
距离注意力温度 τ5~15（控制注意力集中度）
无人机数量3~5 架（可扩展）
每 episode 最大步数500~1000 步

6.4 多智能体训练的特殊挑战

挑战原因DA2RL 的应对
非平稳性友机策略也在变化 → 环境动态变化历史队列提供时序平滑，减缓感知到的变化速度
信用分配全局奖励难以分解到每个无人机5 项独立奖励中 4 项是局部可计算的
维度灾难N 架无人机 → 观测/动作空间 ×N局部观测 + 距离注意力压缩，避免维度爆炸
协调复杂狭窄通道中需要精细排队距离注意力自动关注最近的友机 → 隐式学习排队行为

8. 科研 Tricks —— 从 DA2RL 学到的工程智慧

1

距离注意力 — 让网络"盯着近处"

核心思想：在 3D 导航中，近处障碍物比远处重要得多。直接用距离的负指数作为注意力权重。

w_i = softmax(-d_i / τ)
# d=1m: -1/10=-0.1 → softmax → 高权重
# d=100m: -100/10=-10 → softmax → 接近0

可迁移场景：任何"近处信息比远处重要"的任务——机器人避障、自动驾驶、机械臂操作。

2

多层圆周感知 — 3D 空间的轻量编码

问题：3D Voxel 网格太大，2D LiDAR 信息不全。

方案：在球面上选 L 个纬度层，每层 M 条射线 → L×M 个距离值。同时捕获水平和垂直的空间结构。

L=3（上/平/下）就能区分"需要爬升"还是"需要下降"的障碍物。

3

指数衰减惩罚 — "软约束"的艺术

障碍物惩罚 R_obs = −exp(−d/σ) 有两个关键性质：

  • 远处惩罚 ≈ 0：自由探索不受限
  • 近处惩罚 ↑↑：梯度急剧增大，强烈警告

比"撞到就 reset"更适合训练早期——不需要撞墙才能知道要避开。

4

历史队列 — 为 Critic 提供时序上下文

Actor 只用当前观测做决策，Critic 用当前+历史序列做评估。

好处：Critic 能识别"接近危险"和"远离危险"这两种趋势——而不仅仅看当前位置。训练 Critic 更准确 → Actor 更新的梯度信号更可靠。

5

5 项密集奖励 — 对抗 3D 稀疏导航

3D 导航的奖励极度稀疏。DA2RL 将奖励分解为 5 个每步可计算的分量：目标趋近、障碍规避、防撞、通信保持、时间步。

每项都是连续信号而不是稀疏的 0/1——训练初期就能获得有意义的梯度。

6

多层圆周感知 + 类型编码

射线不仅返回距离，还返回碰撞类型标签（障碍物/友机/目标/边界）。

这使得单一感知模块能同时用于：避障（避开 type=0）、防撞（避开 type=1）、导航（靠近 type=2）——一个感知器覆盖所有需求。

7

参数共享 — 多智能体的简单高效策略

N 架无人机共享同一个 Actor 和 Critic 网络。所有 UAV 是同构的——这是参数共享的前提。

好处：训练样本数 ×N（每架无人机的经验都用于更新同一个网络），训练速度 ×N。

8

在困难场景上做消融对比

DA2RL 的消融实验在狭窄通道这个最困难的子场景上进行——而非在所有场景上平均。

在简单场景上所有方法都差不多，区分度来自困难场景——这确保消融结论的可靠性。

9. 设计哲学 —— 空间注意力 + 时序记忆 = 类人导航

人类飞行员怎么做？

  • 盯着近处：几乎不关心 500 米外的电线杆
  • 记住刚才：知道 3 秒前有个障碍物在左边，现在大概在左后方
  • 软规避：不会撞到东西才急转弯——提前就微调了
  • 协调通行：狭窄通道中自动排队，不会硬挤

DA2RL 如何实现？

  • 距离注意力 = 自动聚焦近处物体
  • 历史特征流 = 记住过去几步的感知结果
  • 指数衰减惩罚 = 提前获得"危险"预警
  • 参数共享 + 局部观测 = 隐式学习协同行为

为什么距离注意力比标准注意力更好？

注意力类型权重计算在 3D 导航中的行为
标准 Self-AttentionQ·K^T / √d_k → softmax权重由特征相似度决定——可能关注远处某个"看起来像障碍物"的东西
距离注意力−d / τ → softmax权重仅由物理距离决定——必然关注最近的危险

关键洞见：在导航任务中，"距离"是对注意力权重的最好先验。标准注意力需要数据驱动学习这个模式，而距离注意力直接把物理知识注入网络结构——更少的训练数据，更可靠的注意力分布。

DA2RL 的核心方法论
不是用更复杂的网络暴力拟合一切，而是在网络结构中注入物理先验（距离=注意力），让网络专心学习它需要学习的东西。
多层圆周感知+距离注意力+历史特征流+5项密集奖励——每一项都是对 3D 导航问题的结构化理解，而不是"多堆几层 Transformer"。

DA2RL 的实操流程

  1. 定义 POMDP：N 架无人机 → 每架有局部观测和独立动作
  2. 设计感知器：多层圆周射线 + 类型编码 → 紧凑且信息丰富
  3. 注入物理先验：距离注意力——远小近大，自动聚焦危险
  4. 添加时序记忆：历史队列 → Critic 看到趋势而不仅是状态
  5. 设计密集奖励：5 项每步奖励 → 解决 3D 导航的稀疏奖励问题
  6. CTDE + 参数共享：训练用全局信息，执行只用局部；同构共享网络
  7. 在困难场景上验证：狭窄通道 → 区分度最高的子任务

10. 学习路线 & 改进方向

学习路线

  1. 理解 POMDP + CTDE：多智能体 RL 的基本框架——为什么需要 CTDE？什么是非平稳性？
  2. 实现多层圆周感知：在仿真中编码 3D 空间的射线检测——L 层 × M 条射线的工程实现
  3. 实现距离注意力：softmax(−d/τ) 的编码——调 τ 观察注意力分布的变化
  4. 实现历史队列 Critic：FIFO 队列 + 时序特征扁平化 → 对比有无队列的 Critic 性能
  5. 设计 5 项奖励：逐项调权——先跑随机策略记录各分量平均值 → 归一化到一致量级
  6. 多智能体训练：参数共享 + CTDE → 观察非平稳性如何影响收敛
  7. 在困难场景上消融：移除单个模块（距离注意力/历史队列/历史特征流）→ 量化贡献
