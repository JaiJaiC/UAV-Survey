1. 系统架构全景图

AC-MPC 的核心闭环：观测 → Cost Map网络(Q,p) → 可微MPC求解QP → 动作(集体推力+体轴角速率) → 四旋翼动力学 → 新观测。绿色区域是Critic（仅训练时使用），负责估计长期价值。

2. 论文精读

2.1 核心研究问题

🎯 Open Research Question
如何将无模型RL（任务灵活、通用奖励优化）的优势与MPC（鲁棒性、在线重规划）的优势结合起来？之前的方案要么把MPC当黑箱（梯度靠采样估计[42]），要么用黑箱优化调MPC参数[29-32]——都无法利用MPC内部的结构信息。本文的回答是：让MPC可微，梯度解析传播。

2.2 核心贡献

  1. 提出AC-MPC架构：将可微MPC作为Actor网络的最后一层，嵌入PPO训练框架
  2. 学习代价函数而非策略：神经网络从观测映射到MPC代价参数（Q_k, p_k），而非直接输出动作——MPC作为solver保证物理可行性
  3. 梯度解析传播：利用可微QP求解器[26]，使梯度通过优化问题本身传播——与之前"黑箱MPC+RL"有本质区别
  4. 两个时间尺度：MPC处理短视界（N步内精确预测），Critic处理长视界（价值估计）——生物启发的设计（系统1/系统2）
  5. 验证鲁棒性优势：抗风扰（11.5N=1.5倍自重）实现83.33%成功率对比AC-MLP的6.5%和纯MPC的0%

2.3 四旋翼模型与感知

动力学模型与Song et al. [14]相同。观测空间分两部分：

  • 机体观测 o_quad = [v, R] ∈ R¹²（线速度 + 旋转矩阵）
  • 赛道观测 o_track = [δp_1, ..., δp_N]：无人机到未来N个门角的相对位置向量——支持任意数量的未来门

控制输入为集体推力 + 体轴角速率（由MPC从单旋翼推力计算得到，保证物理可行性）。所有任务使用相同的观测空间定义，策略从零开始重新训练。

3. 算法公式全解 —— 从MPC到PPO的完整数学链路

论文的数学体系分为四个层次：MPC问题建模 → 二次型代价参数化 → 可微MPC梯度传播 → PPO策略优化。

3.1 非线性MPC → QP近似

Eq.(1)
min_{x,u}   Σ_{k=0}^N l_k(x_k, u_k)   s.t.   x_{k+1} = f(x_k, u_k),   u_k ∈ U
原始的非线性MPC问题——由于实时性要求，通常近似为QP
Eq.(2)
J_Q(x) = Σ_{k=0}^N [x_k; u_k]^T Q_k [x_k; u_k] + p_k [x_k; u_k]
QP形式：Q_k 是二次项矩阵（对称），p_k 是线性项向量。Q_k 和 p_k 就是Cost Map网络的输出
为什么要学习 Q_k 和 p_k？
传统的MPC中，Q和p是手工调参的（比如Q=diag(10,10,1,1,1,1)）。不同任务需要不同的代价——水平飞行和垂直飞行的最优代价肯定不同。
本文用神经网络直接从观测学习Q_k和p_k——"学习代价函数"而非"学习控制策略"。MPC仍然求解优化问题，但优化的目标函数是学出来的。

3.2 Actor-Critic 策略梯度

Eq.(3)
∇_θ J(π_θ) = (1/N) Σ_{i=1}^N Σ_{k=1}^T ∇_θ log π_θ(a^i_k | s^i_k) · A_ω(s^i_k, a^i_k)
A(s_k, a_k) = r(s_k, a_k) + γ V_ω(s_{k+1}) − V_ω(s_k)
标准Actor-Critic：策略梯度 = 对数概率梯度 × 优势函数。关键差异在于π_θ的结构——本文的Actor包含一个可微MPC层

3.3 AC-MPC 策略（核心公式）

Eq.(4) — 核心
u_k ∼ N { diffMPC(x_k, Q(s_k), p(s_k)),   Σ }
u_k控制指令（集体推力 + 体轴角速率）——从高斯分布采样
diffMPC(·)可微MPC求解器——输入当前状态x_k和学到的代价参数Q(s_k), p(s_k)，输出最优控制序列的均值
Q(s_k), p(s_k)Cost Map神经网络的输出——从观测到代价函数的映射
Σ协方差矩阵（可学习的探索噪声）
s_k观测（不是完整状态）——包含机体状态和赛道信息
🔑 Eq.(4) 是整个论文的灵魂
与标准Actor-Critic的区别：Actor的输出不是动作，而是代价函数的参数。动作是通过求解MPC得到的。
与纯MLP-Actor的区别：MLP是"观测→动作"的黑箱映射；AC-MPC是"观测→代价函数→MPC求解→动作"，中间有明确的物理模型。
梯度链路：reward → advantage → policy gradient → ∂(MPC solution)/∂(Q,p) → ∂(Q,p)/∂θ → update Cost Map NN

3.4 可微MPC——技术核心

本文依赖Amos & Kolter (ICML 2017) [26]的可微QP求解器。关键特性：

  • 支持输入约束 u_k ∈ U（如单旋翼推力上下限），但不支持状态约束（如避障）
  • 通过求解QP的KKT条件，解析地计算最优解对代价参数(Q, p)的梯度
  • 梯度通过前向/反向传播：每次前向传播需要完整求解一个QP，反向传播需要求解一个相关的线性系统
⚠️ 计算代价
每次forward+backward都需要求解QP。Training时必须反复求解，推理时只需要forward。这解释了为什么训练时间比AC-MLP长（Table II），但推理时间仍在毫秒级（N=2: 2.5ms, N=5: 3.5ms, N=10: 5.5ms）。

3.5 代价函数Q_k和p_k的结构

在标准MPC中，Q_k通常是对角矩阵或块对角矩阵。本文直接让Cost Map网络输出Q_k和p_k的所有独立参数。对于N步视界，Q_k和p_k的参数量为O(N)，由网络一次性输出。

💡 为什么学Q_k和p_k而不是学参考轨迹？
之前的工作[29]学的是参考轨迹的偏移量——这让MPC追踪一条"最优轨迹"，但如果环境变了（比如突然来风），轨迹就不对了。
学代价函数的好处：MPC仍然自己求解最优控制，所以环境变化时它会自动调整——代价函数告诉它"什么状态好/什么状态坏"，而不是"你应该去哪个位置"。这本质上更鲁棒。

4.1 训练时的一步数据流

Flightmare Simulator (simple model, fast training) │ ├─ Observation Extraction: │ o_quad = [v_x, v_y, v_z, R(3×3)] ← 12D vehicle state │ o_track = [Δp_corner1, Δp_corner2, Δp_corner3, Δp_corner4] × N_gates │ → concatenated into full observation vector │ ├─ FORWARD PASS (Actor): │ │ │ ├─ Cost Map NN: observation → FC layers → Q_k, p_k (for k=0..N) │ │ Q_k: symmetric positive-semidefinite matrices │ │ p_k: linear cost vectors │ │ │ ├─ Differentiable MPC: │ │ solve min Σ [x_k;u_k]ᵀQ_k[x_k;u_k] + p_k[x_k;u_k] │ │ s.t. x_{k+1} = Ax_k + Bu_k, u_k ∈ U │ │ → optimal control sequence mean μ = [u_0*, ..., u_N*] │ │ │ └─ Sample: u_0 ~ N(μ[0], Σ) ← stochastic Gaussian policy │ ├─ Apply Action: │ u_0 → collective thrust + body rates │ → individual rotor thrusts (guaranteed feasible by MPC constraints) │ ├─ Simulate: Flightmare step → next state │ ├─ FORWARD PASS (Critic): │ V(s_k) ← from Critic network │ V(s_{k+1}) ← from Critic network │ ├─ Compute Reward: │ r = progress reward (gate center distance reduction) │ + perception reward (camera-point alignment, optional) │ ├─ Store in Buffer: (s_k, a_k, r_k, s_{k+1}, V(s_k), log_prob(a_k)) │ └─ PPO UPDATE (after collecting batch): │ ├─ Compute GAE advantages: │ A_k = Σ (γλ)^i · δ_{k+i} │ where δ_t = r_t + γ V(s_{t+1}) - V(s_t) │ ├─ Compute policy gradient through MPC: │ ∂log π_θ/∂θ = ∂log π_θ/∂μ · ∂μ/∂(Q,p) · ∂(Q,p)/∂θ │ ↑ sampled ↑ diff MPC grad ↑ NN backprop │ └─ Update θ (Actor) and ω (Critic) via Adam

🏆 阵风实验：为什么83.33% vs 6.5%？
阵风施加11.5N外力（1.5倍自重），从z=10m到z=8m。
AC-MLP失败的原因：纯神经网络学到的"观测→动作"映射在训练分布外完全不可靠——没有遇到过这么大的风，不知道怎么应对。
MPC失败的原因：它追踪一条在无风条件下计算的最优轨迹——风来了，轨迹偏差了，它试图回到轨迹但控制权限不够。
AC-MPC成功的原因：MPC层内置了动力学模型——它知道风在把它吹偏，在线重规划计算出新的最优控制，而学到的代价函数告诉它"快速回到赛道比保持原轨迹更重要"。

5.3 消融实验：MPC视界N的影响 (Fig.3B)

训练曲线：AC-MPC用不同视界N(2,5,10)与AC-MLP对比。AC-MPC略落后于AC-MLP（结构性约束的代价），但三者最终接近。N越大，MPC看得越远，但计算成本更高、学习更难。

5.4 初始条件鲁棒性 (Fig.3C)

在水平赛道中从不同初始位置出发，完成50次试飞。AC-MPC的轨迹束明显更紧（方差小），几乎全是蓝色（成功）；AC-MLP有不少红色（碰撞）。这证明MPC层使策略对初始条件变化更鲁棒。

5.5 Sim-to-Real (Fig.4)

圆形赛道和SplitS赛道在真机上零样本部署成功。仿真中的速度分布和轨迹形状与真机高度一致。这证明：

  1. BEM高保真仿真器足够好（气动模型接近真实）
  2. MPC层的动力学模型在真机上也适用
  3. 学到的代价函数对sim-to-real gap不敏感（因为MPC负责处理动力学差异）

5.6 感知感知飞行 (Fig.5)

同时优化飞行速度和相机指向——一个competing reward问题。AC-MLP和AC-MPC都学到了这个行为，但轨迹呈现出"非自然"的形状——因为需要在速度和感知之间trade-off。AC-MPC的轨迹更平滑。

6. 科研 Tricks —— 哪些可以直接用到你的项目里

1

"学习代价函数"而非"学习动作"

核心思想：不让NN直接输出动作，而是输出MPC的代价参数，让MPC去求解动作。这给策略注入了物理结构先验（动力学模型+约束），大幅提升泛化能力。

代价：牺牲一点样本效率 + 每次forward都要解QP。收益：OOD鲁棒性质的飞跃。

2

两个时间尺度的分工

MPC只负责短视界（N步），Critic负责长视界（无限视界折扣回报）。这避免了让MPC处理无限视界（计算不可行），也避免了让纯RL预测短期物理约束。

生物启发：大脑也有"快系统"（直觉/反应）和"慢系统"（推理/规划）——本文的架构正好对应。

3

可微性 > 黑箱优化

之前的工作[29][42]把MPC当黑箱，用有限差分或贝叶斯优化调参数——计算量爆炸、梯度噪声大。本文用可微MPC[26]，梯度解析计算——几百倍效率提升。

这是一个重要的技术趋势：让传统控制模块可微，嵌入DL pipeline。

4

简单仿真器训练 + 高保真仿真器评估

训练用Flightmare（简单快速），评估用BEM（高保真气动）。训练不需要最高精度——RL需要的是大量试错；评估才需要精度。

这个策略可以节省大量训练时间，同时保证结果可信。

5

观测空间的一致性设计

所有任务（水平/垂直/圆形/SplitS/感知感知）使用完全相同的观测空间。这是好的架构设计——说明观测表示是通用的。如果你的系统需要为每个新任务重新定义观测，那可能观测设计不够好。

6

消融实验设计的层次感

三个维度的消融：(a) wind disturbance → 验证OOD鲁棒性；(b) horizon N → 验证MPC视界影响；(c) initial conditions → 验证对初始状态的鲁棒性。

每个消融只改变一个变量，清晰干净——值得学习。

7

MPC层提供"免费的"输入可行性保证

MPC内部有u_k ∈ U约束，确保输出的控制不会超出执行器物理范围。纯RL可能需要clip或tanh来约束——但那是"硬截断"而不是"最优约束满足"。

对于安全关键系统，这个性质非常有价值。

8

Q_k和p_k的参数量与视界N的关系

Cost Map网络一次性输出所有Q_k和p_k（k=0..N）。这意味着输出维度随N线性增长。论文通过实验找到了trade-off（N=5效果最好）。

如果你用类似架构，要注意网络输出维度和视界的trade-off——N太大可能导致网络容量不够。

7. 设计哲学 —— "结构化策略" 的方法论

❌ 常见误区

  • 把所有问题塞进一个大MLP端到端解决
  • 认为"更多数据+更大网络"就能解决OOD问题
  • 把MPC当黑箱，用采样估计梯度
  • 只为单一任务设计观测和网络
  • 忽视传统控制理论几十年的积累

✅ 本文的做法

  • 结构化最后一层：MPC作为策略的"执行器"，保证物理可行性
  • 学习代价函数而非策略：保留MPC的鲁棒性，用RL优化目标
  • 可微性：让梯度流畅通，高效训练
  • 通用观测空间：一次设计，多任务复用
  • 尊重物理：MPC层内置动力学，RL只学"什么状态好"
🎯 本文的核心方法论
不要用神经网络替代物理模型——让神经网络增强物理模型。
MPC知道"怎么飞"（动力学），但不知道"飞去哪是好的"（代价函数）。
RL知道"什么是好的"（reward），但不知道怎么满足物理约束。
两者的结合点：NN输出代价函数，MPC根据代价函数+动力学求解最优控制。
这就是"结构化策略"的精髓——在策略结构中嵌入已知的物理结构，而不是让数据从头学。

与纯RL（AC-MLP）的本质区别

维度AC-MLPAC-MPC
策略表示观测 → NN → 动作观测 → NN → 代价 → MPC → 动作
物理知识无（从数据中学）有（MPC内置动力学+约束）
泛化机制插值（在训练数据分布内）外推（MPC在线重规划）
安全性无法保证输入可行性由MPC约束保证
训练速度快慢（每个step解QP）

8. 对 UAV 编队 / 多机研究的迁移思路

8.1 直接迁移：分布式编队 MPC + RL

💡 迁移方案
将AC-MPC的架构从单机扩展到多机编队：
每个UAV的Actor = Cost Map NN（学编队代价） → 分布式MPC（解编队QP，含邻居避碰约束） → 控制指令
Critic = 估计编队整体的长期价值（任务完成度、能耗、通信质量等）
Cost Map学习什么？编队保持的权重、避障的激进程度、速度vs精度的trade-off——这些在不同场景下应该不同（空旷→高速，狭窄→精密）

8.2 三个具体可做的方向

方向难度描述
AC-MPC + 仿射变换 ⭐⭐⭐ 将仿射变换（Liu et al. 2025）计算的目标位置作为MPC的参考轨迹，Cost Map学习如何trade-off跟踪精度和避障——结合两篇论文的优势
多机分布式AC-MPC ⭐⭐⭐⭐ 每机独立Cost Map + 独立MPC（含邻居UAV作为动态约束），Critic共享或通过通信融合——实现分布式编队学习
AC-MPC + 固定翼6-DOF ⭐⭐⭐⭐⭐ 将四旋翼的AC-MPC迁移到固定翼（JSBSim/OpenFlight环境），MPC层使用固定翼线性化动力学——挑战在于固定翼的动态耦合更强，QP的建模精度可能不够

8.3 关键挑战

⚠️ 多机场景下需要注意的问题
  • 维度爆炸：N机编队的联合QP维度 = N × 单机维度。直接用集中式MPC不可行
  • 分布式QP的梯度传播：每个UAV的MPC只优化自己的控制——梯度的信息流被截断，可能丢失编队层面的协调信息
  • 非平稳性：其他UAV的策略也在更新——标准的MARL挑战
  • 通信延迟：真机部署时邻居状态可能有延迟，MPC的模型精度受影响
  • 避碰约束：本文的可微MPC[26]不支持状态约束（如||p_i - p_j|| ≥ r_safe），需要改用支持状态约束的可微MPC或把避碰放进代价函数
