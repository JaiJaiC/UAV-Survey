2. Insight & Novelty

2.1 Core Insight

🎯 核心洞察
问：为什么强化学习（RL）和模型预测控制（MPC）各自存在根本性缺陷？
答：纯 RL（AC-MLP）的 Actor 是一个随机初始化的 MLP 前馈网络——没有利用任何系统动力学先验知识，训练初期完全随机探索，面对分布外场景（强风扰、新初始条件）仅能依赖网络记忆泛化，没有在线重规划能力。传统 MPC 虽然利用动力学模型进行在线优化，但代价函数必须手工设计且固定不变——在需要"模式切换"的复杂任务（如垂直俯冲中翻转机身）中，固定代价函数容易陷入局部最优（保守的零推力自由落体），且每个新任务都需要重新手工设计代价函数。

问：本文的核心洞察是什么？
答：将可微分 MPC 作为 Actor 网络的最后一层，由 Neural Cost Map 从观测中动态学习 MPC 的代价函数参数。这样实现三个关键解耦：(1) Cost Map 学的是"当前应该优化什么目标"（代价函数），而非"当前应该输出什么动作"——学"目标"比学"动作"在动力学变化时更泛化；(2) MPC 负责"如何在给定目标下求最优动作"——利用已知动力学模型，输出始终满足物理约束；(3) Critic 负责"从长远看这个状态好不好"——其价值函数 V(s) 自动编码了"门通过后目标应切换到下一个门"的任务级模式，通过 Advantage 信号引导 Cost Map 向高回报方向更新。本质上是将仿人脑的双时间尺度预测机制（短期预测=MPC，长期预测=Critic）嵌入 RL 训练框架。

2.3 Novelty Breakdown

Novelty 1: 架构创新——将可微分 MPC 作为 Actor 最后一层，构成结构化策略网络

【解决的问题】纯 RL（AC-MLP）的 Actor 是黑箱 MLP，训练初期完全随机、动作可能违反物理约束、面对分布外场景缺乏重规划能力。传统 MPC 代价函数需手工设计、固定不变，无法适应任务阶段的动态变化。
【灵感来源】可微分优化层（Differentiable Optimization Layer）的思想——如果 MPC 本质上是求解一个优化问题，而该优化问题可微，那它就可以像普通神经网络层一样插入计算图。Actor 的输出不是动作，而是传递给 MPC 的代价函数参数——MPC 才是真正的"动作生成器"。
【具体设计】观测 $s_k$ → Cost Map MLP → 输出 $Q(s_k), p(s_k)$（MPC 二次代价函数参数）→ diffMPC 求解器 → 最优控制序列 $u^*_{0:N-1}$ → 取 $u^*_0$ + 探索噪声 → 执行。即使 Cost Map 输出不完美，MPC 始终保证动作满足动力学约束和输入限制——这个"安全层"效应在训练初期和分布外场景中至关重要。

Novelty 2: Neural Cost Map——从观测到 MPC 代价函数的端到端可学习映射

【解决的问题】标准 MPC 的代价函数参数（Q 矩阵权重、R 矩阵权重、线性项 p）需要手工设计。对于敏捷飞行——穿越一系列空间分布的门——需要代价函数在不同飞行阶段有完全不同的参数配置。手工编码这种"模式切换逻辑"几乎不可能覆盖所有场景。
【灵感来源】观测中包含了"当前目标门在哪里"的信息——代价函数应该是观测的函数，而非全局常数。将代价函数参数化为一组由神经网络预测的观测相关量，就能通过 RL 自动学习"在每个状态下应该优化什么"。
【具体设计】Cost Map = 2 隐藏层（各 512 维）+ ReLU + Sigmoid 输出层。Q 矩阵约束为对角阵（降维：从 $O((n+m)^2)$ 到 $O(n+m)$）。Sigmoid 输出范围 $[0.1, 100000]$——下限 0.1 确保 Q 正定（QP 有解），上限 100000 防止参数发散。输出维度 = $2T(n_{\text{state}} + n_{\text{input}})$（N=2 时为 68 维）。Cost Map 通过 PPO 策略梯度端到端训练，梯度经 diffMPC 的 KKT 隐式微分反向传播。

Novelty 3: 双时间尺度决策——MPC 短时域优化 + Critic 长时域价值估计协同驱动模式切换

【解决的问题】MPC 的预测时域 N=2（约 40ms）太短——无法"看到"远处的下一个门。如果仅靠 MPC 的短视优化，飞行器会在通过当前门后没有动力转向下一个目标。稀疏奖励（仅过门时 +10.0）加剧了这一问题——在数百个中间步中没有任何正奖励信号。
【灵感来源】Critic 的价值函数 V(s) 天然编码了"从状态 s 到任务结束的总期望回报"——这是无限时域的。当 Critic 学会"离下一个门更近 = 更高价值"后，Advantage 函数 A(s,a) = r + γV(s') - V(s) 会引导 Cost Map 输出能让飞行器朝高价值区域移动的代价函数参数。
【具体设计】Fig. 2 的价值函数热力图直接展示了这一机制——每通过一个门，高价值区域（黄色）立即从旧门"跳变"到新门。这种离散的模式切换不是手工编码的（没有 if-else），而是 Critic 从稀疏奖励中通过 TD 学习自动涌现的。Cost Map 将 Critic 的长期价值信号转化为 MPC 可执行的短期代价函数——这桥接了"长期目标"与"短期优化"之间的时间尺度鸿沟。

4. Task — What Problem Does It Solve?

4.1 Problem Formulation

核心问题 $$ \max_{\theta,\phi} \; \mathbb{E}_{\tau \sim \pi_\theta} \left[ \sum_{t=0}^{T} \gamma^t r(s_t, a_t) \right] \quad \text{s.t.} \quad a_t \in \mathcal{U},\; x_{t+1} = f(x_t, a_t) $$ $$ \text{其中} \quad a_t \sim \mathcal{N}\big(\text{diffMPC}\big(x_t, Q_\theta(o_t), p_\theta(o_t)\big), \sigma^2\big) $$
$\pi_\theta$ = 参数为 $\theta$ 的策略（Actor，包含 Cost Map MLP + 可微分 MPC 求解器） |  $V_\phi$ = 参数为 $\phi$ 的价值函数（Critic），估计从状态 $s$ 出发的无限时域折扣总回报  |  $r(s_t, a_t)$ = 即时奖励：门进度（稠密）+ 过门 +10（稀疏）+ 碰撞 -10 + 完赛 +10  |  $f(x_t, a_t)$ = 四旋翼 13 维非线性动力学模型（位置、速度、四元数姿态、角速度） |  $Q_\theta(o_t), p_\theta(o_t)$ = Neural Cost Map 从观测 $o_t$ 预测的 MPC 二次代价函数系数  |  $\text{diffMPC}(\cdot)$ = 以 $Q, p$ 为代价函数求解 MPC 得到的最优控制序列首项 $u^*_0$  |  核心约束：(1) 控制输入必须在物理可行范围内（单电机推力上下界，映射到集体推力+体速率后约束）；(2) 部署时需实时求解 MPC（推理时间 13.5ms @ N=2，满足 50Hz 控制频率）；(3) Critic 仅在训练时使用，部署时丢弃。

类别内容维度/范围
输入（观测）飞行器状态：线速度（3D）+ 旋转矩阵（3x3=9D）= 12D；赛道信息：未来 N=2 个门的 4 角点相对位置（8角×3D=24D）；经 running mean-std 在线归一化36 维（12 veh + 24 track）

4.3 Task Type

这是一个学习与控制架构的系统级创新——不是纯控制理论（不提出新的 MPC 变体），不是纯 RL（不提出新的 RL 算法），而是在架构层面将可微 MPC 作为 RL 策略网络的归纳偏置（inductive bias）嵌入 Actor-Critic 框架。验证场景：四旋翼敏捷穿门飞行（horizontal / vertical / circular / SplitS / perception-aware 五种赛道配置）。核心验证点：(1) 能否学习需要"模式切换"的复杂机动（垂直俯冲——处于输入空间奇点，需要翻转机身使推力向下）；(2) 对训练分布外扰动的鲁棒性（风扰 11.5N、初始条件从 1m³ 扩大到 3m³）；(3) 零样本 sim-to-real 迁移（仿真训练的 Cost Map 直接部署到真实四旋翼，无微调、无域随机化）。

5. Challenge — Why Is It Hard?

5.1 Essence (First Principles)

💡 第一性原理诊断
核心矛盾：端到端学习的灵活性与模型驱动控制的鲁棒性之间存在根本性的"表达能力 vs 泛化能力"权衡。

（1）纯 RL 的"无先验"脆弱性：AC-MLP 的 Actor 是一个随机初始化的 MLP——从零学习 s → a 映射，没有任何物理先验。这导致：(a) 训练初期完全随机探索，动作可能违反物理约束（超出推力极限、要求不可能的姿态变化率）；(b) 学到的映射本质上是"记忆"——训练分布内的 s 映射到 a，分布外的 s 只能靠插值/外推，无任何理论保证；(c) 没有在线纠正机制——一旦因扰动偏离预期轨迹，无法根据当前状态重新计算最优动作。实验证据：AC-MLP 在 11.5N 风扰下成功率仅 6.5%（vs AC-MPC 83.33%）。

（2）传统 MPC 的"手工设计"天花板：标准跟踪 MPC 需要预计算参考轨迹和手工设计代价函数权重。(a) 参考轨迹的生成本身是一个难题——在垂直俯冲任务（需要翻转机身使推力向下加速下降）中，时间最优轨迹可能包含奇异性（欧拉角描述的姿态在翻转点附近数值条件恶化），导致轨迹优化器失败或输出保守解；(b) 代价函数权重是固定的——无法在"平飞保持高度"和"翻转后全力加速"两个阶段之间自动切换；(c) 手工调参是一个新任务一套参数的不可扩展过程。实验证据：跟踪 MPC 在垂直飞行中平均速度仅 4.25 m/s（vs AC-MPC 10.05 m/s），因为它选择了保守的"零推力自由落体"策略。

（3）将可微分优化嵌入 RL 训练的计算代价：每个前向传播需求解一个完整 QP，每个反向传播需通过 KKT 条件计算隐式梯度。N=2 时训练需 11.5h（vs AC-MLP 21min），N=10 时需 39.6h。这种计算开销是因为每次 PPO 更新都需要对 mini-batch 中的每条轨迹运行可微 MPC 的前向+反向——而 MPC 求解本身是迭代的（iLQR 通常需 5-15 次迭代收敛）。

5.3 Key Challenges

  • 挑战 1 — 如何让梯度稳定地反向传播通过 MPC 求解器？diffMPC 的前向是 iLQR 迭代（5-15 次 LQR 反向+前向 rollout），反向需要在收敛点处求解线性系统 $\frac{\partial^2 J}{\partial u^2} \cdot \frac{\partial u^*}{\partial \theta} = -\frac{\partial^2 J}{\partial u \partial \theta}$。这个线性系统的条件数取决于 QP 的 Hessian——如果 Q 接近奇异（训练初期），条件数极大，梯度噪声会淹没有用信号。Cost Map 的 Sigmoid 设下限 0.1 正是为了缓解这一问题——确保 Q 始终可逆。
  • 挑战 2 — 稀疏奖励下 Critic 如何学到有意义的长期价值？在两次门通过之间可能有数百个时间步没有正奖励（只有稠密的进度奖励）。Critic 需要从这些弱信号中推断出"哪些状态区域更有价值"。GAE（Generalized Advantage Estimation, λ=0.95）通过在不同时间尺度上平滑 TD 误差来降低方差——这是关键设计选择。如果 λ 太小（偏重大偏差），优势估计噪声过大；如果 λ 太大（偏重大方差），训练不稳定。
  • 挑战 3 — Cost Map 输出空间的维度与约束管理：N=2 时 Cost Map 输出 68 维——这些参数必须同时满足：(a) Q 的每个对角元 ≥ 0.1（正定性），(b) 不同时间步的 Q/p 应体现时序一致性（不会 t=0 时鼓励向东、t=1 时鼓励向西），(c) 对状态分布的微小变化不应产生剧烈响应（平滑性）。Sigmoid 解决了 (a)，但 (b) 和 (c) 完全靠 RL 训练自然涌现——在某些失败案例中，Cost Map 可能学到不稳定的、振荡的 Q/p 序列。

6. Method & Key Formulas

6.1 Core Algorithm

AC-MPC 的核心算法（Algorithm 1）是一个修改的 PPO 训练循环，关键区别在于 Actor 的前向和反向传播。每次 PPO 迭代：(1) 用当前策略（Cost Map + diffMPC + 探索噪声）收集轨迹 buffer；(2) 用 Critic 计算 GAE 优势估计；(3) PPO-clip 目标更新 Cost Map——梯度流经 diffMPC 的 KKT 隐式微分；(4) MSE 回归更新 Critic。部署时：丢弃 Critic 和探索噪声，Cost Map 输出 Q/p → diffMPC 确定性求解 → u*_0 直接执行。训练的复杂度全部封装在 diffMPC 层的 forward（求解 QP）和 backward（KKT 隐式微分）实现中——PPO 算法其余部分无需修改。

6.2 Key Formulas

LaTeX rendered by MathJax. 显示公式用 $$...$$，行内公式用 $...$。⚠ 符号解释请用中文。

Group A: 二次规划形式的 MPC——从一般最优控制到可求解的 QP

Eq.(1) — 一般最优控制问题 $$ \pi(x) = \arg\min_{\mathbf{u}} \; J(x) \quad \text{s.t.} \quad x_0 = x,\; x_{k+1} = f(x_k, u_k),\; u_k \in \mathcal{U} $$
$J(x)$ = 从状态 $x$ 出发的总代价函数（待最小化） |  $f(x_k, u_k)$ = 四旋翼 13 维非线性离散动力学模型  |  $\mathcal{U}$ = 控制输入的可行约束集（单电机推力 $[0, T_{\max}]$，映射为集体推力和体角速率后约束） |  核心意义：这是 MPC 的数学定义——在动力学约束和输入约束下，找到最小化代价的控制序列。传统 MPC 中 $J$ 由人工设计；AC-MPC 中 $J$ 的参数 $(Q, p)$ 由神经网络从观测中动态预测。
Eq.(2) — 二次 MPC 代价函数（本文实际使用的 QP 形式） $$ J_Q(x) = \sum_{k=0}^{N} \begin{bmatrix} x_k \\ u_k \end{bmatrix}^T Q_k \begin{bmatrix} x_k \\ u_k \end{bmatrix} + p_k^T \begin{bmatrix} x_k \\ u_k \end{bmatrix} $$ $$ Q_k = \text{diag}(Q_{k,x_1}, \ldots, Q_{k,u_1}, \ldots), \quad Q_{k,i}, p_{k,i} \in [0.1,\; 100000] $$
$Q_k$ = 第 $k$ 步的二次项权重矩阵——被约束为对角阵（将参数从 $O((n+m)^2)$ 降到 $O(n+m)$），对角元由 Cost Map 经 Sigmoid 输出，下限 0.1 确保半正定性（QP 有唯一解） |  $p_k$ = 第 $k$ 步的线性项系数向量——通过将参考跟踪代价 $(x - x_{\text{ref}})^T Q (x - x_{\text{ref}})$ 展开，$p_k$ 等价于 $-2 Q x_{\text{ref}}$ 的线性项。这意味着 Cost Map 通过调整 $p_k$ 实际上在"隐式选择"一个等价的目标状态 $x_{\text{ref}}$。 |  $N$ = MPC 预测时域长度（主干实验 N=2，约 40ms 预测窗口；消融实验测试 N=5, 10） |  为什么采用二次形式？一般非线性 MPC 的求解太慢——将代价函数近似为 QP 后，可利用 iLQR 的高效性（每步仅需解 LQR 的 Riccati 方程 + 前向 rollout + 线搜索——毫秒级收敛）。

Group B: PPO 策略梯度——通过可微分 MPC 的反向传播链

Eq.(3) — PPO-Clipped 策略梯度目标函数 $$ J^{\text{PPO}}(\theta) = \frac{1}{N} \sum_{i=1}^{N} \sum_{k=1}^{T} \min\left( r_{k}^{i}(\theta) \hat{A}_{k}^{i},\; \text{clip}(r_{k}^{i}(\theta), 1-\varepsilon, 1+\varepsilon) \hat{A}_{k}^{i} \right) $$ $$ r_{k}^{i}(\theta) = \frac{\pi_{\theta}(a_{k}^{i} \mid s_{k}^{i})}{\pi_{\theta_{\text{old}}}(a_{k}^{i} \mid s_{k}^{i})},\quad \hat{A}_{k}^{i} = \text{GAE}(r_{k}^{i}, V_{\phi}(s_{k}^{i}), V_{\phi}(s_{k+1}^{i}), \lambda=0.95, \gamma=0.99) $$
$r_k^i(\theta)$ = 新策略 $\pi_\theta$ 与旧策略 $\pi_{\theta_{\text{old}}}$ 在状态 $s_k^i$ 下选择动作 $a_k^i$ 的概率比——用于重要性采样修正  |  $\hat{A}_k^i$ = 优势函数估计——使用 GAE (Generalized Advantage Estimation) 在偏差-方差之间平滑，参数 $\lambda=0.95$（接近 1：偏向低方差），$\gamma=0.99$（接近 1：重视长期回报） |  $\varepsilon = 0.2$ = PPO 的剪切范围——限制单次更新的策略变化幅度，防止策略崩溃  |  梯度链路（核心）：$\nabla_\theta J^{\text{PPO}} = \frac{\partial J}{\partial \pi} \cdot \frac{\partial \pi}{\partial u^*} \cdot \frac{\partial u^*}{\partial (Q,p)} \cdot \frac{\partial (Q,p)}{\partial \theta}$——其中 $\frac{\partial u^*}{\partial (Q,p)}$ 是可微 MPC 的核心贡献：通过 QP 的 KKT 条件隐式微分，在 iLQR 收敛点解析计算"优化问题解相对于代价函数参数的梯度"，无需展开整个迭代过程。
Eq.(4) — AC-MPC 的随机策略（训练 vs 部署） $$ u_k \sim \begin{cases} \mathcal{N}\big(\text{diffMPC}(x_k, Q_\theta(s_k), p_\theta(s_k)),\; \sigma^2 I\big) & \text{训练时（探索）} \\[6pt] \text{diffMPC}(x_k, Q_\theta(s_k), p_\theta(s_k)) & \text{部署时（确定性）} \end{cases} $$
训练时：在 MPC 最优解周围加高斯噪声 $\mathcal{N}(0, \sigma^2)$——噪声加在 MPC 的输出端（动作空间）而非 Cost Map 的输出端（代价函数参数空间）。这是一个精细的设计选择：在动作空间加噪声能更直接地探索"不同动作的效果"，且噪声的大小由 PPO 自适应调节 $\sigma$。 |  部署时：$\sigma = 0$，策略变为确定性的——MPC 的在线重规划能力确保即使无随机性，策略也能根据当前状态灵活调整动作。这与纯 RL（部署时 MLP 输出是固定映射，无重规划）形成本质区别。

Group C: 敏捷飞行任务奖励——将"快速安全穿门"编码为标量信号

Eq.(5) — 门进度奖励（每时间步的稠密信号） $$ r(k) = \|g_k - p_{k-1}\| - \|g_k - p_k\| - b \cdot \|\omega_k\| $$ $$ \text{过门奖励：} r(k) \mathrel{+}= 10.0 \qquad \text{碰撞惩罚：} r(k) = -10.0 \qquad \text{完赛奖励：} r(k) \mathrel{+}= 10.0 $$
$g_k$ = 当前目标门中心的三维位置  |  $p_k, p_{k-1}$ = 飞行器在当前时刻和前一时刻的位置  |  $\|g_k - p_{k-1}\| - \|g_k - p_k\|$ = 从上一时刻到当前时刻向目标门靠近的距离——正值表示接近门（正奖励），负值表示远离门（负奖励）。这是经典的"进度奖励"形式——等价于在位置空间中对门方向的投影分量。本质上是稠密的、每步都有信号的奖励——这对 MPC 的短时域优化至关重要（N=2 步内就能感受到奖励变化）。 |  $b\|\omega_k\|$ = 体速率惩罚（$b=0.01$）——抑制过大的角速率变化，鼓励平滑飞行。权重 0.01 很小——不会阻止快速机动所需的急剧角速率变化（如垂直俯冲中的快速翻转），仅起正则化作用。 |  稀疏奖励：过门 +10.0（阶段性正反馈）、碰撞 -10.0（强负反馈立即终止）、完赛 +10.0（终端正反馈）。这些稀疏信号提供了"里程碑"级别的学习引导，弥补稠密进度奖励无法区分"接近门但方向不对"和"接近门且即将通过"的模糊性。

Sigmoid 输出范围[0.1, 100000]Q/p 参数的允许取值区间下限 0.1：确保 Q > 0（严格正定，QP 有唯一解）——这是 MPC 求解的必要条件。上限 100000：防止参数在训练中无限增长（作者观察到不加限制时权重会发散）。100000 足够大以提供充分的表达范围，同时避免数值溢出
Critic 网络结构2 隐藏层 × 512 宽 + ReLU + 标量输出从 36 维观测映射到状态价值 V(s)与 Cost Map 对称设计，简化工程实现。标量输出对应单个状态的价值——GAE 利用时序差分构造优势函数，不需要价值分布的完整信息
PPO 剪切范围 ε0.2策略更新的最大比例变化——$r_t(\theta)$ 被截断在 $[1-\varepsilon, 1+\varepsilon]$标准 PPO 默认值。由于 diffMPC 的梯度比普通 MLP 更"尖锐"（经过优化求解器的非线性变换），ε=0.2 有助于防止单次更新导致的策略崩塌
GAE 参数 λ0.95优势估计中偏差-方差的权衡参数0.95 接近 1 = 偏向低方差（利用更多步的 TD 误差信息加权平均）。因为 diffMPC 的梯度本身已经引入额外方差（穿过优化求解器），需要用平滑的优势估计来补偿
角速率惩罚系数 b0.01控制输入平滑性的正则化权重0.01 很小——仅起平滑作用，不主导总奖励。如果 b 太大，策略将回避快速机动所需的急剧角速率变化（垂直俯冲中的快速翻转会被抑制），牺牲任务性能换取平滑性
门观测数量 N_gates2观测中包含的未来门数量（当前门 + 下一个门）2 个门提供足够的"前瞻"信息——Cost Map 可以根据下一个门的位置提前调整代价函数（如提前开始转向）。为什么不是 3 或更多？增加门数量会增加观测维度（每个门 12 维）但不一定改善性能——因为 MPC 的 N=2 短时域决定了它只能"看到"很近的未来，更远的门信息主要对 Critic 的长期价值估计有用

6.4 AC-MPC vs AC-MLP vs 传统 MPC 的本质差异

🧠 AC-MPC（本文）

  • Actor 最后一层 = 可微分 MPC 求解器
  • 学习的是代价函数参数 Q/p（"学目标"）
  • MPC 提供在线重规划 + 动力学约束保证
  • Cost Map 实现观测驱动的模式切换
  • 训练慢（11.5h @ N=2），但有物理先验
  • 风扰成功率 83.33%，分布外成功率 90.37%
  • 推理 13.5~70ms（取决于 N）

🔲 AC-MLP（纯 RL baseline）

  • Actor = 纯 MLP 前馈网络（无结构）
  • 直接学习观测 → 动作的映射（"学动作"）
  • 无在线重规划——泛化靠记忆
  • 隐式学习模式切换（无显式机制）
  • 训练极快（21min），但无物理先验
  • 风扰成功率仅 6.5%（几乎完全失败）
  • 推理 0.5ms（极快）

📐 传统跟踪 MPC

  • 代价函数手工设计、固定不变
  • 需要预计算参考轨迹 + 手工代价
  • 在线重规划（MPC 天然能力）
  • 无模式切换——代价函数不随观测变化
  • 无需训练（0h），但依赖手工工程
  • 垂直飞行保守（陷入零推力局部最优），平均速度仅 4.25 m/s
  • 每个新赛道需重新设计代价 + 参考轨迹

7. Fundamental Principles — 核心技术原理详解

理解 AC-MPC 需要掌握三个关键基础原理：（1）可微分 MPC——如何将优化问题求解器变成可训练的网络层；（2）双层时间尺度——Critic 的长期价值评估如何弥补 MPC 的短视域；（3）代价函数学习 vs 策略学习——为什么学习"目标"比学习"动作"在分布外更具泛化性。

7.1 可微分 MPC 原理 — 如何将优化问题求解器嵌入神经网络计算图

原理 1 — QP 的 KKT 条件隐式微分 $$ u^* = \arg\min_{u} \; J(u; \theta) \quad \Rightarrow \quad \frac{\partial u^*}{\partial \theta} = -\left(\frac{\partial^2 J}{\partial u^2}\right)^{-1} \frac{\partial^2 J}{\partial u \partial \theta} $$
$u^*$ = MPC 求解器输出的最优控制序列（N 步 × 4 维控制 = 4N 维） |  $\theta$ = Cost Map 网络的参数（通过链式法则，Cost Map 输出 Q/p → Q/p 决定 MPC 代价函数 → 代价函数决定最优解） |  $J(u; \theta)$ = 由 $\theta$ 参数化的二次代价函数（通过 Q 和 p 间接参数化） |  隐函数定理的核心：$u^*$ 是 argmin 的解——在最优解处，一阶必要条件 $\frac{\partial J}{\partial u} = 0$ 是隐函数方程。对这个方程两边关于 $\theta$ 求全微分，即可得到上式。这不需要"展开"优化迭代过程——只需要在收敛点处解一个线性系统。矩阵 $\frac{\partial^2 J}{\partial u^2}$ 就是 QP 的 Hessian——对于凸 QP 始终可逆。 |  数值实现：实际计算中，不直接求逆矩阵，而是通过解线性方程组来计算梯度——这等价于在收敛点做一次额外的"LQR 反向递推"，计算代价与一次 iLQR 迭代相当。

可微分 MPC 的三个实际操作：前向传播 = 给定 (Q, p) → iLQR 迭代求解 QP → 返回最优控制序列 $u^*_{0:N-1}$。反向传播 = (1) 接收上游梯度 $\frac{\partial L}{\partial u^*}$ → (2) 在 iLQR 收敛点通过 KKT 隐式微分计算 $\frac{\partial u^*}{\partial (Q,p)}$ → (3) 链式法则乘以上游梯度 → (4) 返回 $\frac{\partial L}{\partial Q}, \frac{\partial L}{\partial p}$ 给 Cost Map MLP。这个流程对 PPO 训练代码完全透明——从 PPO 视角，diffMPC 就是一个特殊的"激活函数"，有标准的 forward() 和 backward() 接口。

前向传播（推理时：Cost Map + diffMPC）

步骤 1：观测 $s_k$（36 维归一化）→ Cost Map MLP 前向 → 输出 $Q(s_k), p(s_k)$（68 维，N=2）

步骤 2：将 $Q, p$ 填入 MPC 的 QP 形式（式 2）→ 以当前状态 $x_k$ 为初始条件 → iLQR 求解（5-15 次迭代收敛）→ 最优序列 $u^*_0, \ldots, u^*_{N-1}$

步骤 3：取第一项 $u^*_0$ 作为输出 → 训练时加 $\mathcal{N}(0,\sigma^2)$ 噪声，部署时直接输出

总耗时：Cost Map（0.5ms）+ diffMPC（13ms）= 13.5ms @ N=2

反向传播（训练时：PPO 损失 → Cost Map 梯度）

步骤 1：PPO 损失对动作的梯度 $\nabla_{u^*} L^{\text{PPO}}$（标准 RL 反向传播）

步骤 2：diffMPC 的 backward() ——在 iLQR 收敛点处解 KKT 线性系统 → $\nabla_{Q,p} L^{\text{PPO}} = \nabla_{u^*} L^{\text{PPO}} \cdot \frac{\partial u^*}{\partial (Q,p)}$

步骤 3：$\nabla_{Q,p} L^{\text{PPO}}$ 通过 Cost Map MLP 的标准反向传播 → $\nabla_\theta L^{\text{PPO}}$

训练总耗时：AC-MPC (N=2) 11.5h vs AC-MLP 21min——瓶颈在步骤 2

7.2 双层时间尺度原理 — Critic 长期价值评估如何引导 Cost Map 的短期优化

原理 2 — 优势函数将 Critic 的长期评估转化为 Actor 的训练信号 $$ A(s_k, a_k) = r(s_k, a_k) + \gamma V_\phi(s_{k+1}) - V_\phi(s_k) $$ $$ \text{MPC（短时域 N 步）}：\min_{u_{0:N-1}} \sum_{k=0}^{N} \ell(x_k, u_k; Q, p) \quad \text{Critic（长时域 $\gamma$-折扣）}：V(s) \approx \mathbb{E}\left[\sum_{t=0}^{\infty} \gamma^t r_t \mid s_0 = s\right] $$
$A(s_k, a_k)$ = 在状态 $s_k$ 下采取动作 $a_k$ 相对于 Critic 预期水平的"优势"——正值表示"这个动作比平均好"  |  $V_\phi(s_k)$ = Critic 对状态 $s_k$ 的长期价值估计——从当前到任务结束的总期望折扣回报  |  $r + \gamma V(s_{k+1})$ = TD target——即时奖励 + 后续所有步的折扣期望回报  |  MPC vs Critic 的时间尺度分工：MPC 在 $N=2$ 步（约 40ms）的窗口内优化——处理的是"姿态稳定性、推力分配、短期轨迹"级别的决策。Critic 在整个 episode 的尺度上评估——处理的是"当前离完赛还有多远、下一个门在哪个方向"级别的知识。两者通过优势函数 $A(s,a)$ 连接：Critic 说"这个方向长期来看价值更高"→ $A$ 增大 → PPO 更新 Cost Map 产生指向该方向的 $Q/p$ → MPC 在短时域内执行朝该方向的最优动作。

模式切换的具体机制（以垂直穿门为例）：(1) 在初始阶段，飞行器在门的上方——Critic 学到的 $V(s)$ 在"下方"（门的位置）有更高的值。(2) Advantage $A(s,a)$ 对"向下的动作"取正值——引导 Cost Map 学习输出使 MPC 朝下方优化的 $Q/p$。(3) 飞行器通过第一个门后——Critic 的 $V(s)$ 热力图中的高值区域立即跳变到下一个门（上方）。(4) Advantage 现在对"向上的动作"取正值——Cost Map 自动调整 $Q/p$ 使 MPC 优化方向反转。(5) 这个"自动反转"不是硬编码的——它是 Critic 通过 TD 学习自然涌现的门通过感知。

7.3 代价函数学习 vs 策略学习 — 为什么学习"目标"比学习"动作"更具泛化性

原理 3 — 代价函数学习的归纳偏置优势 $$ \underbrace{\pi_{\text{AC-MPC}} = \text{MPC} \circ \text{CostMap}}_{\text{结构化：代价函数 → 优化求解 → 动作}} \quad \longleftrightarrow \quad \underbrace{\pi_{\text{AC-MLP}} = \text{MLP}}_{\text{非结构化：观测 → 动作}} $$
核心论点：AC-MPC 学习的不是"什么动作"而是"什么目标"。Cost Map 输出的是 MPC 应该优化的方向（二次代价函数的最小值点隐式定义了一个目标状态），MPC 负责根据动力学模型计算实现该目标的具体动作。 |  为什么这种"目标-动作解耦"带来泛化优势？"朝门飞"这个目标在不同的动力学条件下（仿真 vs 真实、无风 vs 有风）含义相同——Cost Map 输出的代价函数（指向门）在语义层面不变。但"朝门飞应该用什么推力 + 什么姿态"这个动作取决于当前的动力学条件——MPC 利用建模的动力学 $f(x,u)$ 重新计算了适配当前条件的动作。相比之下，AC-MLP 将"目标"和"动作"混合在一个黑箱映射中——当动力学变化时（如从仿真到真实），相同的观测可能需要不同的动作，但 MLP 没有模型来"重新计算"——它只能依赖泛化。
💡 三个原理的系统级协同
可微分 MPC 提供了"将优化作为网络层"的梯度基础设施 → 双层时间尺度通过 Critic 的价值函数解决了 MPC 短视域无法判断长期好坏的困难 → 代价函数学习通过"学目标而非学动作"的归纳偏置提供了 sim-to-sim 和 sim-to-real 泛化能力。三者环环相扣，使得 AC-MPC 成为一个"既有模型驱动控制的可靠性（约束满足 + 在线重规划），又有数据驱动学习的灵活性（自动发现代价函数 + 无需专家）"的统一框架。这也是为什么 AC-MPC 在风扰中成功率为 83% 而纯 RL 仅 6.5%——不是因为 AC-MPC"更聪明"，而是因为它的架构本身就包含了应对扰动的机制（MPC 在线重规划）。

9. Potential Flaw & Extensions

9.1 Limitations

  • 假设前提：(1) 需要解析的系统动力学模型 $f(x_k, u_k)$——如果模型不准确或完全未知（如软体机器人、变形飞行器），diffMPC 的预测和梯度都将不可靠。(2) diffMPC 仅支持输入约束 $u \in \mathcal{U}$（推力上下界），不支持状态约束 $x \in \mathcal{X}$（如"禁止进入某区域"）——这意味着避障等安全约束只能通过奖励函数间接编码（间接且不可靠），而非作为硬约束。(3) 训练需要仿真环境——PPO 的试错式学习在真实系统上直接进行既不安全也不可行（碰撞代价 -10.0 在仿真中只是一条日志，在真实世界是摔机）。
  • 适用场景：目前仅在四旋翼敏捷穿门飞行这一特定场景验证——虽然框架声称通用，但其在其他机器人形态（腿足机器人、机械臂、自动驾驶）和其他任务（操作、导航、多智能体协调）上的有效性尚未展示。感知感知飞行实验展示了多目标能力，但两个目标的 reward shaping 仍需人工设计。
  • 可扩展性：训练时间随 MPC 时域 N 超线性增长（N=2: 11.5h, N=5: 22.1h, N=10: 39.6h）。对于需要更长预测视野的任务（如迷宫导航、动态避障），N 需要更大——此时训练和推理都可能超出实时预算。实验发现 N=2 对穿门任务足够——但这依赖于该任务"短视+高频重规划"的特性能补偿短时域——对于规划密集型任务，短时域可能是致命的。

9.2 Extension Directions

  • 维度扩展：(1) 从四旋翼 → 其他机器人平台——任何可以写出动力学模型和二次代价函数的系统都可接入 AC-MPC。关键是动力学模型的可微分性——如果模型本身也是可学习的（如神经网络残差动力学），整个系统可以同时学习动力学和代价函数。(2) 从单任务 → 多任务统一策略——训练一个 Cost Map 在未见过的门布局上零样本泛化（真正的"学到穿门"而非"记住赛道"）。
  • 条件放松：(1) 利用新兴可微分 MPC 库（Theseus, PyPose）支持状态约束——将避障等安全约束作为 MPC 的硬约束而非奖励项。(2) 视觉观测替代状态观测——将 Cost Map 的输入从已知位姿+门位置替换为图像（CNN 编码），实现"从像素到 MPC 代价函数"的端到端管道。(3) 离线 RL / 从数据中学习——利用预先收集的数据集训练 Cost Map，消除对在线交互仿真的依赖。
  • 约束加入：(1) 将 Lyapunov 稳定性验证或可达性分析作为 diffMPC 输出的后验证层——确保即使 Cost Map 输出异常的 Q/p 导致 MPC 得出危险动作，也能被拦截。(2) 多智能体扩展——每个机器人独立 Cost Map + diffMPC，但通过共享 Critic 或交互信息实现协调。

10. Motivation — First-Principles Derivation

10.1 Idea Derivation Chain

  1. 问题的本质是什么？
    机器人控制领域存在两条主流技术路线——模型驱动控制（MPC 等）和强化学习——但各自有根本性缺陷，且两者的缺陷恰好互补。MPC 可靠但僵化——在线重规划保证鲁棒性，但代价函数必须手工设计，无法适应复杂任务的动态模式切换。RL 灵活但脆弱——可以从奖励中自主学习任何行为，但学到的策略是黑箱前馈网络，缺乏在线重规划和动力学约束保证。这不是"哪个更好"的问题——而是"能否将两者的优势融合"的问题。
  2. 前人如何处理？为什么不够？
    三条路线：(i) 零阶优化调 MPC 参数（贝叶斯优化、策略搜索）——不利用梯度，样本效率极低，只能学固定参数；(ii) 可微 MPC + 模仿学习——有梯度但依赖专家，无法探索超越专家；(iii) 纯 RL——不需要专家也不需要模型，但面对分布外场景脆性显著。每条路线都只解决了一半问题。
  3. 如果把可微 MPC 直接塞进 Actor 网络呢？
    关键洞察：Actor 的输出不一定是动作——它可以是传递给 MPC 的"指令"（代价函数参数）。Cost Map 学的是"这个状态下 MPC 应该优化什么"（学目标），MPC 负责"给定目标如何求最优动作"（执行）。这种解耦意味着：Cost Map 可以在仿真中学习（学"语义级"的代价函数），部署时 MPC 用真实动力学执行（利用模型的物理保真度）——自然地实现了 sim-to-real 迁移。
  4. 如果 MPC 的短视域是个问题呢？
    MPC 的 N=2 步时域只能"看到"约 40ms 的未来——无法判断当前动作对"2 秒后通过下一个门"的影响。这就是 Critic 的作用——Critic 通过 TD 学习估计长期价值 V(s)，通过优势函数 A(s,a) 向 Cost Map 传递"从长远看这个方向好不好"的信号。MPC 负责短期执行，Critic 负责长期引导——两层时间尺度互补。
  5. 最简单的方案是什么？
    在标准 PPO 的 Actor 中，将最后一层 MLP 替换为可微 MPC 求解器。Cost Map = 2 层 MLP（输出 Q/p），Critic = 另一个 2 层 MLP（输出 V(s)）。训练流程与 PPO 完全一致——唯一的区别是 Actor 的梯度需要穿过 diffMPC 的 backward()。这个"简单方案"之所以此前从未实现，需要三个条件的成熟：(a) 可微 MPC 的软件基础设施（Amos et al. 2018 后逐渐成熟）；(b) PPO 在大规模机器人仿真中的训练经验（近年的积累）；(c) 对 RL 和 MPC 各自优劣的深刻理解——两者不是"选一个"，而是"找出各自最适合的角色"。

10.2 One-Sentence Motivation

💡 一句话动机
纯 RL 策略在训练分布内可以超越专家（21min 训练即可），但遇到训练未见过的风扰时几乎完全失效（成功率仅 6.5%）——因为它的 Actor 是一个黑箱 MLP，没有利用飞行动力学模型来做在线重规划——那为什么不把 MPC 直接塞进 Actor 网络作为最后一层，让 MPC 提供"物理推理引擎"，而让 RL 来学习"这个推理引擎应该以什么为目标"（代价函数），从而同时获得 RL 的灵活性和 MPC 的鲁棒性？

10.3 How You'd Arrive at This Idea

  • Step 1 — 观察现象：注意 AC-MLP（纯 RL）和跟踪 MPC 在敏捷飞行任务中的失败模式是严格互补的。AC-MLP 在训练分布内表现最强（7.74 m/s 水平，10.56 m/s 垂直），但风扰成功率仅 6.5%。跟踪 MPC 风扰下也不成功（0%），但在无风扰+标准初始条件下非常稳定。AC-MLP 垂直飞行速度 10.56 m/s，MPC 仅 4.25 m/s（保守策略）。这意味着：AC-MLP 擅长"找到最优行为"，MPC 擅长"安全执行"——两者的优势完全不重叠，结合它们有望获得 1+1>2 的效果。
  • Step 2 — 分解问题：将"控制策略"分解为两个正交子问题——(A) 目标选择：当前应该往哪个方向优化？这需要任务级知识（"下一个门在哪个方向"）和灵活性（"根据当前飞行阶段调整激进程度"）；(B) 动作执行：给定优化方向，如何用动力学模型计算具体动作？这需要物理知识（$f(x,u)$）和优化能力（求解约束 QP）。传统 MPC 把 (A) 和 (B) 混在一起——代价函数同时定义了"目标"和"如何实现"。
  • Step 3 — 匹配工具：(A) 用 RL 学习"目标选择"——因为 RL 天然擅长从奖励中自动发现复杂行为；(B) 用 MPC 执行"动作生成"——因为 MPC 天然擅长利用动力学模型做在线优化；(C) 用可微分优化连接两者——因为梯度是 RL 的训练信号穿过 MPC 的唯一高效通道。Cost Map 是 (A) 和 (B) 之间的接口——它以神经网络形式实现 (A)，以 Q/p 的形式输出给 (B) 的 MPC。
  • Step 4 — 组装方案：AC-MPC = Cost Map（2 层 512 MLP + Sigmoid）+ diffMPC（iLQR QP 求解器）+ Critic（2 层 512 MLP），用标准 PPO 训练，梯度经 diffMPC 的 KKT 隐式微分反向传播。部署时丢弃 Critic 和探索噪声——Cost Map → diffMPC 组成确定性策略。在风扰测试中：AC-MPC 83.33% vs AC-MLP 6.5% vs MPC 0%——验证了"架构级鲁棒性"优于"数据级鲁棒性"。

11. Key Takeaways

11.1 Lessons Learned

  • 方法论层面：这是一个教科书级的"归纳偏置注入"案例——不是让 RL 从零学习一切（弱偏置），也不是让 MPC 完全由人类设计（强偏置、零学习），而是在 RL 架构中精确地注入一个结构化的动力学模块（MPC），该模块在训练时可微（允许梯度流动）、在部署时物理可解释（满足约束、可重规划）。对 UAV 控制领域的核心启示：不要问"RL 还是 MPC"——要问"MPC 的哪部分应该用 RL 来学习，哪部分应该保留为物理模型"。具体到我们的研究方向：视觉伺服中，投影模型应该保留为可微分层（像 MPC 一样），而"从图像特征到期望运动"的映射可以用学习；编队控制中，相对运动学应保留为结构化层，而"从邻居状态到期望队形调整"的策略可以学习。
  • 技术技巧层面：(1) Q 矩阵对角化——用最小的参数化代价（丧失状态间耦合建模）换取最稳定的 QP 求解（对角正定矩阵的条件数远优于稠密矩阵）；(2) Sigmoid 输出限幅 [0.1, 1e5]——同时解决"QP 需正定"（下限 > 0）和"参数发散"（上限 < ∞）两个问题，是机器学习约束和控制理论约束之间的精巧桥接；(3) 在动作空间加噪声而非代价空间——在 MPC 输出动作上直接添加高斯噪声，保证探索的动作"接近最优"（不会因为 Cost Map 的随机波动产生完全离谱的动作）；(4) 在线观测归一化——每训练步更新均值和标准差，避免训练分布漂移导致的归一化过时。
  • 实验/写作层面：Fig. 2 的价值函数热力图是整篇论文最精彩的可视化——它把一个抽象的"双时间尺度协同"概念转化为直观的"高价值区域随門切换而跳变"的热力图，读者一目了然。Fig. 3 的三项消融测试（风扰、学习曲线、OOD 初始条件）构成"三角论证"——每一项分别揭示 AC-MPC 在一个维度上相对两个 baseline 的优势，且各项优势的机制互不相同（重规划 vs 价值引导 vs 动力学约束），形成了无可辩驳的论证链。写论文时应当学习这种"通过关键可视化传达核心机制"+"通过互补消融构成完整论证"的实验设计范式。

11.2 Directly Applicable to My Research

  • 对于视觉伺服（IBVS）方向：AC-MPC 的结构可以直接适配到基于图像的视觉伺服——将图像特征坐标（目标在图像中的位置、门角点的像素坐标）作为 Cost Map 的输入（替代当前的"飞行器状态+门三维位置"）。Cost Map 学习从图像特征到 MPC 代价函数参数的映射。MPC 的动力学模型除了四旋翼动力学外，还包含可微的相机投影模型（针孔投影）——使 MPC 能够预测"当前动作将如何影响下一帧的图像特征坐标"。这带来了传统 IBVS 完全不具备的两大能力：(1) 代价函数是学出来的——不需要手工设计特征误差的加权矩阵；(2) MPC 预测未来的特征运动——可以在图像空间中做 look-ahead 优化，提前补偿已知的视觉约束（如视野边界）。
  • 对于敏捷飞行控制方向：AC-MPC 提供了一个在"鲁棒性"和"性能"之间取得更好平衡的控制框架。如果你已经有一个可用的四旋翼动力学模型，用 AC-MPC 替代纯 RL 可以以训练时间增加（11.5h vs 21min）为代价，换取分布外鲁棒性的量级提升（83% vs 6.5% 风扰成功率）。对于安全至上的应用（近地高速飞行、穿越狭窄缝隙），这个 trade-off 完全值得——因为一次失败的真实飞行代价远大于几十小时的 GPU 训练。
  • 对于控制架构设计方向：本文展示了"可微分优化层作为神经网络组件"的通用设计模式——可微 MPC 只是可微优化层的一个实例。其他可类比扩展包括：可微路径规划器（iPlanner）、可微状态估计器（可微 EKF）、可微 SLAM（Differentiable SLAM）。任何一种传统机器人模块，只要核心是求解优化问题，都可以被做成可微层嵌入学习管道——这是一个可以系统性地将"经典机器人学"与"现代深度学习"融合的范式，值得在我们的 UAV 控制架构研究中持续探索。
