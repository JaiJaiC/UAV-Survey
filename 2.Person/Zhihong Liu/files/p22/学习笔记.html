一、问题背景：自主着陆为什么难？

1.1 着陆阶段的特殊挑战

自主着陆（Autonomous Landing）是无人机飞行全过程中最关键也最危险的阶段。据统计数据，无人机事故中有相当大比例发生在着陆阶段。着陆过程面临以下独特挑战：

  • 地面效应（Ground Effect）：接近地面时，旋翼下洗气流与地面相互作用，产生复杂的气动干扰，飞行器动力学特征发生显著变化。
  • GPS 不可靠：近地区域 GPS 信号易受多路径效应、遮挡和干扰影响，定位精度急剧下降——而在着陆阶段恰恰需要厘米级精度。
  • 实时性要求极高：着陆窗口短，控制延迟可能导致过冲、侧翻甚至坠毁。控制频率通常需要 10Hz 以上。
  • 环境不确定性：光照变化（逆光/背光/黄昏）、着陆平台纹理变化、风扰等因素对视觉系统构成严峻考验。
  • 单相机信息不足：单目相机缺乏深度信息（scale ambiguity），从 2D 图像恢复 3D 运动指令是一个天然的 ill-posed 问题。

核心矛盾
传统视觉着陆方法遵循"感知 → 位姿估计 → 控制"的串行流水线。这一范式存在两个根本性问题：(1) 感知和控制被分离设计，缺乏联合优化，上游误差不可逆地传播至下游；(2) 每个子问题（目标检测、位姿估计、控制器设计）都需要人工先验——特征选择、阈值设定、参数标定——这些在开放环境中极易失效。

1.3 为什么选择深度强化学习？

深度强化学习（Deep Reinforcement Learning, DRL）提供了一种全新的范式——端到端学习（End-to-End Learning）：直接从原始感知输入映射到控制输出，消除中间环节，实现感知与控制的联合优化。具体到着陆任务：

  • 端到端：单目相机图像 → 速度指令（vx, vy, vz, yaw rate），没有检测、位姿估计等中间模块。
  • 无需标注数据：DRL 通过与环境的交互（试错）自主学习策略，无需人工标注的训练数据集。
  • 无需 GPS：仅依赖机载单目下视相机和惯性测量单元（IMU），不依赖任何外部定位设施。
  • 鲁棒性潜力：通过与仿真环境的大量交互，策略可以习得对光照变化和部分遮挡的不变性。
  • 单阶段着陆：不需要传统方法中"悬停检测→跟踪下降"的两阶段流程，实现一气呵成的着陆。

二、方法论：改进 DQN 架构

2.1 从 DQN 到 Double DQN + Dueling

本文的核心方法基于 DQN（Deep Q-Network），并引入了两项关键改进：

DQN 基础

经典的 DQN（Mnih et al., 2015）使用深度神经网络近似动作价值函数 Q(s, a)，通过经验回放（Experience Replay）和目标网络（Target Network）稳定训练。贝尔曼最优方程为：

Q*(s, a) = E[r + γ · maxa' Q*(s', a')]

局限：max 操作符导致系统性过估计（overestimation）偏差——同一网络既用于选择动作又用于评估，噪声被放大。

Double DQN 改进

Double DQN（van Hasselt et al., 2016）通过解耦动作选择和动作评估来解决过估计问题：用在线网络选择最优动作，用目标网络评估其价值。

yt = r + γ · Q(s', argmaxa' Q(s', a'; θ); θ−)

效果：显著减轻 Q 值过估计，在 Atari 游戏中比标准 DQN 获得更高真实分数，尽管估计的 Q 值更"保守"。

Dueling Architecture

Dueling 架构（Wang et al., 2016）将 Q 函数分解为两个独立流：

Q(s, a) = V(s) + A(s, a) − meana' A(s, a')

其中 V(s) 为状态价值（当前位置"多好"），A(s, a) 为优势函数（某个动作比平均好多少）。

优势：在无需精确估计每个动作价值的情况下，也能学习哪些状态更有价值——特别适合某些状态下动作选择影响微小的场景。

为什么 Dueling 架构适合着陆任务？
在着陆任务中，存在大量"动作无关紧要"的状态。例如，当无人机正对 Marker 中心正上方且姿态稳定时，任何小幅度的水平移动都可能导致偏离——此时 V(s)（当前状态的质量）比 A(s, a)（不同动作的差异）更具信息量。Dueling 架构能够有效捕获这种状态价值主导的特性，加速学习并提高最终策略质量。

三、强化学习问题设计（State / Action / Reward）

3.1 状态空间（State）

状态是 DRL agent 感知环境的唯一信息源。本文采用端到端视觉感知：

  • 原始相机图像：无人机底部安装的单目相机实时采集的图像，缩放到 84×84 像素，转换为灰度图。
  • 无额外状态信息：不显式传入无人机位置、速度、姿态等状态量——所有有用信息都必须从图像中自动提取。
  • Aruco Marker：着陆平台上放置 Aruco 标记作为视觉信标，为 agent 提供清晰的对准参考。Aruco 是一种二进制方形基准标记，具有快速检测和高鲁棒性特点。
  • 帧堆叠：通常使用连续 4 帧作为输入（84×84×4），以提供运动信息（隐式速度感知）。

3.2 动作空间（Action）—— 连续动作离散化

无人机着陆是一个连续控制问题，但 DQN 仅支持离散动作。本文通过对连续动作空间进行离散化来解决这一矛盾：

控制量物理意义离散取值说明
vx机体 x 轴线速度{−v_max, 0, +v_max}左右移动
vy机体 y 轴线速度{−v_max, 0, +v_max}前后移动
vz机体 z 轴线速度{−v_max, 0, +v_max}下降/上升
ω偏航角速率 (yaw rate){−ω_max, 0, +ω_max}旋转对准

每个速度分量有 3 个离散档位，组合后总动作数为 34 = 81 个离散动作。实际上，部分无意义组合被剔除（如同时上升和下降不可能并存仅 vz 方向可区分），最终有效动作空间约为数十个。

离散化的局限性
离散化使得 DQN 可用于连续控制任务，但也引入了量化误差——无人机只能以零、正最大、负最大三档速度移动，难以实现精细的悬停和微调。这也为后续工作（使用 Actor-Critic 方法如 DDPG 实现真正连续控制）提供了动机。

3.3 奖励函数设计（Reward）

奖励函数是 DRL 最关键的工程设计，直接决定了策略的行为。本文基于距离 + 对准 + 稀疏奖励构建多目标奖励：

奖励函数核心组成

R = Rdist + Ralign + Rsuccess + Rcrash

Rdist距离奖励惩罚无人机与着陆平台中心的水平距离和垂直距离。距离越小，惩罚越轻（负奖励的绝对值越小）。
Ralign对准奖励根据 Aruco Marker 在图像中的位置与图像中心的偏移量给出奖励。Marker 越接近图像中心，奖励越高。
Rsuccess成功着陆当无人机到达着陆平台上方且高度小于阈值（如 0.05 m）且水平偏移和速度均低于阈值时，给予大额正奖励。
Rcrash坠毁惩罚无人机飞出边界、与地面剧烈碰撞或姿态倾角过大时，给予大额负奖励并终止 episode。
奖励设计的核心洞察
DRL 着陆中奖励设计的难点在于奖励稀疏性——成功着陆是一个罕见事件，如果只有稀疏的最终奖励，agent 几乎不可能通过随机探索发现正确行为。本文通过引入密集的中间奖励（水平距离、对准偏差）提供学习"梯子"，引导 agent 逐步接近目标。这种"密集引导 + 稀疏目标"的组合是 DRL 任务中经典的 reward shaping 策略。

四、实验设计与结果分析

4.1 仿真环境

实验在仿真环境中进行，使用四旋翼动力学模型和虚拟下视相机。关键仿真设定：

  • 降落平台：地面放置 Aruco Marker 的方形平台，Marker 为黑白二进制图案，提供高对比度视觉特征。
  • 初始状态：无人机从平台上方随机位置出发，高度约 5~10 m，水平偏移 ±5 m，随机偏航角。
  • 光照变化：仿真中引入随机光照强度、方向和色温变化，以提升策略对真实环境中光照变化的鲁棒性。
  • 部分遮挡：训练中随机添加噪声和遮挡模式，模拟真实场景中 Marker 可能被尘土、阴影或无人机自身结构部分遮挡的情况。
  • 控制频率：10 Hz（每 0.1 s 输出一次速度指令），底层姿态控制器以更高频率运行。
  • 终止条件：成功着陆（触地且姿态速度合规）或坠毁（倾角超限/飞出边界），单次 episode 最长持续时间有限制。

4.3 定性分析

实验观察到以下关键行为：

  • 学习过程：初期 agent 随机探索，频繁飞出边界。约 100 episodes 后开始学会"看到 Marker → 向图像中心移动"，500 episodes 后形成稳定着陆策略。
  • 着陆轨迹：训练后的策略表现出"先水平对准、再垂直下降"的智能行为——并非人为指定，而是 agent 自主习得的有效策略。
  • 光照鲁棒性：经过 domain randomization 训练的策略，在未见过的光照条件下（暗光、逆光）仍能保持较高成功率，验证了端到端 DRL 方法的鲁棒性潜力。
  • 部分遮挡：当 Marker 被部分遮挡时（遮挡面积 < 40%），策略仍能维持跟踪与下降；遮挡 > 60% 时性能显著下降——这是纯视觉方法的固有局限。
与传统方法的定性对比
传统"检测→跟踪→控制"方法需要检测到 Marker 的所有四个角点，一旦部分遮挡即失败。而 DRL 策略通过端到端学习，能够利用 Marker 的局部特征（甚至仅依赖于图像纹理的整体分布信息）进行决策，因此在部分遮挡下展现出优于传统方法的鲁棒性——这是端到端学习范式的一个核心优势。

    Phase 4 (600+ eps):    策略趋于稳定，reward 波动减小，着陆轨迹一致性好

五、学术意义：开启 DRL + UAV 研究方向

5.1 核心贡献总览

  1. 端到端范式的早期验证：在 2018 年 CCC 会议时间节点上，将 DRL 端到端学习应用于无人机自主着陆是一项前瞻性尝试。该工作证明了"图像→动作"直接映射在着陆任务上的可行性，无需 GPS、外部动捕或分离式感知-控制系统。
  2. 改进 DQN 在机器人控制中的有效应用：将 Double DQN + Dueling Architecture 组合引入无人机控制场景，验证了这些改进在真实机器人任务中的有效性——而不仅仅是在 Atari 游戏上。
  3. 连续动作离散化的实践经验：通过将四维连续速度指令离散化为有限组合，使得基于值函数的 DQN 方法可以处理连续控制问题，为后续研究提供了方法论参考。
  4. Domain Randomization 在视觉着陆中的应用：在仿真训练中引入光照和纹理随机化，使策略具备一定的 sim-to-real 迁移能力，这是当时相对新颖的训练策略。

5.3 论文的独特价值

方法论桥梁

本文巧妙地将当时主要应用于游戏 AI 的 DQN 及其变体（Double DQN、Dueling）迁移到无人机控制这一物理世界任务中。它不是简单的"换场景"，而是针对连续控制、视觉感知、实时性约束等机器人领域的特殊需求进行了系统性适配——这种跨领域迁移能力体现了作者对 DRL 本质的深入理解。

实践先导

2018 年，中国学术界将 DRL 应用于无人机控制的团队寥寥无几。本文是国防科大在该方向的早期探索，为团队后续一系列高水平成果（从 CCC 到 RA-L、TRO、ACM CSUR）积累了宝贵的方法论经验和仿真/实验基础设施。

引用影响力

20+ 次引用（以 CCC 会议论文标准而言表现良好），被后续关于视觉导航、DRL 无人机控制、自主着陆的论文广泛引用。在 Google Scholar 上，该论文的引用呈稳定增长趋势，反映了该方向持续升温的研究兴趣。

读后感与启示

这篇论文是理解刘志红整个研究脉络的绝佳起点。它展示了一个博士生（应为硕士阶段）如何从具体问题出发（无人机着陆），选择一个当时前沿但尚未成熟的技术路线（DRL），做出合理的工程判断（离散动作、奖励设计、改进 DQN），最终开辟出一个产出了数十篇论文的系统性研究方向。对于有志于从事 DRL 或机器人控制研究的学生而言，这篇论文的方法论——如何将通用 AI 算法适配到特定物理系统——具有标本级别的参考价值。
