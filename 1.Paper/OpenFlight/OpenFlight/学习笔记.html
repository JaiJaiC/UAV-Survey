传统控制的弊端

传统的固定翼无人机控制采用多层架构——"上层路径规划 → 中层速度/姿态追踪 → 下层执行器控制"。这种分层 PID/LQR 结构会引入 级联误差和长时间延迟，无法充分释放无人机的机动性极限。固定翼动态耦合的难点在于：高度非线性、强欠驱动、速度/爬升率深度耦合（必须保持向前飞行以提供升力），在极限机动中控制面动作激进，常规控制难以处理。

V1 — 速度级控制 (Velocity-Level Control)

动作 → 物理量（反归一化）

yaw_rate    = a[0] × 0.5        # ±0.5 rad/s
airspeed    = 20 + a[1] × 6     # 14~26 m/s
climb_rate  = a[2] × 10 × 3.28  # ±10 m/s → ft/s

观测（6D，极坐标表示）

obs[0] = Δψ / 2π         # 航向偏差
obs[1] = tanh(d_h × .01) # 水平距离
obs[2] = tanh(d_v × .02) # 垂直距离
obs[3] = ω_z / 4π        # 偏航角速率
obs[4] = tanh((v-20)×.05)# 空速偏差
obs[5] = tanh(v_z × .1)  # 下降速率
特征：RL 只负责高层导航决策，底层姿态稳定交给 PID。最简单、收敛最快。

V2 — 执行器级控制 (Actuator-Level Control) ⭐ 核心创新

动作 → 直接舵面

aileron_cmd   = a[0]              # [-1, +1]
elevator_cmd  = a[1]              # [-1, +1]
throttle_cmd  = 0.5 + a[2] × 0.5  # [0, 1]

观测（11D，增加姿态+角速度）

obs[0:3]  = 目标信息（同 V1）
obs[3:5]  = 速度信息
obs[5:8]  = 姿态角 (roll/pitch/yaw)
obs[8:11] = 角速度 (p/q/r)
突破性尝试：彻底摒弃 PID，消除两级控制延迟与累积误差，直接探寻物理边界下的极限机动。训练难度极高但最终性能碾压全场——比传统控制器减少 27.6% 飞行时间。

V3 — 姿态级控制

中间层：RL 输出目标姿态（roll/pitch），PID 负责内环稳定。是 V1→V2 的过渡方案。

roll_cmd  = a[0] × π       →  roll_hold()
pitch_cmd = a[1] × π/2     →  pitch_hold()
airspeed  →  airspeed_hold_w_throttle()

3. 论文精读

3.1 核心贡献

  1. 首次将 RL 的执行器级控制 (ALC) 应用于固定翼最短时间飞行
  2. 基于 6-DOF 高保真模型 + 风洞实验气动系数（非简化质点模型）
  3. 系统比较 4 种 SOTA RL 算法（A2C、TD3、SAC、PPO）在两种控制层级的表现
  4. PPO-ALC 比传统控制器减少 27.6% 飞行时间，且标准差极低（1.39s）

3.2 6-DOF 动力学模型

以往很多 RL 飞行控制研究使用简化质点模型，本文使用完整 6-DOF 刚体动力学：

# 平动方程（牛顿第二定律）
m(v̇ + ω × v) = F

# 转动方程（欧拉方程）
I·ω̇ + ω × I·ω = M

# 力分解
F = F_aero + F_gravity + F_thrust
M = M_aero + M_thrust

气动力基于 Skywalker X8 的风洞实验和 CFD 数据，JSBSim 负责数值积分。这意味着策略必须在高度非线性的气动耦合环境中工作。

3.5 ALC 学到了什么——极限机动策略

PPO (ALC) 展现出令人震撼的飞行策略：

  • 主动降速换转弯：转弯时主动降低空速来换取极小转弯半径快速掉头，随后猛加推力
  • 激进但稳定：极其激进的转弯动作（极小半径 + 高角速度），但没有任何速度振荡，轨迹平滑
  • 端到端协同：升降舵、副翼、油门三者协同配合，而非独立控制——这是 PID 级联结构做不到的
关键启示：ALC 比 VLC 飞行时间缩短 22.0%，比传统 Dubins 缩短 27.6%。这证明端到端 RL 直接控制舵面是完全可行且性能最佳的——只要加上合理的终止条件（如角速度超限 = 坠毁）。

4. 算法公式全解 —— 从 POMDP 到 PPO

论文的数学体系分为五个层次：问题建模 → 物理约束 → 接口归一化 → 奖励信号 → 策略优化。每层都对应到具体代码。

4.0 POMDP 形式化 （Section 3.1，未编号公式）

POMDP = ⟨ S, A, T, R, Ω, O, γ ⟩
七元组定义了 RL 智能体在不完全观测环境中的决策框架
符号含义OpenFlight 中的具体对应
S真实状态集合JSBSim 内部完整 6-DOF 状态（50+ 属性，含气动系数、发动机内部状态等）
A动作集合3 维连续向量 ∈ [−1, +1]³，输出范围由 tanh 激活函数保证
T状态转移 s' ~ T(s,a)JSBSim 的 Runge-Kutta 4 阶数值积分——非解析函数，真实体现物理规律
R奖励函数Eq.(13) 密集奖励 = k₁·Δd_h + k₂·Δd_v + k₃·Δψ
Ω观测集合6D (VLC) 或 11D (ALC) 归一化向量——极坐标表示
O观测函数 o ~ O(s)_get_obs() 方法——从 50+ 维物理状态中提取 6~11 维
γ折扣因子0.99（SB3 PPO 默认），未来奖励的衰减率
为什么是 POMDP 而不是 MDP？
固定翼无人机在真实飞行中无法直接感知完整状态（如气动力系数 α、β、精确的风场等）。RL 策略必须从不完整的传感器读数中做决策——这恰好是 POMDP 的典型场景。_get_obs() 只返回 6~11 维，而 JSBSim 内部有 50+ 个属性——这就是 "Partial Observation" 在代码层面的体现。

4.2 6-DOF 飞行动力学 —— Eq.(2)~(6)

这些公式全部封装在 JSBSim 的 C++ 核心中，Python 层不可见——但理解它们对理解"为什么 ALC 这么难学"至关重要。

Eq.(2): 平动方程（牛顿第二定律在旋转参考系）

Eq.(2)
m · ( v̇ + ω × v ) = F
m飞行器质量，Skywalker X8 ≈ 8 kg
v = [u, v, w]T机体坐标系线速度 (m/s)
ω = [p, q, r]T机体坐标系角速度 (rad/s)
F总外力 (N)，分解见 Eq.(4)
ω × v科里奥利项——旋转参考系中出现的"伪力"。
这就是为什么转向时飞机会侧滑——不是外力推的，是坐标系在转。

Eq.(3): 转动方程（欧拉方程）

Eq.(3)
I · ω̇ + ω × I · ω = M
I惯性矩阵（3×3 对称正定）。绕不同轴的转动惯量不同：绕滚转轴容易，绕俯仰轴困难——这是飞机设计的根本约束。
M总外力矩 (N·m)
ω × I·ω陀螺效应——旋转体抵抗改变转轴方向。解释了为什么偏航运动会耦合到滚转——这是固定翼"荷兰滚 (Dutch Roll)"模态的数学根源。

Eq.(4): 力与力矩的分解

Eq.(4)
F = Faero + Fgravity + Fthrust
M = Maero + Mthrust
力分量来源JSBSim 实现
Faero气流流过机翼/机身产生（升力 L、阻力 D、侧力 Y）x8.xml 中基于风洞实验的气动系数表
Fgravity重力 mg，在机体坐标系下投影（随姿态变化）JSBSim 自动根据姿态角计算
Fthrust螺旋桨/电机推力，沿机体 x 轴electric800W.xml 电机模型
Maero气动力 × 力臂（升力作用点不在重心）x8.xml 中与 α, β 相关的力矩系数

Eq.(5)~(6): 气动力坐标变换

Eq.(5)
[Faero]body = Rbw(α, β) · [−D, Y, −L]T
Eq.(6)
Rbw = ⎡ cosα·cosβ   −cosα·sinβ   −sinα ⎤
          ⎢ sinβ          cosβ           0     ⎥
          ⎣ sinα·cosβ   −sinα·sinβ    cosα ⎦
DDrag（阻力）——沿气流方向，阻碍飞行
YSide force（侧力）——垂直于对称面
LLift（升力）——垂直于气流方向，向上
α攻角（Angle of Attack）——机体 x 轴与气流方向的夹角
β侧滑角（Sideslip）——气流偏离对称面的角度
关键耦合：Rbw 中有 sinα 和 cosα——升力和阻力会同时投影到体轴系的 x 和 z 方向。这就是为什么改变攻角会同时影响前进和升降——固定翼最核心的气动耦合。ALC 策略必须学会处理这种耦合。

Eq.(7): VLC 观测向量（6 维）

Eq.(7)

4.5 奖励函数 —— Eq.(13)

Eq.(13)
rt = k₁ · Δdh + k₂ · Δdv + k₃ · Δψg
其中 k₁ = 0.25,   k₂ = 1.0,   k₃ = 20
每个 Δ 定义为 前一步 − 当前步（距离减小=正奖励，距离增加=负反馈）
项计算权重典型单步值贡献
Δdhdh(t−1) − dh(t)0.25~10 m/step~2.5
Δdvdv(t−1) − dv(t)1.0~2 m/step~2.0
Δψg|Δψ(t−1)| − |Δψ(t)|20~0.1 rad/step~2.0
总计~6.5
量级平衡原理：三项贡献都在 ~2 量级，谁也不"淹没"谁。论文明确指出：只需调整权重使三者的绝对值接近即可，避免量级差异过大导致小项被忽略。这不是理论推导，是工程经验。

reward_dist_horiz = (pre_horiz_dist - curr_horiz_dist) * 0.25  # k₁
reward_dist_vert  = (pre_vert_dist - curr_vert_dist)           # k₂=1.0
reward_heading    = (pre_heading_err - curr_heading_err) * 20.0 # k₃
reward = reward_dist_horiz + reward_dist_vert + reward_heading

4.6 PPO: 策略优化 —— 论文引用 [20]，未重新推导

论文使用 Schulman et al. (2017) 提出的 PPO 算法，没有重新推导公式——但理解 PPO 对解释论文实验结果至关重要。

核心：Clipped Surrogate Objective

PPO
LCLIP(θ) = 𝔼t[ min( rt(θ)·At,   clip(rt(θ), 1−ε, 1+ε)·At ) ]
其中 rt(θ) = πθ(at|st) / πθ_old(at|st)   （概率比 / importance sampling ratio）
ε = 0.2（SB3 默认）——策略更新的最大幅度限制
符号含义OpenFlight 中的值
πθ(a|s)新策略：当前参数下选动作 a 的概率高斯分布 N(μθ(s), σθ)
πθ_old旧策略：上次更新前的参数每 2048 步（n_steps）复制一次
rt(θ)概率比：r>1=新策略更喜欢该动作—
At优势函数：A>0 好动作，A<0 坏动作用 GAE 计算
εClip 范围0.2（限制 r 在 [0.8, 1.2]）

这个公式的核心机制——分情况讨论

情况 A：At > 0（好动作，想增加概率）

L = min(rt, 1+ε) · At

rt 被限制在 1.2 以下。即使新策略把这个动作的概率翻了 10 倍（r=10），也只按 1.2·A 算——防止过度乐观。

情况 B：At < 0（坏动作，想减少概率）

L = max(rt, 1−ε) · At

rt 被限制在 0.8 以上。即使概率降到 0（r=0），也只按 0.8·A 算——防止过度惩罚，保留策略多样性。

为什么 ALC 任务中只有 PPO 能收敛？
ALC 的动作空间是舵面直接控制——轻微的动作变化就可能让飞机进入不可恢复的状态（失速尾旋）。PPO 的 clipping 阻止了单次更新中策略做大幅改动——策略只能"小步试探"。SAC（离线算法）用 replay buffer 旧数据更新，在复杂气动耦合下价值估计可能严重偏离；A2C（无 clipping）梯度方差大，策略会剧烈震荡甚至崩溃。

GAE: 广义优势估计（PPO 内部使用的优势函数）

GAE
AtGAE(γ,λ) = Σl=0∞ (γλ)l · δt+l
其中 δt = rt + γ·V(st+1) − V(st)   （TD 误差）
γ = 0.99（折扣因子），λ = 0.95（GAE 平滑参数——bias-variance 权衡）

直观理解：λ=1 → 蒙特卡洛回报（高方差低偏差），λ=0 → 一步 TD（低方差高偏差），λ=0.95 → 在两者之间取折中，对远期奖励略有平滑但不完全忽略。

PPO 总损失函数（SB3 实现）

Ltotal = LCLIP − c₁ · LVF + c₂ · S[πθ](st)
LCLIP裁剪后的策略损失（主项）权重 1.0
LVF价值函数损失 = (V(st) − Rt)²vf_coef = 0.5
S策略熵（鼓励探索的 bonus）ent_coef = 0.0（论文未使用）

PPO epoch × 10：同一批 2048 条数据重复用 10 次。每 epoch：计算 r_t(θ) = exp(log_prob_new − log_prob_old) → L_CLIP = mean(min(r_t·A_t, clip(r_t, 0.8, 1.2)·A_t)) → L_VF = mean((V_new(s_t) − R_t)²) → L_total = L_CLIP − 0.5·L_VF。

梯度更新：loss.backward() → clip_grad_norm_(max=0.5) → Adam(lr=2.5e-4).step()。PPO 的 clipping 确保单次更新的策略变化有限——这对 ALC 任务的训练稳定性至关重要。

6.3 随机种子：什么是 T0~T4？

论文 Fig.3 中每个子图有 5 条不同颜色的曲线，标注为 T0, T1, T2, T3, T4。

什么是随机种子？

"随机种子"是一个整数，用来初始化伪随机数生成器。相同种子 → 完全相同的随机序列 → 完全相同的训练结果。不同种子 → 不同的随机序列 → 不同的训练轨迹。

RL 训练中的随机来源：① 神经网络权重初始化 ② 目标点随机采样 ③ 动作探索噪声 ④ 环境初始条件的微小扰动。

为什么需要 5 次？

因为 RL 训练具有高度随机性。跑一次的结果不可信。

同一个代码、同一套超参数、只换了 seed——PPO-ALC 的两次训练可能天差地别：Seed 42 收敛到 180，Seed 99 停在 60。只跑一次你无法知道真实性能。

5 次求平均 + 标准差，才能诚实反映算法表现。

# 论文的实际做法（伪代码）
seeds = [0, 1, 2, 3, 4]
results = []
for seed in seeds:
    set_all_random_seeds(seed)
    model = PPO(policy="MlpPolicy", ...)  # 不同种子 → 不同初始化权重
    model.learn(total_timesteps=2_000_000)

6.4 训练曲线阴影区域的数学含义

论文 Fig.3 中的半透明阴影区域 = 均值 ± 1 标准差（不是 min~max 包络线）：

阴影上界 = mean(t) + std(t)
阴影下界 = mean(t) − std(t)
约 68% 的训练 run 在该区间内（假设近似正态分布）

阴影窄 = ?

5 次训练几乎一致 → 算法很稳定 → 你可以信任这个结果。例如 PPO-VLC 的阴影就很窄——5 次都收敛到相似水平。

阴影宽 = ?

5 次训练波动大 → 算法对 seed 敏感 → 需要更多 seed 或调参。例如 PPO-ALC 的阴影比 VLC 稍宽——论文说的 "slightly increased variability"。

阴影突然变宽 = ?

某次训练崩溃了（可能是策略更新幅度太大导致不可恢复的坏策略）→ 这是严重问题。例如 A2C-ALC 的阴影在后期暴宽——某些 seed 的策略已经崩了。

6.5 完整训练循环（Phase 1~5）

16 个环境并行采样：每个环境独立运行——obs → policy(obs) → action(含探索噪声) → JSBSim.step(action) → obs', reward, done。每 env 单独维护自己的 JSBSim 实例。

收集 n_steps=2048 步经验：16 envs × 128 steps/env = 2048 transitions。存储在 RolloutBuffer 中——每步记录 (obs, action, reward, done, value, log_prob)。

计算 GAE 优势：At = Σ(γλ)l·δt+l，从最后一步向第一步递推。Critic 的价值估计 V(s) 作为 baseline。

PPO 更新 ×10 epoch：同一批 2048 条数据重复用 10 次。每次：计算 rt=πnew/πold → 计算 LCLIP=min(rt·A, clip(rt,0.8,1.2)·A) → 反向传播 → 梯度裁剪(max=0.5) → Adam 更新。

评估 & 保存：每 1000 steps 用当前策略跑 5 个 eval episode（无探索噪声），记录平均奖励。如果当前是最优模型则保存。回到 Phase 1，共循环 ~976 轮。

6.6 目标点随机化

每次 episode 重置时，目标点在球坐标系中随机采样（而非笛卡尔坐标的均匀分布）：

d ~ U(dset−50, dset+50)    距离：450~550m
θ ~ U(1.41, 1.73) rad       仰角：限制高度在 [20, 180]m
φ ~ U(−π, +π) rad         方位角：全方向均匀
球坐标 → NED：xg=d·sinθ·cosφ, yg=d·sinθ·sinφ, zg=d·cosθ+100
为什么用球坐标？如果对 x,y,z 分别做均匀随机，目标点会集中在"对角线"方向。球坐标保证在三维空间中各方向概率密度均匀——RL 策略才能真正学到全方向的泛化能力。

8. 科研 Tricks —— 哪些可以直接用到你的项目里

1

用领域知识压缩观测空间

核心思想：原始传感器数据 ≠ 好的观测。用极坐标代替笛卡尔坐标表示目标位置。

# ❌ 原始位置差
Δx, Δy, Δz

# ✅ 极坐标表示——对飞行控制更有语义
Δψ = arctan2(Δy, Δx)    # 我该朝哪个方向飞？
d_h = √(Δx² + Δy²)       # 还有多远（水平）？
d_v = Δz                 # 还有多高？

维度越低、语义越清晰，RL 学得越快。花最多时间在观测设计上。

2

Tanh 非线性归一化 —— 保持"近场敏感"

问题：线性归一化在近距离时数值趋零、梯度消失。

d=500m: tanh(5.0)→0.999  # 饱和防过冲
d=50m:  tanh(0.5)→0.462  # 仍有梯度
d=5m:   tanh(0.05)→0.05  # 精细分辨

归一化不是简单的 (x-min)/(max-min)。根据物理意义选择归一化函数——在关键区间"拉伸"梯度。还有 log 归一化、指数归一化等。

3

奖励塑形的量级平衡

第一原则不是"想奖励什么"，而是先调权重让各项贡献量级接近：

水平距离 ~10m/step × 0.25 = 2.5
垂直距离  ~2m/step × 1.0  = 2.0
航向偏差 ~0.1rad  × 20   = 2.0

实用流程：①跑随机策略记录每项平均值 → ②调权使量级 ±20% 内 → ③根据优先级微调。

4

硬约束用终止，软约束用奖励

# ✅ 安全性 = 直接终止 episode
if altitude < 10 or altitude > 190:
    done = True

# ✅ 性能优化 = 密集奖励
reward = k1·Δd_h + k2·Δd_v + k3·Δψ

好处：不需要调惩罚系数 → 策略不因害怕惩罚而保守 → 训练更稳定。

5

5 次随机种子 + 独立训练

RL 训练具有高度随机性（初始化、目标采样、探索噪声）。跑一次的结果不可信。

论文每个算法跑 T0-T4 共 5 次独立训练，报告均值和方差。这是任何 RL 论文的底线。

刚好收敛 = 可能是运气；刚好没收敛 = 换 seed 可能就好了。

6

统一条件做公平比较

VLC 和 ALC 使用完全相同的奖励函数。

否则你无法判断：ALC 更好是因为奖励设计得好，还是控制层级本身更优？

做消融实验的铁律：一次只改一个变量，其余全部固定。

7

选对算法 —— 不是越新越好

ALC 任务中只有 PPO 可靠收敛。为什么？

PPO 的 clipped surrogate objective 阻止策略在复杂动作空间中做大幅更新——当直接控制舵面时，动作轻微变化就可能导致失稳。PPO 的"保守更新"恰好匹配了这个需求。

算法选择要匹配任务特性：稳定性 > 样本效率 → PPO > SAC。

8

多进程并行的工程智慧

num_envs = 16
env = SubprocVecEnv([make_env()
    for _ in range(num_envs)])

16 个 JSBSim 实例同时运行，经验汇总后统一更新。这是 CTDE（集中训练、分散执行） 的朴素但高效的实现。

加速迭代周期本身就是一种"算法提升"——比花里胡哨的模型改进更实用。

9. 设计哲学 —— "Less is More" 不是口号，是方法论

❌ 新手常见误区

  • 一上来就上 LSTM、Attention、Transformer
  • 网络越深越好、参数越多越好
  • 奖励函数越复杂越"智能"
  • 换最新算法 = 性能提升
  • 跑一次收敛就写论文

✅ OpenFlight 的做法

  • 2 层 MLP + 64 神经元 → 击败 PID
  • 6 维观测（极坐标）→ 比 11 维原始状态更有效
  • 3 项奖励（量级平衡）→ 驱动复杂机动
  • 不换算法换设计：观测 + 奖励 + 终止条件
  • 5 次独立训练 → 报告方差

网络越复杂的代价

代价说明
训练越慢16 并行的实时仿真等不起大网络推理
超参数越多调参调到崩溃——学习率、层数、head 数、dropout 率……
越容易过拟合仿真里完美，真机上一团糟（sim-to-real gap 更大）
🎯 OpenFlight 的核心方法论
简单的观测 + 简单的网络 + 简单的奖励 = 鲁棒的性能。
你的设计智慧比你的模型复杂度重要得多。把时间花在理解问题本质、设计好的观测表示、设计好的奖励信号上——而不是调网络层数和换激活函数。

作为 RL 小白的实操流程

  1. 定义 POMDP：状态、动作、观测、奖励、终止条件
  2. 用领域知识设计观测：信息压缩，维度越低越好
  3. 设计奖励：量级平衡 > 直觉正确
  4. 用终止条件处理安全约束：不要用惩罚
  5. 选一个稳定算法：PPO 是不错的默认选择
  6. 从简化版开始：先证明框架可行
  7. 多 seed 跑实验：至少 3 次独立训练
  8. 统一条件做消融：一次只改一个变量
