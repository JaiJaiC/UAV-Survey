2. Insight & Novelty

2.1 Core Insight

Core Insight
Why can't traditional robot safety methods handle open-world contextual safety requirements?
Traditional safety methods (CBF, HJ Reachability, temporal logic) require experts to pre-specify all safety constraints -- but in open worlds, safety is context-dependent: the same corridor demands different navigation strategies when crowded versus empty. A robot cannot anticipate every possible context before deployment and must therefore autonomously reason from visual observations about what constitutes safe behavior at runtime.

What is this paper's core insight?
Use a VLM as a "contextual safety reasoning engine." The VLM does not merely detect objects (e.g., a wet floor sign) -- it understands their semantic meaning ("a wet floor sign implies the surrounding area may be hazardous") and converts this understanding into executable spatial safety constraints. The key is threefold: (1) structure the VLM's reasoning into safety predicates such as ON(floor), AROUND(wet_floor_sign), BETWEEN(cones); (2) ground these predicates to pixel-level safe sets in physical space; (3) enforce safety via CBFs with probabilistic guarantees under VLM perception uncertainty.

2.3 Novelty Breakdown

Novelty 1: VLM-Driven Contextual Safety Reasoning -- Robot Autonomously Infers Safety Rules Without Prior Knowledge

[Problem Solved] Traditional safety methods require experts to manually specify safety constraints for each scenario, but open-world scenarios are infinitely variable. Existing language-driven safety methods support natural language descriptions of safety rules but still require users to provide them -- the robot cannot reason autonomously.
[Inspiration] VLMs excel at open-vocabulary understanding. Gemma 3 27B is sufficiently powerful for understanding image semantics. Existing work only uses VLMs for traversability estimation or trajectory selection -- never for autonomous safety constraint reasoning.
[Design] A structured safety predicate system: ON (traversable surfaces), NEAR (collision-risk objects), AROUND (hazardous area perimeters), BETWEEN (barriers formed by object arrangements). The VLM outputs JSON-formatted safety reasoning (safety_logic rationale + semantic classes + unsafe/safe region predicates). Chain-of-thought reasoning and safety logic instructions boost VLM reliability from 61% to 91%.

Novelty 2: Semantic-to-Spatial Grounding -- Converting NL Safety Constraints to CBF Barrier Functions

[Problem Solved] VLM outputs are natural language safety constraints (e.g., "AROUND(wet_floor_sign)") that cannot directly control a robot. A method is needed to convert these semantic constraints into executable spatial safe sets.
[Design] (1) Open-vocabulary segmentation (SAM3) localizes semantic classes identified by the VLM; (2) Construct pixel safe sets per predicate type -- ON/NEAR selects class pixels, AROUND applies morphological dilation, BETWEEN computes convex hulls; (3) Depth projection to 3D point cloud, accumulated into a 2D grid map with n_safe/n_unsafe counts per cell; (4) Estimate P[x is safe] = n_safe/(n_safe+n_unsafe) and threshold to obtain safe set S; (5) Construct CBF barrier function via signed distance field h(x).

Novelty 3: Probabilistic Safety Guarantees Under Perception Uncertainty

[Problem Solved] VLM perception is imperfect -- it can miss hazardous regions. How to provide safety guarantees in a probabilistic sense when the VLM may make detection errors?
[Design] Define a detection probability function m(delta x) modeling the Bernoulli probability of detecting an unsafe region from relative position delta x. Define inverse regularized distance d^(-1)(r; c, l) = c/(r+l). Via non-negative supermartingales and Ville's inequality, derive a probabilistic trajectory-length safety guarantee: if the system collects k*(t) observations at each timestep satisfying a bound on E[d^(-1)(r)], then P[exists t: x_t not in S] is at most delta -- the probability of leaving the safe set at any point along the trajectory is bounded by delta.

4. Task -- What Problem Does It Solve?

4.1 Problem Formulation

Core Problem -- Contextual Safety (Problem 1) $$ \text{Given: } \dot{x} = f(x) + g(x)u,\quad G,\quad \{I_t\} $$ $$ \text{Find } u_{\text{safe}} \text{ s.t. } (i)\; \{s_1,...,s_K\} \text{ inferred from } I_t, G;\; (ii)\; s_k \rightarrow S_k \subset \mathbb{R}^n;\; (iii)\; P\left[x \in \bigcap_k S_k\right] \ge 1-\delta $$
$x \in \mathbb{R}^n$ = robot state (2D position + heading) | $u$ = control input (linear velocity + yaw rate) | $G$ = high-level safety guide (e.g., "follow pedestrian social norms") | $\{I_t\}$ = streaming RGB-D observations | $s_k$ = contextual safety constraints (predicate form) | $S_k$ = grounded spatial safe sets | $1-\delta$ = probabilistic safety guarantee level | Core Challenge: the robot has no prior environment knowledge (no map, no predefined safety rules) and must infer what is safe purely from visual observations at runtime.

4.3 Task Type

This is a perception-reasoning-control fusion system-level safety framework, with core task type being contextual safety navigation. Specific scenarios include: (1) Contextual buffers -- identifying implicitly hazardous objects (e.g., forklifts in a warehouse) and maintaining safe distances around them; (2) Contextual barriers -- recognizing forbidden regions formed by object arrangements (e.g., traffic cones forming a line), understanding them as no-go zones rather than mere independent obstacles; (3) Contextual adaptive traversal -- distinguishing traversable from non-traversable surfaces (sidewalks vs. grass), even though both are physically traversable. Platform: mobile robot (differential-drive in simulation, Boston Dynamics Spot in real-world experiments); scenarios: warehouse, hospital, outdoor residential areas.

5. Challenge -- Why Is It Hard?

5.1 Essence (First Principles)

First-Principles Diagnosis
Core contradiction: safety is context-dependent, but traditional safety methods are context-free.

(1) The "Context Gap": Traditional obstacle detection only identifies "physical obstacles" -- but a row of traffic cones is not an obstacle; the "line" they form represents a forbidden zone. A wet floor sign is not an obstacle; it implies the surrounding area may be slippery. Understanding this semantic meaning requires world knowledge that traditional pipelines lack.

(2) The "Reasoning-Grounding Gap": Even if a VLM understands that "a wet floor sign means surrounding danger," converting this into an executable spatial constraint (e.g., "stay at least 1 meter away from the sign") is a language-to-space mapping problem. Language understanding yields abstract results; spatial constraints must be precise -- bridging the two requires a carefully designed grounding mechanism.

(3) The "Uncertainty Gap": VLMs can fail -- missing hazardous regions or misclassifying objects. If the robot enters a truly hazardous area at the moment the VLM misses it, safety is compromised. Traditional CBF theory assumes barrier functions are deterministic, but CORE's barrier functions are constructed online from noisy perception -- requiring safety guarantees under uncertainty.

5.3 Key Challenges

  • Challenge 1 -- Online Contextual Reasoning: The VLM must identify all safety-relevant semantics from a single image frame within seconds (~4.1s in CORE) and correctly apply spatial operators (when to use AROUND, BETWEEN, NEAR). VLM selection is critical -- Gemma 3 27B achieves 91.0%/85.0% safe/unsafe predicate accuracy, while Llava 34B manages only 49.0%/56.0%.
  • Challenge 2 -- Robust Semantic Grounding: VLM outputs must be precisely mapped to physical space. Segmentation noise causes "ghost obstacles" that incorrectly block safe paths. Depth projection errors misallocate safe/unsafe classifications to grid cells. Grounding errors are the dominant failure mode in real-world experiments.
  • Challenge 3 -- Safety Enforcement Under Perception Uncertainty: Build grid-count-based probabilistic safe sets, threshold to obtain S, construct SDF barrier function, and provide probabilistic guarantees when the VLM errs. Matching theory (Theorem 1) to empirical safety performance (93.3%) requires careful calibration of the detection probability function m(delta x).

6. Method & Key Formulas

6.1 Core Algorithm

CORE is a three-stage perception-reasoning-control system. Stage 1 (Contextual Safety Reasoning) uses a VLM to infer safety predicates from images; Stage 2 (Semantic Grounding) converts predicates into grid-count-based probabilistic safe sets and signed distance field barrier functions; Stage 3 (CBF Safety Enforcement) synthesizes minimally-invasive safe control via QP optimization and provides theoretical probabilistic safety guarantees.

6.2 Key Formulas

Group A: Image-Space Safe Set Construction

Eq.(2) -- Image-Space Safe Set $$ I_{\text{safe}} = \left( \bigcup_j I_j^{\text{safe}} \right) \setminus \left( \bigcup_k I_k^{\text{unsafe}} \right) $$
$I_j^{\text{safe}}$ = pixel set corresponding to the j-th safe predicate (e.g., all floor pixels for ON(floor)) | $I_k^{\text{unsafe}}$ = pixel set for the k-th unsafe predicate (e.g., dilated pixels for AROUND(wet_floor_sign)) | Logical meaning: safe region = union of all safe regions minus union of all unsafe regions. Unsafe takes priority over safe -- if a pixel belongs to both safe and unsafe regions, it is ultimately classified as unsafe.

Group B: Probabilistic Safety Estimation and Barrier Function Construction

Eq.(3)(4) -- Probabilistic Safety and SDF Barrier Function $$ P[x \text{ is safe}] = \frac{n_{ij}^{\text{safe}}}{n_{ij}^{\text{safe}} + n_{ij}^{\text{unsafe}}} $$ $$ h(x) = \begin{cases} \min_{y \in \partial S} \|x - y\| & \text{if } x \in S \text{ (safe)} \\ -\min_{y \in \partial S} \|x - y\| & \text{if } x \notin S \text{ (unsafe)} \end{cases} $$
$n_{ij}^{\text{safe}}, n_{ij}^{\text{unsafe}}$ = safe/unsafe point counts in grid cell (i,j) (accumulated from depth projections) | $S = \{x \mid P[x \text{ is safe}] \ge \tau\}$ = thresholded safe set (tau = 0.5) | $h(x) \ge 0$ = robot in safe region; $h(x) < 0$ = robot has left safe region | SDF gradient: $\nabla \hat{h}(x) = \frac{x - y^*(x)}{\|x - y^*(x)\|}$, where $y^*(x)$ is the nearest point on the safe set boundary -- efficiently obtained via grid lookup.

Group C: CBF Safety Control Synthesis

Eq.(5)(6)(7) -- CBF Forward Invariance and Safe Control $$ x(0) \in S \implies x(t) \in S \quad \forall t \ge 0 \quad \text{(Forward Invariance)} $$ $$ \langle \nabla h(x), f(x) + g(x)u \rangle \ge -\alpha(h(x)) \quad \text{(CBF Condition)} $$ $$ u_{\text{safe}} = \arg\min_u \|u - u_{\text{nom}}\|_2^2 \quad \text{s.t. CBF Condition} $$
$f(x), g(x)$ = control-affine dynamics | $\alpha(\cdot)$ = extended class-K function (CORE uses alpha(x)=0.25x) | $u_{\text{nom}}$ = nominal controller output (waypoint following + PID) | CBF condition meaning: the derivative of barrier function h along the system trajectory must be at least -alpha(h(x)). As h(x) approaches 0 (robot nears the safety boundary), this condition forces control to increase h (move away from boundary), maintaining forward invariance.

Group D: Probabilistic Guarantees Under Perception Uncertainty

Eq.(9)(10)(11) -- Detection Probability and Safety Guarantee $$ d^{-1}(r; c, \ell) = \frac{c}{r + \ell} \quad \text{(Inverse Regularized Distance)} $$ $$ \mathbb{E}[d^{-1}(r)]_{\{k\}} \le \int_0^\infty \frac{c(1 - m(r))^k}{(r + \ell)Z} dr \quad \text{(Expected Upper Bound After k Observations)} $$ $$ \text{Theorem 1: If } \mathbb{E}[d^{-1}(r)]_{\{k^*\}} \le \frac{c(\delta - \gamma)}{\ell} \text{, then } P[\exists t: x_t \notin S] \le \delta $$
$m(\delta x)$ = Bernoulli detection probability function (probability of detecting an unsafe region from relative position) | $r$ = distance to nearest undetected unsafe region | $c, \ell$ = positive hyperparameters (control function shape) | $k^*$ = minimum number of observations needed to satisfy safety condition | $\delta$ = total allowable risk (CORE sets delta=0.1, i.e., 90% trajectory safety) | $\gamma$ = risk share allocated to worst-case context | Key design: the measurement function m(delta x) handles temporal correlation in observations -- if a region is hard to detect at time t, it may be similarly hard at t+1. A two-stage decomposition (worst-case risk separation + conservative m) yields practical probabilistic guarantees.

7. Fundamental Principles

7.1 Control Barrier Functions (CBFs)

CBFs are grounded in Nagumo's theorem on set invariance. For a control-affine system $\dot{x} = f(x) + g(x)u$, a continuously differentiable function $h: \mathbb{R}^n \to \mathbb{R}$ is a CBF for safe set $S = \{x \mid h(x) \ge 0\}$ if there exists an extended class-$\mathcal{K}$ function $\alpha$ such that $\sup_u [\nabla h(x)^T (f(x)+g(x)u)] \ge -\alpha(h(x))$ for all $x$. This inequality ensures that when $h(x)$ approaches zero, the control can always push the system back into the interior, guaranteeing $S$ is forward invariant. CORE adapts this principle by constructing $h(x)$ online from VLM-derived safety semantics rather than assuming a pre-designed barrier function. The key insight is that the CBF condition can be enforced as a linear constraint in a quadratic program, making it computationally efficient (solved via CVXPY in milliseconds) while maintaining the formal forward invariance property when $h$ is correct.

7.2 Signed Distance Fields (SDFs) as Barrier Functions

The signed distance field $h(x)$ maps each point to its signed Euclidean distance to the boundary of safe set $S$: positive inside $S$, negative outside. SDFs possess the Eikonal property $\|\nabla h(x)\| = 1$ almost everywhere, ensuring the CBF gradient constraint has uniform scaling and is well-conditioned for QP solvers. In CORE, the SDF is not computed analytically but approximated via grid lookup on the 2D occupancy grid -- for each cell, the nearest boundary cell is found via Euclidean distance transform, yielding an efficient $O(N)$ computation where $N$ is the number of grid cells. The gradient $\nabla h(x)$ is approximated as the normalized vector from the current position to the nearest boundary point $y^*(x)$, which is exact when the safe set boundary is differentiable.

7.3 Open-Vocabulary Visual Grounding

The grounding pipeline rests on two pillars. First, SAM3 (Segment Anything Model 3) provides class-agnostic segmentation that, when prompted with VLM-identified semantic labels, localizes arbitrary object categories without retraining. This is critical because CORE cannot anticipate which semantic classes will appear in an open world. Second, the depth projection + grid voting mechanism converts 2D pixel classifications to a 3D probabilistic occupancy representation. Each pixel with depth $d$ is back-projected to a 3D point and assigned to the corresponding 2D grid cell. Over multiple frames, each cell accumulates counts $n_{ij}^{\text{safe}}$ and $n_{ij}^{\text{unsafe}}$, implementing a form of Bayesian occupancy filtering where $P[\text{cell } (i,j) \text{ is safe}] = n_{ij}^{\text{safe}} / (n_{ij}^{\text{safe}} + n_{ij}^{\text{unsafe}})$. This voting mechanism naturally handles segmentation noise by requiring consensus across multiple observations.

7.4 Probabilistic Safety via Supermartingale Theory

CORE's Theorem 1 provides a probabilistic safety guarantee rooted in Ville's inequality for non-negative supermartingales. A stochastic process $M_t$ is a supermartingale if $\mathbb{E}[M_{t+1} \mid \mathcal{F}_t] \le M_t$, where $\mathcal{F}_t$ is the filtration (history). The key construction defines a process based on $d^{-1}(r_t; c, \ell) = c/(r_t + \ell)$, where $r_t$ is the distance to the nearest undetected unsafe region. Under suitable conditions on the detection probability function $m(\delta x)$, this process is a non-negative supermartingale. Ville's inequality then bounds the probability that the supremum of this process exceeds a threshold, which translates into: the probability that the robot ever enters an undetected unsafe region is at most $\delta$. The two-stage decomposition (worst-case risk allocation $\gamma$ + conservative $m(\cdot)$) makes this bound practically useful despite the inherent conservatism of martingale inequalities. The detection function $m(\delta x)$ captures the spatial dependency of VLM perception quality -- objects farther away or at oblique angles are harder to detect, and this is explicitly modeled rather than assumed away.

7.5 Chain-of-Thought Reasoning for Structured Safety Output

The VLM is prompted with a safety logic instruction that enforces a structured reasoning pipeline: (1) enumerate all observable entities, (2) classify each as safety-relevant or neutral, (3) assign appropriate spatial operators (ON/NEAR/AROUND/BETWEEN), (4) justify each classification in natural language, (5) output final JSON with safe and unsafe predicate lists. This structured CoT approach transforms the VLM from a "black-box classifier" into a transparent safety reasoner whose intermediate steps can be inspected. The ablation study (removing CoT drops safe predicate accuracy from 91% to 61%) empirically validates that explicit reasoning steps are necessary -- the VLM cannot reliably jump from raw pixels to safety predicates without intermediate deliberation. This principle generalizes beyond CORE: any safety-critical LLM/VLM application should enforce structured, auditable reasoning traces rather than trusting end-to-end black-box outputs.

7.6 Separation of Concerns: Reasoning vs. Grounding vs. Enforcement

CORE's architecture embodies the systems principle of separation of concerns across three independent modules. The VLM reasoning module handles semantic understanding (what objects mean for safety) but not spatial precision. The grounding module handles spatial accuracy (where safe/unsafe regions are in metric space) but not semantic interpretation. The CBF enforcement module handles dynamic safety (how to stay safe given robot dynamics) but not perception. Each module can be independently improved, tested, and replaced: a better VLM improves reasoning without touching grounding; a better depth sensor improves grounding without retraining the VLM; a different CBF formulation improves enforcement without affecting perception. This modularity is essential for safety-critical systems where failures must be traceable to specific components.

9. Potential Flaw & Extensions

9.1 Limitations

  • Assumptions: (1) Assumption 1 (safe initialization) requires the robot's current position to be safe before any hazard is perceived as unsafe -- in practice this means the VLM must identify hazards before the robot enters the hazardous area. If the hazardous object is too distant or occluded, this assumption may fail. (2) The perception uncertainty model assumes average error rates and does not account for per-frame prediction uncertainty -- the VLM can be "confidently wrong" in a single frame, and CORE's model treats all frames equally.
  • Applicable scenarios: Currently only handles spatial safety constraints (expressed via spatial safe sets), not temporal safety constraints (e.g., "must not do Y before completing X"). For dynamically changing environments (moving people, opening/closing doors), temporal safety may be more important than spatial safety.
  • Scalability: VLM inference latency (4.1s) limits the robot's maximum safe speed -- in high-speed scenarios, the 4-second delay between VLM reasoning and control execution could allow the robot to enter hazardous areas before detection. There is an implicit tension between the observation count k* in the probabilistic guarantee theorem and the robot's actual speed.

9.2 Extension Directions

  • Dimensional extension: From spatial safety constraints to spatiotemporal safety constraints -- use Signal Temporal Logic (STL) to encode safety rules with time windows. For example, "during an emergency, must not remain at an exit for more than 5 seconds." This would enable CORE to handle more dynamic safety scenarios.
  • Condition relaxation: (1) Develop uncertainty-aware VLMs -- have the VLM not only output safety constraints but also confidence levels for each constraint. High-confidence constraints can be enforced aggressively; low-confidence constraints require more conservative handling. (2) Reduce VLM inference latency through distillation to make CORE applicable to higher-speed robot platforms.
  • Constraint addition: (1) Couple CORE's contextual safety reasoning with language-driven planners -- let safety reasoning not only block unsafe plans but actively guide the planner toward generating contextually safe task plans. (2) Consider higher-order dynamics scenarios -- e.g., jointly considering velocity and acceleration constraints with contextual safety rules in high-speed driving.

10. Motivation -- First-Principles Derivation

10.1 Idea Derivation Chain

  1. What is the essence of the problem?
    Robot safety is not merely "don't collide" -- in the real world, "safety" is tightly coupled with context. The same action may be safe or dangerous depending on the scenario. But traditional safety methods define safety as fixed, context-free constraint sets -- they cannot distinguish "crowded corridor" from "empty corridor," "emergency" from "normal operation."
  2. How have prior works handled this? Why is it insufficient?
    Three routes: (i) language-conditioned safety based on user instructions -- user describes safety rules, system grounds and executes; (ii) formal verification-based planning safety -- verifies logical consistency of planned action sequences; (iii) perception-based safety -- detects physical obstacles and avoids them. Route (i) still requires users to provide instructions; route (ii) cannot handle perception-level context; route (iii) completely lacks semantic understanding.
  3. What if the robot could "see" and "think" for itself?
    VLMs possess two key capabilities: (i) open-vocabulary object recognition -- can identify object categories beyond training data; (ii) world knowledge reasoning -- understands that "wet floor sign" is not just a sign but signals "surrounding area may be slippery." If a VLM is used for "online safety reasoning," the robot can autonomously understand safety implications in a scene.
  4. What else is needed to make reasoning results executable?
    VLM reasoning is at the language level -- it needs to be grounded to physical space (via segmentation + depth projection + grid map). The grounded safe set needs enforcement -- CBF is the most suitable choice because it is decoupled from the planner and provides formal guarantees. But CBF theory traditionally assumes deterministic barrier functions -- a probabilistic extension is needed to handle VLM perception uncertainty.
  5. What is the simplest complete solution?
    VLM observes one frame, outputs safety predicates, grounds to spatial safe set, CBF enforces, continues to next frame. The essence of this pipeline is connecting a common-sense reasoning engine (VLM) to safety control theory (CBF), with perception grounding as the bridge.

10.2 One-Sentence Motivation

One-Sentence Motivation
If a human can judge where is safe and where is dangerous within seconds of entering a completely unfamiliar room -- why can't a robot? CORE uses a VLM for safety reasoning, segmentation + projection for grounding to physical space, and CBF for enforcement -- giving robots, for the first time, the ability to "autonomously assess safety risks upon entering an unfamiliar environment."

10.3 How You'd Arrive at This Idea

  • Step 1 -- Observe the phenomenon: In Vijay Kumar group's SPINE and ROBOGUARD work, safety is defined through language instructions or semantic maps -- but these are all "humans telling the robot what is safe." Is there a way for the robot to judge for itself?
  • Step 2 -- Identify the capability gap: (1) VLMs can "understand" hazards in images (GPT-4V can answer "what safety hazards are in this image?"); (2) but VLM outputs are text -- not control commands; (3) CBFs can enforce safety constraints -- but need a barrier function, not a piece of text.
  • Step 3 -- Design the bridging mechanism: Bridge "VLM's textual reasoning" to "CBF's barrier function" -- through structured output (predicates) + semantic grounding (segmentation + projection + grid) + SDF (signed distance field). This bridge is CORE's most central engineering innovation.
  • Step 4 -- Handle uncertainty: Imperfect VLM means imperfect barrier function, requiring probabilistic safety guarantees. Borrow supermartingale techniques from stochastic safety analysis, model "VLM may miss detections" as a probabilistic event, and provide theoretical guarantees.

11. Key Takeaways

11.1 Lessons Learned

  • Methodology level: This is paradigm-level work at the intersection of LLM/VLM and control theory. CORE does not merely use a VLM as a classifier or detector -- it uses it as an online safety reasoning engine. This is an essential upgrade in the role of LLMs/VLMs in robot safety: from "tool" to "decision-maker." For UAV domain: during flight in unknown environments, UAVs can similarly use VLMs to reason about contextual safety from first-person perspective (e.g., "that area has circling birds, may be unsafe").
  • Technical skill level: (1) Structured prompt engineering -- not simply asking the VLM "is this safe?", but standardizing the reasoning process through safety logic + CoT + JSON format -- the reliability boost is decisive (61% to 91%); (2) Grounding pipeline design -- segmentation to pixel sets to point cloud to grid voting to SDF -- each step carefully chooses algorithms balancing precision and efficiency; (3) Theoretical framework for probabilistic safety -- incorporating VLM perception errors into safety analysis rather than simply ignoring them.
  • Experiment/writing level: The paper's comparative experiment design is well-layered -- Oracle (upper bound), CORE (this paper), No Context (no reasoning), Geometric (pure geometry) -- four tiers demonstrate the stepwise contribution of contextual reasoning. The ablation study is also convincing: removing CoT causes safe detection to drop from 91% to 61%, directly proving the necessity of reasoning.

11.2 Directly Applicable to My Research

  • For UAV safety navigation: CORE's "VLM reasoning + grounding + CBF enforcement" architecture can be directly adapted to UAV scenarios. During low-altitude flight, UAVs need to understand contextual hazards (power lines, tree branches, crowd gathering areas) -- all beyond traditional obstacle detection. Key adaptation point: CBF needs to consider UAV's 6-DOF dynamics and aerial-specific safety constraints (e.g., minimum flight altitude).
  • For multi-robot collaboration: CORE is currently single-robot -- combining with ROBOGUARD or SPINE-HT's multi-robot framework could build a context-aware multi-robot safety system. Core challenge: different robots' VLM reasoning results may be inconsistent -- how to make team-optimal decisions amid conflicting contextual safety inferences?
  • For human-robot interaction: CORE's high-level safety guide G (e.g., "follow pedestrian social norms") is essentially a human-robot interaction interface -- human operators can define safety values in natural language without writing code. This provides a safety foundation for non-technical users to operate robots.
