2. 核心洞察与创新点

2.1 核心洞察

🎯 核心洞察
问：为什么四足机器人的 sim-to-real 策略在仿真中表现良好、但在真实世界中能耗依旧居高不下？
答：因为现有的 sim-to-real 方法在仿真中优化的目标（如机械功率 $\sum \tau_i \dot{q}_i$、足端接触力、电机扭矩惩罚项等）只是电池总功耗的粗略近似——它们忽略了 FOC 驱动损耗、电池内阻压降、待机功耗、机械传动摩擦等仿真器无法建模的复杂物理/电气过程。手工设计的 proxy 不仅需要领域专家反复调参，而且优化方向可能与真实总功耗不一致甚至相反。

问：本文的核心洞察是什么？
答："难仿真不等于不可优化"——只要目标在真实世界中可以被测量，就可以用数据驱动的方式学习一个局部有效的 surrogate（替代模型），并在仿真中用它来指导策略优化。关键在于：（1）用少量真实数据训练 LSTM 网络预测电池总电流（从可观测的电机状态 $[\tau_t, \dot{q}_t]$ 到真实电流 $i_t$）；（2）在仿真中用 KL 散度约束将策略更新限制在 surrogate 可信的分布范围内，防止策略 exploit surrogate 的预测误差；（3）通过层级选择和多轮迭代，让 measurement model 和 policy 协同进化。

2.2 灵感来源→洞察映射

灵感来源→ 启发产生的洞察类型
传统 sim-to-real 中 domain randomization 只能处理"仿真不准"——无法处理"仿真根本没有"的量→ 对于电池总功耗这种在仿真器中完全缺失的量，domain randomization 完全无能为力——需要一个从真实数据中学习的测量模型来"补全"仿真器的盲区问题定义
Offline RL / model-based RL 中的"学习世界模型"范式→ 不是学习整个动力学世界模型（太复杂且不必要），而是只学习目标相关的测量模型——将高维状态映射到一个标量（功耗）。这是"最小必要模型"思路：学得越少，需要的数据越少，泛化越可靠方法
Trust-region policy optimization（TRPO）和 PPO 中的 KL 散度约束→ 将 KL 散度从一个"稳定训练"的工具升级为一个"防止策略利用 surrogate 漏洞"的安全约束——策略不能偏离 anchor 太远，因为 surrogate 在远离数据采集分布的地方预测不可靠数学 / 安全
DAgger (Dataset Aggregation) 和 evolutionary strategies 中的精英选择→ 每轮迭代都聚合所有历史真实数据来改进模型（DAgger 思路），同时通过"仿真 Top-K → 真实评估最优"的层级筛选来降低真实世界测试次数（evolutionary 思路），将两个互补的方法融合在一个框架中架构

2.3 创新点分解

创新点 1: 数据驱动的难仿真目标测量模型——将"仿真器盲区"变为"可优化目标"

【解决的问题】电池总功耗（包括电机驱动损耗、待机功耗、电池内阻损耗、机械摩擦等）在通用仿真器（Isaac Gym / MuJoCo）中缺乏精确的物理模型。传统方法靠手工设计解析 proxy（如 $\sum \tau_i \dot{q}_i + \frac{r}{k^2}\tau_i^2$），但无法捕捉完整的系统级功耗特性，且优化方向可能与真实总功耗不一致。
【灵感来源】既然人类无法为每个机器人写出准确的功耗公式，为什么不直接从数据中学？电池总电流是直接可测的物理量——只需要一个从"仿真器可见状态"到"真实电流"的轻量映射。
【具体设计】训练一个 LSTM 网络，输入为电机扭矩 $\tau_t$ 和角速度 $\dot{q}_t$（这些是仿真器可见的），输出为预测的电池总电流 $\hat{i}_t$。LSTM 的选择是因为功耗具有时序依赖性（电机加减速的历史影响当前功耗）。模型对所有特征和输出进行均值-标准差归一化。一旦训练完成，这个模型作为仿真器的"插件"，为策略提供功耗奖励：$r_{\text{power}} = -\hat{i}_t$。

创新点 2: KL 散度约束 + 多参数扫描——防止策略利用 surrogate 的预测漏洞

【解决的问题】将学习的 surrogate 嵌入仿真作为奖励函数后，RL 优化器（PPO）可能会exploit surrogate 的不准确性——找到仿真中"预测功耗极低"但真实世界中功耗并不低的轨迹。这是所有 model-based RL 都面临的 "reward hacking" 问题。
【灵感来源】Trust-region 方法的核心思想：策略更新步长不能太大，否则进入模型不准确的区域。KL 散度是衡量策略分布变化的自然度量。
【具体设计】约束策略 $\pi$ 与 anchor 策略 $\pi^{\text{anchor}}$（数据采集时的策略）之间的 KL 散度不超过阈值 $c$：$\mathbb{E}_{\mathbf{s}\sim\pi}[\text{KL}(\pi(\cdot|\mathbf{s}) \parallel \pi^{\text{anchor}}(\cdot|\mathbf{s}))] \leq c$。同时，对奖励权重 $\lambda \in \{0.5, 1, 5\}$ 和 KL 系数 $c \in \{0.2, 0.5, 1\}$ 进行网格扫描，每轮产生多个候选策略（而不是只训练一个），用多样性对抗不确定性。

创新点 3: 层级策略选择 + 数据聚合——仿真筛选 + 真实验证 + 模型迭代的三位一体闭环

【解决的问题】真实机器人测试昂贵且耗时（一次测试需要部署、充电、确保安全），不可能将所有候选策略都在真机上跑。但仅靠仿真（带 surrogate）排名又不可靠——surrogate 的排名和真实排名不一定一致。
【灵感来源】Evolutionary strategies 的精英选择 + DAgger 的迭代数据聚合——用仿真做粗筛（Top-6），用真实世界做精筛（选最优），每一轮的真实数据又用来改进下一轮的 surrogate。
【具体设计】每轮扫描产生多组策略 → 在仿真中用 surrogate 评估 → 取 Top-6 精英策略在真实机器人上部署测试 → 选真实功耗最低的作为下一轮 anchor → 所有真实测试数据加入训练集（从不丢弃历史数据）→ 更新 LSTM 模型 → 进入下一轮。经 3–4 轮迭代，策略和 surrogate 协同进化，共同逼近真实最优。

3. 系统架构

3.0 架构总览 —— 从仿真策略到真实功耗优化的闭环

数据采集 真实机器人

部署当前策略 Unitree Go1 执行行走运动；记录 ($\tau_t, \dot{q}_t, i_t$) 三元组。
累积全部历史数据 DAgger 风格：从不丢弃前轮数据。

测量模型 LSTM 训练

LSTM: $\hat{i}_t = f(\tau_t, \dot{q}_t)$ 输入：电机扭矩和角速度。输出：预测的电池总电流。
归一化与训练 均值-标准差归一化；MSE 损失；在所有累积数据上训练。

策略微调 仿真 + Surrogate

带 KL 约束的 PPO $r = r_{\text{sim}} + \lambda \cdot (-\hat{i}_t)$; KL ≤ c vs anchor。
参数扫描 网格扫描 $\lambda \in \{0.5,1,5\}$, $c \in \{0.2,0.5,1\}$ → 多组候选策略。

筛选与迭代 真实验证

层级筛选：仿真 Top-6 用 surrogate 奖励对候选策略排序，保留前 6 名。
真实机器人部署 真实表现最优的 → 下一轮 anchor；全部数据 → 训练集。
策略 → 真实数据 数据 → 模型 模型 → 奖励 → 策略 仿真排名 → 真实验证
关键瓶颈测量模型在分布外准确度下降；KL 约束不可或缺。
核心创新数据驱动 surrogate 替代手工设计的功耗 proxy。
为什么有效协同进化：更好的策略 → 更多样化的数据 → 更准确的模型。
当前局限需要 3–4 轮迭代；每轮仍需真实机器人测试。

4. 任务定义

4.1 问题建模

核心问题 $$ \min_{\pi} \; \mathbb{E}_{\pi}\left[\sum_{t} i_t\right] \quad \text{s.t.} \quad \dot{x}_{\text{cmd}} \approx v_{\text{target}},\quad \text{simulator has no model of } i_t $$
$i_t$ = 电池总电流（真实世界中可测量，但仿真器中完全不存在）  |  $\pi$ = 运动控制策略（将机器人状态映射为关节目标位置）  |  $v_{\text{target}}$ = 目标前进速度（0.5 / 0.8 / 1.1 m/s）  |  $\dot{x}_{\text{cmd}}$ = 指令速度  |  关键约束：不能牺牲速度跟踪精度来省电——指令速度必须被满足。策略必须在节能与任务执行之间找到 Pareto 最优平衡点。

4.2 输入 / 输出

类别内容维度
输入（策略）机器人本体感知状态（关节角度/速度、IMU、基座速度）+ 用户速度指令 $v_{\text{cmd}}$~60 维观测（Unitree Go1: 12 关节）
输出（策略）12 个关节目标位置（由 PD 控制器跟踪）→ 行走步态12 维动作空间
优化目标最小化电池总功耗（电流积分），同时保持速度跟踪精度在 0.5–1.1 m/s 速度下净功耗降低 24–28%
测量模型输入电机扭矩 $\tau_t$ 和角速度 $\dot{q}_t$（仿真中已知）24 维（12 关节 × 2）
测量模型输出预测的电池总电流 $\hat{i}_t$（真实电流的 surrogate）1 维标量

4.3 任务类型

这是一个 sim-to-real 策略优化问题，但与传统的 domain randomization 不同（后者解决"仿真器不准"），本文解决的是"仿真器中完全不存在"的目标量。该任务位于 model-based RL 与迭代学习的交汇处：从真实数据中学习一个局部世界模型（只建模目标指标，而非完整动力学），用它来指导仿真中的策略搜索，再用真实世界测试来修正和更新模型。平台：Unitree Go1 四足机器人（12 自由度，约 12 kg），场景：平地行走 + 室外长距离测试，核心指标：电池总功耗净降低百分比。

5. 关键挑战

5.1 本质（第一性原理分析）

💡 第一性原理诊断
核心矛盾：仿真器是一个物理近似——它知道"质量"、"摩擦"、"关节扭矩"，但完全不知道"电池电流"、"电机驱动芯片发热"、"FOC 开关损耗"。

（1）"已知的未知" vs "未知的未知"：Domain randomization 处理的是"已知的未知"（比如摩擦系数不准确——将其随机化），但电池总功耗属于"未知的未知"——仿真器中根本没有这个变量，因此 randomization 完全无效。

（2）手工 proxy 的维数灾难：为每个机器人和每个目标设计精确的解析 proxy，需要完整的电-机-热耦合模型——在实践中几乎不可能。更糟的是，proxy 的梯度方向可能与真实目标相悖（例如，机械功率 proxy 在 0.5 m/s 下仅节省 11.8% 净功耗，而数据驱动方法节省了 28.4%）。

（3）Surrogate 的分布外（OOD）陷阱：基于真实数据训练的 LSTM 在训练分布内预测准确，但 RL 优化天然会探索新颖步态模式——这些步态的 ($\tau, \dot{q}$) 分布可能远偏离训练数据。如果策略 exploit 了 surrogate 的预测误差，整个优化就会崩溃。

综合来看：你需要（a）从真实数据中学到的 surrogate 来建模仿真器缺失的量；（b）一个约束来防止策略进入 surrogate 不可信的区域；（c）持续的真实世界验证来修正 surrogate。这三个需求构成一个先有鸡还是先有蛋的循环——本文通过迭代闭环框架同时解决了它们。

5.2 先前方法及其局限性

先前方法核心思想为何对难仿真目标失效
手工设计的解析 Proxy 使用物理公式，例如 $\sum \max(\tau_i \dot{q}_i + \frac{r}{k^2}\tau_i^2, 0)$（机械功率 + 焦耳热） （1）每个目标都需要领域专家设计；（2）无法捕捉 FOC 驱动损耗、电池内阻、待机功耗；（3）可能给出错误的梯度方向——解析 proxy 仅节省 5–12% 净功耗，而本文方法达 24–28%
Domain Randomization 随机化仿真中的物理参数，使策略对参数变化具有鲁棒性 核心问题：只能处理仿真中"存在但不准确"的量，无法处理"根本不存在"的量。电池电流在 Isaac Gym 中根本不存在——无论怎么随机化也无法生成它。
系统辨识 + 高保真仿真器 通过实验测量机器人参数，构建更精确的仿真模型 （1）耗费大量人力——每台机器人都需重新辨识；（2）许多过程（FOC 动力学、电池老化）无法用简单参数化模型捕捉；（3）总会不断出现新的"难仿真"目标
纯真实世界 RL 直接在真实机器人上运行 RL，从真实数据中学习 （1）样本效率极低——数千个 episode 不可行；（2）安全隐患——探索可能导致摔倒；（3）无并行性——只能在单台机器人上串行采集数据

5.3 关键挑战

  • 挑战 1 —— 测量模型的 OOD 泛化问题：LSTM 在训练分布内预测准确，但仿真中的 RL 优化会探索新颖步态模式——其 ($\tau, \dot{q}$) 分布可能远偏离训练数据。如果没有策略更新的约束，优化器会 exploit surrogate 的预测误差，导致"仿真中改善但真实中恶化"。解决手段：KL 散度约束将更新限制在 trust region 内，确保 surrogate 预测的可靠性。
  • 挑战 2 —— 真实世界测试的样本效率：每轮需要在真实机器人上部署 Top-6 策略——每次测试都需要部署、监控和充电。如何在有限测试预算内找到显著改进（~3-4 轮 × 6 策略 = 18-24 次真实测试）？层级选择是关键，但仿真排名与真实排名的相关性决定了效率上限。
  • 挑战 3 —— 跨速度的统一优化：不同速度下的最优步态可能差异很大。如何确保优化后的策略在所有目标速度上都优于基线，而不是在某个速度上改善却在另一个上恶化？通过统一训练中覆盖多个速度指令，以及最终评估中验证全速度谱来解决。

6.2 关键公式

LaTeX 渲染由 MathJax 支持。显示公式使用 $$...$$，行内公式使用 $...$。

公式组 A: LSTM 测量模型 —— 从电机状态到电池电流

Eq.(1) — 数据驱动的测量模型 $$ \hat{i}_t = f_{\text{LSTM}}(\tau_t, \dot{q}_t; \theta) \qquad \mathcal{L}_{\text{MSE}} = \frac{1}{T}\sum_{t=1}^{T}(\hat{i}_t - i_t^{\text{real}})^2 $$
$\hat{i}_t$ = LSTM 预测的 $t$ 时刻电池总电流  |  $\tau_t$ = 12 个关节的电机扭矩向量  |  $\dot{q}_t$ = 12 个关节的角速度向量  |  $f_{\text{LSTM}}$ = 带有隐状态的 LSTM 网络，捕捉功耗中的时序依赖性  |  $i_t^{\text{real}}$ = 在真实机器人上测量的 ground-truth 电池电流  |  为什么用电机状态 ($\tau, \dot{q}$) 作为输入？这些量在仿真中精确已知，且是功耗的主要物理决定因素。LSTM 学习从机械功率到电气总功率的非线性映射（包括电机效率、驱动损耗等）。

公式组 B: 组合奖励 —— 任务奖励 + 数据驱动功耗奖励

Eq.(2) — 组合奖励函数 $$ r(\mathbf{s}_t, \mathbf{a}_t) = r_{\text{sim}}(\mathbf{s}_t, \mathbf{a}_t) + \lambda \cdot r_{\text{power}}(\hat{i}_t) $$ $$ r_{\text{power}}(\hat{i}_t) = -\hat{i}_t = -f_{\text{LSTM}}(\tau_t, \dot{q}_t; \theta) $$
$r_{\text{sim}}$ = 标准仿真奖励（速度跟踪误差 + 姿态稳定性 + 动作平滑度 + 存活奖励）  |  $r_{\text{power}}$ = 数据驱动的省电奖励：预测电流的负值（电流越低 → 奖励越高）  |  $\lambda$ = 功耗奖励权重（扫描值：$\{0.5, 1, 5\}$）——控制"省电 vs 完成任务"之间的权衡  |  关键设计选择：直接使用预测电流的负值——因为 LSTM 已学到真实总电流的完整映射，封装了仿真器无法建模的所有损耗分量。

公式组 C: KL 散度约束 —— 防止策略分布偏移

Eq.(3) — KL 散度约束 $$ \mathbb{E}_{\mathbf{s} \sim \pi}\left[ D_{\text{KL}}\left( \pi(\cdot \mid \mathbf{s}) \;\|\; \pi^{\text{anchor}}(\cdot \mid \mathbf{s}) \right) \right] \leq c $$
$\pi$ = 正在优化的策略  |  $\pi^{\text{anchor}}$ = anchor 策略（上一轮最优，用于数据采集）  |  $D_{\text{KL}}$ = KL 散度——衡量两个策略动作分布之间的差异  |  $c$ = KL 散度上限（扫描值：$\{0.2, 0.5, 1\}$）  |  为什么这个约束是必要的？LSTM 仅在 $\pi^{\text{anchor}}$ 产生的 ($\tau, \dot{q}$) 分布附近预测可靠。如果新策略偏离太远，LSTM 预测会变成"幻觉"。KL 约束将更新限制在"信任区域"内，防止 exploit surrogate 的弱项。

公式组 D: 解析功耗 Proxy（基线对比）

Eq.(4) — 解析功耗 Proxy（传统方法） $$ r_{\text{analytical}} = -\sum_{i=1}^{12} \max\left( \tau_i \dot{q}_i + \frac{r}{k^2} \tau_i^2,\; 0 \right) $$
$\tau_i \dot{q}_i$ = 第 $i$ 个关节的机械功率（扭矩 × 角速度）  |  $\frac{r}{k^2} \tau_i^2$ = 焦耳热损耗（$r$ = 电机绕组电阻，$k$ = 扭矩常数）  |  $\max(\cdot, 0)$ = 仅惩罚正功率  |  为什么不够？（1）忽略了 FOC 驱动开关损耗和待机功耗；（2）忽略了电池内阻；（3）忽略了电机效率变化；（4）$r$ 和 $k$ 随温度和老化变化。仅节省 5–12% 总功耗，而数据驱动方法达 24–28%。

7. 深层原理

理解本文需要把握三大核心原理：（1）为什么数据驱动 surrogate 优于手工 proxy——信息论视角；（2）为什么 KL 散度约束能防止策略 exploit——信任域优化视角；（3）为什么层级选择加数据聚合能实现高效闭环学习——主动学习视角。

7.1 数据驱动 Surrogate 的信息论优势

原理 1 —— Surrogate 建模了仿真器无法计算的互信息 $$ I(i_t^{\text{real}}; \tau_t, \dot{q}_t) > I(i_t^{\text{real}}; \tau_t, \dot{q}_t \mid \text{physics proxy}) $$
直观理解：手工 proxy 已经利用了 ($\tau, \dot{q}$) 的部分信息，但留下了大量"残差信息"未被建模——FOC 开关损耗、电池非线性放电、电机效率曲面。LSTM 从数据中自动发现并利用这些残差信息。手工 proxy 只用到了信息的"主成分"，而 LSTM 学到了完整的"非线性流形"。  |  为什么用 LSTM 而不是 MLP？功耗具有时序依赖性——加速和减速阶段的特性不同，关节间的耦合存在时序模式。LSTM 的隐状态以自然方式建模这些依赖关系。

手工 proxy 只利用了 ($\tau, \dot{q}$) 信息量的"主成分"，而 LSTM 学会了利用完整的"非线性流形"——包含电机效率曲面、FOC 开关模式、电池放电曲线。

7.2 KL 散度约束的信任域原理

原理 2 —— 为什么约束策略更新步长 $$ \mathbb{E}\left[ J(\pi) \right] \approx \mathbb{E}\left[ \hat{J}(\pi) \right] \quad \text{仅当 } \pi \approx \pi^{\text{anchor}} $$ $$ |\mathbb{E}[J(\pi)] - \mathbb{E}[\hat{J}(\pi)]| \propto D_{\text{KL}}(\pi \parallel \pi^{\text{anchor}}) $$
$J(\pi)$ = 真实世界中的期望总功耗（未知）  |  $\hat{J}(\pi)$ = 使用 LSTM surrogate 在仿真中估计的总功耗  |  核心关系：Surrogate 的预测误差大致与策略偏离 anchor 的程度成正比。当接近 anchor 时，LSTM 预测接近 ground truth；当偏离很远时，误差可能急剧增大——甚至出现符号反转（预测功耗更低但真实功耗更高）。KL 约束维持"$\pi$ 不能离开 surrogate 的置信区域"这一要求。这类似于 TRPO 的 line search——如果 KL 超过阈值 $c$，减小更新步长直到约束满足。

🧠 KL 约束的工作原理

每次更新后，检查 $\mathbb{E}[D_{\text{KL}}(\pi \parallel \pi^{\text{anchor}})]$。如果超过阈值 $c$，减小更新步长（类似于 TRPO 的 line search）直到满足约束。这防止了如"大步幅、低频"突变为"小步幅、高频"这样的剧烈变化。

为什么扫描 $c \in \{0.2, 0.5, 1\}$？不同的 $c$ 值对应不同的"探索半径"，为每个阶段找到最优的探索-利用平衡。

🔧 没有 KL 约束会发生什么

Reward hacking 示例：PPO 可能发现一种步态：扭矩极大但速度接近零（$\tau_i \gg 0, \dot{q}_i \approx 0$）。LSTM 从未见过这种组合——可能预测极低功耗。但实际中，虽然机械功率低，但电机堵转电流巨大 → 真实功耗其实非常高。KL 约束阻止了这种 OOD 跳跃，因为与正常行走策略的行为差异太大。

7.3 主动学习视角：层级选择 + 数据聚合

原理 3 —— 层级选择的信息采集效率 $$ \pi^* = \arg\min_{\pi \in \Pi^{\text{elites}}} J_{\text{real}}(\pi) \quad \text{其中 } \Pi^{\text{elites}} = \text{TopK}_{\pi \in \Pi^{\text{all}}} \hat{J}_{\text{sim}}(\pi) $$ $$ D_{k+1} = D_k \cup \{(\tau, \dot{q}, i)_t \mid t \in \text{来自测试 } \Pi^{\text{elites}} \text{ 的轨迹}\} $$
$\Pi^{\text{all}}$ = 本轮的全体候选策略（网格扫描产生约 9 个策略）  |  $\hat{J}_{\text{sim}}$ = 使用 LSTM surrogate 估计的功耗  |  $\Pi^{\text{elites}}$ = Top-6 精英集合  |  $J_{\text{real}}$ = 真实测试得到的实际功耗  |  $D_k$ = 累积训练数据集  |  主动学习视角：选 Top-6 进行真实测试是利用 surrogate 来选择"最具信息量"（最可能最优）的策略。新数据扩展了 LSTM 的训练覆盖范围，因为精英策略探索了新颖的步态区域。DAgger 风格的"从不丢弃旧数据"防止灾难性遗忘，保证覆盖范围单调增长。
💡 三大原理的协同交互
数据驱动 surrogate 填补仿真器盲区 → KL 约束确保 surrogate 预测仅在其置信区域内被使用 → 层级选择 + 数据聚合使 surrogate 与策略协同进化 → 三者形成闭环：更好的策略 → 更多样化数据 → 更准确的 surrogate → surrogate 引导策略找到更优步态。这个闭环是 24–28% 功耗降低背后的根本原因。

9. 局限与延伸

9.1 局限性

  • 假设条件：（1）目标量必须可靠可测——电池电流容易，步态噪声受环境背景影响，电机磨损需长期运行；（2）测量模型输入特征（$\tau, \dot{q}$）必须在仿真中精确已知；（3）速度跟踪精度不能明显恶化——需要精心调节 $\lambda$。
  • 适用场景：仅在平地行走和 3 个固定速度（0.5, 0.8, 1.1 m/s）上验证——在非平坦地形（斜坡、楼梯、碎石路）和连续速度谱上的泛化未经验证。
  • 可扩展性：每轮需要约 6 次真实机器人测试——单机器人可行，但扩展到更大参数空间或多目标优化（功耗 + 噪声 + 发热）会导致更高的 Pareto 前沿搜索成本。

9.2 延伸方向

  • 多目标：从单目标（功耗）到多目标（功耗 + 噪声 + 热 + 冲击）。每个目标一个独立 LSTM，组合奖励 = $r_{\text{sim}} + \sum_k \lambda_k \cdot r_k$。权重调节需要偏好学习或 Pareto 前沿跟踪
  • 减少真实测试：用贝叶斯优化替代网格扫描，利用 surrogate 不确定性选择候选策略。多机器人并行数据采集，各机器人独立 adaptor 层
  • 地形泛化：每种地形单独训练 LSTM，或使用条件输入（地形类型编码）训练统一模型。添加地形高度图到 LSTM 输入
  • 在线自适应：部署后持续小规模在线微调——用实时电流更新 LSTM，适应电池老化和电机温度漂移

9.4 值得深挖的方向

🔬 值得深挖的方向
将该框架应用于无人机续航优化——从地面机器人到空中机器人的自然延伸。无人机难仿真目标包括：（1）电池总功耗——ESC 驱动损耗、螺旋桨气动效率非线性（Blade Element Theory 不足）；（2）电机发热——悬停和巡航阶段温升不同，过热导致退磁；（3）振动与噪声——对侦察任务至关重要。框架可直接迁移：在真实无人机上飞基线 → 记录 IMU/ESC + 电流/温度/振动 → 训练 LSTM → 嵌入 AirSim/Gazebo → 带 KL 约束的 PPO → 迭代。挑战更大（3D 动力学、坠毁风险、预算更紧），但回报也更大——24–28% 的功耗节省意味着 25–40% 的额外飞行时间。

10. 动机推导

10.1 想法推导链

  1. 问题的本质是什么？
    足式机器人面临一个根本性矛盾：最重要的指标（功耗、噪声、发热、可靠性）恰恰是仿真器最差的部分。仿真器知道关节扭矩和速度，但不知道电池电流、PWM 开关损耗或齿轮箱摩擦热。这造成了系统性鸿沟：在仿真中"优化得很好"的策略，在真实世界中可能功耗极高。
  2. 先前工作如何处理？为什么不够？
    三条路线：（i）手工 proxy——将电-机系统压缩为 $\sum \tau_i \dot{q}_i + \frac{r}{k^2}\tau_i^2$，丢失大量信息（仅节省 5–12%）。（ii）Domain randomization——无法生成仿真中不存在的物理量。（iii）纯真实机器人 RL——样本效率低且危险。
  3. 如果不是改造仿真器，而是从数据中"外挂"一个模块呢？
    核心洞察：电池电流直接可测。电机扭矩和速度在仿真中精确已知。从真实数据训练一个轻量 LSTM 来学习 ($\tau, \dot{q}$) → $i$ 的映射，然后将其嵌入仿真。这就是用数据"填补"仿真器的盲区。
  4. 但 RL 会 exploit 这个模型——如何防止？
    KL 散度约束限制新策略不能偏离数据采集策略太远。这确保优化只在 LSTM 预测可信的区域内进行。
  5. 单轮不够——如何形成持续改进？
    迭代闭环：更好的策略 → 更多样化数据 → 更准确模型 → 更大的 trust region → 下一轮策略可以更优。

10.2 一句话动机

💡 一句话动机
仿真器能计算关节扭矩和速度，但对电池电流一无所知——那为什么不直接用一个轻量神经网络从真实机器人数据中学出 ($\tau, \dot{q}$ → $i$) 的映射，嵌入仿真作为能耗奖励，在 KL 约束下微调策略，让模型和策略通过迭代共同进步？

10.3 如果让你来提出这个想法，你会怎么推

  • 第 1 步 —— 观察：手工 proxy 仅节省 5–12% 真实功耗——信息丢失太多。直觉：与其写公式，不如从数据中学。
  • 第 2 步 —— 分解：功耗优化 = 找一个（a）能跟踪速度且（b）最小化真实电流的步态。（b）在仿真中缺失 → 从数据中学它。
  • 第 3 步 —— 匹配合适的工具：LSTM 处理时序功耗依赖性；KL 约束确保安全探索；PPO 保证稳定优化；网格扫描 + 层级选择应对不确定性。
  • 第 4 步 —— 组装：数据驱动 surrogate + KL 约束 + 层级选择 + 迭代数据聚合 = 完整框架。在 Unitree Go1 上验证了 24–28% 的净功耗降低。

11. 核心收获

11.1 经验教训

  • 方法论层面：本文展示了一种优雅的"用数据补充仿真器"的范式。不是对抗仿真器的局限（让它更精确）、也不是忽略它（让策略更鲁棒），而是承认仿真器在某些方面永远不够精确，并用真实数据训练一个小型"翻译"模型。这种"最小必要模型"（只针对目标相关量，而非完整动力学）极大降低了学习难度。
  • 技术技巧：（1）LSTM 输入仅用扭矩和速度——仿真中最可靠的量；（2）KL 约束 + 参数扫描 = 用计算量来对抗 surrogate 的不确定性；（3）数据累积（从不丢弃旧数据）避免灾难性遗忘——简单但常被忽视的设计。
  • 实验/写作层面：选择"总功耗"作为案例研究非常巧妙——一个直观、易量化、实践意义强且手工 proxy 确实效果不佳的指标，使对比极具说服力。

11.2 可直接应用于我的研究

  • 对于无人机运动优化：无人机领域面临完全相同的问题——电池功耗、电机温度、螺旋桨噪声都无法在仿真中精确建模。该框架可直接迁移：在真实无人机上飞基线 → 采集数据 → 训练 LSTM → 嵌入仿真 → 带 KL 约束的 PPO → 迭代。无人机测试更快（每次飞行仅数分钟）。
  • 对于无人机多目标优化：同时优化续航、噪声和热安全——每个目标独立 LSTM，组合奖励带可向量化权重。权重调节可利用偏好学习或约束优化。
  • 对于 sim-to-real 研究：核心哲学——"学习仿真到现实的残差映射"——具有广泛适用性。在无人机中，可学习仿真到真实的气动残差、IMU 噪声变换等，然后迭代微调策略。
