GaussFly 遵循 Real-to-Sim-to-Real 三阶段范式：真实场景采集 → 3DGS场景重建 → Isaac Sim 仿真训练 → 零样本真机部署。核心闭环：RGB图像 → Frozen ResNet-50 → 对比学习特征 → 拼接时序+本体+目标 → PPO策略(2层MLP) → [v, ω]速度指令 → 无人机执行

为什么用 3 帧时序嵌入？
单帧图像是静态快照——无法区分"正在靠近障碍物"还是"静止不动"。3 帧时序堆叠隐式编码了运动信息（光流、深度变化），让策略感知动态趋势。这是视觉运动策略中非常经典且有效的技巧。

2. 核心方法 —— 三层技术栈

💡 核心设计智慧：将视觉运动策略拆解为"重建-表征-决策"三个独立可优化的模块
GaussFly 的核心贡献不是单一技术突破，而是系统性地串联了三个领域的最新成果：3DGS 场景重建（计算机视觉）、对比表征学习（自监督学习）、PPO 策略优化（强化学习）。每个模块都可以独立改进，互不耦合。

2.1 层一：平面约束 3DGS 场景重建

3D Gaussian Splatting (3DGS) 是 2023 年提出的新型场景表示方法——用一组 3D 高斯椭球体显式表示场景，通过可微光栅化渲染新视角图像。GaussFly 对其做了两项关键改进：

平面约束 (Planar Constraint)

在优化 3D 高斯时，对位于平面区域（地面、墙壁）的高斯拉平——鼓励高斯椭球体在平面方向上延展、在法线方向上压扁。

效果：消除"漂浮物"伪影（floaters），产生几何准确的平面表面。

法线一致性 (Normal Consistency)

相邻高斯椭球体的法线方向应保持一致（如果它们在同一个平面上）。

效果：平面更加平滑连续，减少渲染噪声——这对后续的对比学习特征质量至关重要。

为什么 3DGS 优于 NeRF？NeRF 渲染慢（MLP 网络推理），3DGS 通过光栅化实现了 >100 FPS 的实时渲染。对于需要在 30Hz 实时仿真中训练 RL 策略的场景，渲染速度直接决定了训练可行性。且 3DGS 的显式几何表示比 NeRF 的隐式表示更容易添加平面约束。

2.2 层二：对比表征学习 (InfoNCE)

直接从 3DGS 渲染图像中端到端训练 RL 策略计算成本过高——每步都需要高维图像输入。GaussFly 的解决方案是：先预训练一个紧凑的视觉编码器，然后冻结它。

对比学习框架

InfoNCE
Lcontrast = −log [ exp(sim(z_i, z_j⁺) / τ) / Σk=1N exp(sim(z_i, z_k) / τ) ]
z_i, z_j⁺ 是正样本对（同一场景不同视角/光照/时间的渲染），z_k 是负样本。τ 是温度参数。目标：拉近正样本、推远负样本——学习对视角和光照不敏感的表征。
组件配置设计依据
编码器ResNet-50 (在 ImageNet 预训练)先用 ImageNet 提供通用视觉先验，再用对比学习微调到 3DGS 域
特征维度d = 2048 → 投影到 128高维特征 → 低维紧凑嵌入，减少后续 RL 策略的输入维度
正样本同一物理位置、不同渲染参数相机位姿微扰 ±5°, ±0.1m、光照微调——模拟真实世界的感知变化
负样本batch 内其他所有样本标准 SimCLR 风格——大批量是关键（通常 256-1024）
温度 τ0.07 (典型值)控制正/负样本间的区分度——τ 越小越"严格"
对比学习的核心直觉
一个好的视觉表征应该对任务无关的变化不敏感（光照、视角、纹理细节），但对任务相关的变化敏感（障碍物位置、目标方向）。InfoNCE 通过正负样本对比，隐式地教会编码器这个"不变性"——这正是 noise-resilient latent features 的来源。

2.3 层三：PPO 视觉运动策略学习

前两层做完后，RL 策略的输入已经从 224×224×3 = 150,528 维（原始图像）压缩到了 3×128 + 4 + 3 = 391 维（紧凑特征）。这是一个 ~385 倍的维度压缩，且保留了任务关键的视觉信息。

网络组件结构参数量
特征提取器ResNet-50 (Frozen) + 投影头 MLP(2048→128)~23M (冻结，不参与RL梯度)
策略网络FC(391→256) + ReLU + FC(256→256) + ReLU~166K
Actor 头FC(256→2): [μ_v, μ_ω] + [log σ_v, log σ_ω]~1K
Critic 头FC(256→1): V(s)~0.3K
可训练参数总计~167K
关键设计决策：解耦表征与策略
ResNet-50 不参与 PPO 的反向传播。这意味着：(1) 对比学习可以用大批量、长训练独立优化；(2) PPO 训练不需要处理 150K 维的输入梯度——内存和速度大幅优化；(3) 表征的稳定性保证策略不会因视觉特征突变而崩溃。这就是 ablation 中 "contrastive pre-training vs ImageNet" 对比的核心——解耦后的表征更鲁棒。

3. 论文精读

3.1 核心贡献

  1. 首次提出 Real-to-Sim-to-Real 流水线用于视觉运动策略：真实采集 → 3DGS 重建 → 仿真训练 → 零样本迁移，无需任何域随机化或微调
  2. 平面约束 3DGS：在 3DGS 优化中引入平面约束和法线一致性，生成几何精确的场景用于 RL 训练
  3. 对比解耦表征：用 InfoNCE 预训练 ResNet-50，将高维图像压缩为对噪声鲁棒的低维特征——表征和策略独立优化
  4. 零样本 Sim-to-Real：策略在仿真中训练后直接部署到 DJI Tello Edu，无需任何真实数据微调

🏆 关键发现
对比学习预训练比 ImageNet 预训练提升 20 个百分点——这证明了域内对比学习的价值。ImageNet 的特征虽然有通用性，但 3DGS 渲染图像与自然图像存在显著域差异（纹理、光照模型、几何结构），需要对比学习来"桥接"这个 gap。
D3QN 的离散化动作是瓶颈——连续速度控制需要精细的动作粒度，离散化导致的量化误差在现实任务中不可忽视。

4. 算法公式全解

GaussFly 的数学体系分为四个层次：3DGS 场景表示 → 对比学习目标 → 策略优化 → 奖励函数。

4.1 3DGS 场景表示

3DGS
G = { (μ_k, Σ_k, c_k, α_k) }k=1K
每个高斯椭球体由中心 μ_k、协方差 Σ_k、颜色 c_k（球谐函数）、不透明度 α_k 参数化
μ_k ∈ ℝ³高斯中心位置——椭球体在3D空间中的坐标
Σ_k = RSSᵀ协方差矩阵——控制椭球体的形状和方向。R 旋转矩阵，S 缩放矩阵
c_k颜色——用球谐函数编码视角依赖的外观 (view-dependent color)
α_k ∈ [0,1]不透明度——在 alpha blending 中的混合权重
K场景中高斯数量——典型场景几十万到几百万个

平面约束损失

L_planar
Lplanar = Σk∈P ‖n_k − n̄neighbor‖²
P 是被检测为"平面"的高斯集合。约束每个高斯的法线方向与邻域平均法线一致。

完整 3DGS 优化目标

L3DGS = (1−λ) · L1 + λ · LSSIM + βp · Lplanar + βn · Lnormal
L₁ 和 L_SSIM (λ=0.2) 是原始 3DGS 的渲染损失——保证视觉质量。β_p, β_n 控制平面约束和法线约束的权重。

4.2 对比学习损失

InfoNCE
LNCE = −(1/N) Σi=1N log [ exp(sim(f(x_i), f(x_i⁺)) / τ) / Σj=1N exp(sim(f(x_i), f(x_j)) / τ) ]
f(x) = proj(ResNet-50(x)): 224×224×3 → 128 维嵌入。sim(a,b) = a·b / (‖a‖‖b‖) 是余弦相似度。τ 是温度参数。

正样本生成策略

变换类型范围目的
相机位姿微扰±5° (rotation), ±0.1m (translation)学习视角不变性
光照强度变化×[0.7, 1.3]学习光照不变性
色温微调±500K学习白平衡不变性
高斯噪声σ = 0.02模拟传感器噪声

4.3 PPO 策略优化

观测定义

O
o_t = [ z_t, z_{t−1}, z_{t−2}, v_x, v_y, v_z, ψ, Δx_g, Δy_g, Δz_g ]
z_t = f(I_t) ∈ ℝ¹²⁸ 是当前帧的对比特征嵌入。3 帧堆叠提供时序信息。本体状态 4 维 + 目标 3 维 = 总共 3×128+7 = 391 维。

PPO Clipped Objective

PPO
LCLIP(θ) = 𝔼t[ min( rt(θ)·At,   clip(rt(θ), 1−ε, 1+ε)·At ) ]
rt(θ) = πθ(at|ot) / πθ_old(at|ot),   ε = 0.2
策略网络只有 2 层 MLP。ResNet-50 完全冻结——梯度不流经编码器。

4.4 奖励函数

r = rgoal + rcol + rstep + rprog
奖励项值触发条件设计意图
r_goal+100到达目标 (‖p−p_g‖ < ε)稀疏终端奖励——最强的正信号
r_col−50碰撞障碍物强惩罚——安全约束的硬编码
r_step−0.1每一步鼓励最短路径——拖延有代价
r_progλ·(d_{t−1} − d_t)每一步密集进度奖励——向目标靠近=正奖励
奖励设计的量级关系
r_goal (+100) >> r_col (−50) >> r_step (−0.1) ~ r_prog (~1-2/step)。成功到达的奖励远大于单步惩罚——策略不会因为害怕撞而永远不动。碰撞惩罚次之——让策略学会避障但不至于过度保守。步数惩罚和进度奖励在"底噪"层面微调行为。

5. 控制系统数据流全解

5.1 训练时数据流（一步闭环）

Isaac Sim (3DGS 场景渲染, 30Hz) │ ├─ 相机渲染: I_t ∈ ℝ^(224×224×3) ──► Frozen ResNet-50 ──► z_t ∈ ℝ^128 │ (当前帧 + 前2帧 → 3 个 128 维嵌入) │ ├─ IMU + 里程计: [v_x, v_y, v_z, yaw] ∈ ℝ^4 │ ├─ 目标信息: [Δx, Δy, Δz] ∈ ℝ^3 │ ├─ 拼接: o_t = [z_t, z_{t-1}, z_{t-2}, v_x, v_y, v_z, ψ, Δx, Δy, Δz] ∈ ℝ^391 │ ├─ 策略网络: FC(391→256)+ReLU → FC(256→256)+ReLU │ │ │ ├─► Actor: FC(256→2) → μ_v, μ_ω + log σ_v, log σ_ω │ │ 采样: a = tanh(μ + ε·σ), ε~N(0,I) │ │ a = [v_cmd, ω_cmd] ∈ [−1,1]² │ │ │ └─► Critic: FC(256→1) → V(o_t) │ ├─ 速度控制器: [v, ω] → Tello 动力学模型 │ p_{t+1} = p_t + Δt · [v·cos(ψ), v·sin(ψ), 0] │ ψ_{t+1} = ψ_t + Δt · ω │ ├─ 奖励计算: │ r = r_goal + r_col + r_step + r_prog │ └─ PPO 更新 (每 N 步): GAE → L_CLIP → Adam 更新 θ (仅策略网络)

5.2 推理时 (真机部署) 数据流

DJI Tello Edu 机载相机 (720p, 30fps) │ ├─ Resize → 224×224×3 │ ├─ ResNet-50 (Frozen, 加载预训练权重) → z_t ∈ ℝ^128 │ ├─ 维护 3 帧嵌入缓冲区: [z_{t-2}, z_{t-1}, z_t] │ ├─ IMU 读取: [v_x, v_y, v_z, yaw] │ ├─ 目标位置 (预设): [Δx, Δy, Δz] │ ├─ 拼接 → o_t ∈ ℝ^391 │ ├─ PPO 策略推理: model.predict(o_t, deterministic=True) │ → [v, ω] 速度指令 │ ├─ DJI Tello SDK: send_rc_control(v, ω) │ └─ 30Hz 闭环循环 ←────┘

注意：推理时不需要 Isaac Sim、不需要 3DGS 渲染。3DGS 仅在训练时用于生成仿真数据。真机部署只需要 ResNet-50 前向推理 (~5ms on edge GPU) + 策略网络前向 (<1ms) = 完全可以 30Hz 运行。

并行环境数88 个 Isaac Sim 实例并行

仿真频率30 Hz与真机控制频率匹配
特征提取器ResNet-50 (Frozen)不参与 PPO 训练——梯度阻断

6.2 训练流程 (Phase 1-4)

场景重建阶段：iPhone 拍摄 ~200 张真实场景照片 → COLMAP SfM 生成稀疏点云 → 平面约束 3DGS 优化 (~30min) → 导出 .ply 场景文件。

对比学习预训练：在 3DGS 场景中渲染多视角/多光照图像 → 构建正负样本对 → InfoNCE 训练 ResNet-50 (~2h) → 冻结编码器权重。

PPO 策略训练：8 个 Isaac Sim 实例并行 → ResNet-50 提取特征 (冻结) → 2层MLP 策略 → PPO 更新 (~5h, 1M steps)。

零样本部署：加载冻结编码器 + 训练好的策略 → 替换 Isaac Sim 为 Tello 机载相机 → 开环推理 → 30Hz 控制闭环。

6.3 仿真环境设置

  • 场景初始化：每 episode 随机无人机起始位置和目标位置——保证策略学习到全场景的泛化能力
  • 障碍物：3DGS 场景中的静态几何（墙壁、家具、走廊结构）
  • 终止条件：到达目标 (成功) / 碰撞障碍物 (失败) / 超过最大步数 (超时)
  • 渲染多样性：训练时随机微调光照参数（强度、色温）——这本身就是一种隐式的域随机化
为什么 zero-shot 能工作？
三个关键因素：(1) 3DGS 渲染质量高——平面约束消除了伪影，渲染图像与真实图像视觉相似；(2) 对比学习提供了域鲁棒特征——编码器对光照/纹理变化不敏感；(3) Tello 的简单动力学——小型四旋翼的动力学历程比固定翼简单得多，仿真 gap 本来就小。三者叠加使得 zero-shot 迁移成为可能。

8. 科研 Tricks —— 可以直接用到你项目里的技巧

1

解耦表征与策略 —— "先学看，再学做"

核心思想：不要让 RL 策略直接从像素学习——视觉表征和决策策略是两个应独立优化的子系统。

# Phase 1: 视觉预训练 (self-supervised)
encoder = ResNet50(pretrained=True)
train_contrastive(encoder, 3dgs_scene)  # InfoNCE
encoder.freeze()

# Phase 2: 策略训练 (RL)
policy = PPO(MLP(encoder.output_dim → action_dim))
train_rl(policy, encoder)  # encoder 不更新

好处：表征训练可以用大批量 + 长训练；策略训练输入维度大幅降低；两者互不干扰。

2

3DGS 几何约束 = 更好的训练数据

渲染训练数据时，数据的几何正确性比纹理真实性更重要。平面约束消除的 floaters 伪影如果出现在训练数据中，RL 策略可能学到利用这些"虚拟捷径"——真机上不存在这些伪影，策略就会失败。

这个原则推广到任何 sim-to-real 场景：确保仿真中的几何结构准确——纹理可以用域随机化弥补，但几何错误无法弥补。

3

时序帧堆叠编码运动信息

单帧图像是静态的，无法传达运动信息。3 帧堆叠是视觉运动策略中最经典、最有效的技巧——前向差分隐式编码了光流和深度变化。

# 简单的帧堆叠实现
frame_buffer = deque(maxlen=3)

def get_obs(new_image):
    frame_buffer.append(encoder(new_image))
    return concat(list(frame_buffer))  # 3×128 维

这比显式计算光流更简单且效果更好——网络自己学习如何利用时序信息。

4

正样本 = 域随机化的"软"版本

在对比学习中，正样本的生成方式直接决定了编码器对什么"不敏感"。GaussFly 的正样本变换（相机微扰 + 光照变化 + 色温变化 + 噪声）相当于告诉编码器：

"这些变化不改变任务相关的信息，请学习对它们不变"——这是一种比显式域随机化更优雅的方法。

5

稀疏 + 密集混合奖励

# r_goal:      稀疏——只有到达才有 (最难的 RL 设置)
# r_col:       稀疏——只有碰撞才有
# r_step:      密集——每一步的小惩罚 (鼓励高效)
# r_prog:      密集——每步的进度奖励 (引导探索)

纯稀疏奖励 → 探索困难。纯密集奖励 → 策略可能学"原地转圈刷分"。混合设计：稀疏奖励定义目标，密集奖励加速收敛。

6

Real-to-Sim-to-Real —— 反直觉但有效

传统范式是 Sim-to-Real（在仿真训练，迁移到真实）。GaussFly 的 Real-to-Sim-to-Real 多了一步"真实→仿真"的场景重建：

优势：真实场景的 3DGS 重建比手动搭建仿真场景更精确、更高效——特别是对于室内复杂环境。这为 sim-to-real 研究开辟了新范式。

7

Grad-CAM 验证策略注意力

Grad-CAM 热力图不仅是一个可视化工具——它是一个"信赖度检查"：策略是否在关注有意义的区域（障碍物边界、路径间隙），还是在关注与任务无关的纹理或背景？

如果 Grad-CAM 显示策略关注天花板花纹而非前方障碍物，你的策略显然有问题——这比直接看成功率更早发现问题。

8

冻结编码器 = 训练稳定性 + 推理速度

在 RL 训练中冻结视觉编码器带来三个好处：

  1. 梯度不流经编码器 → 节省 GPU 显存 (ResNet-50 反向传播需要大量显存)
  2. 表征稳定 → 策略不会因为编码器权重更新而"失忆"
  3. 推理更快 → 真机部署时编码器只需前向推理

9. 设计哲学 —— "分而治之"的视觉运动策略方法论

❌ 常见误区

  • 直接从像素端到端训练 RL 策略——视觉表征和决策混在一起
  • 认为仿真环境必须是人工搭建的 3D 模型
  • 对比学习只是"锦上添花"——在视觉 RL 中它是核心
  • 忽略 3DGS 的几何质量——觉得"看起来差不多就行"
  • Sim-to-Real 一定要域随机化或微调

✅ GaussFly 的做法

  • Phase 1: 重建——从真实数据自动创建高保真仿真场景
  • Phase 2: 表征——对比学习将图像压缩为鲁棒的紧凑特征
  • Phase 3: 决策——轻量 PPO 策略在紧凑特征上高效训练
  • 三阶段解耦——每阶段独立优化，互不耦合
  • 几何 > 纹理——精确几何比真实纹理更重要
🎯 GaussFly 的核心方法论
不要端到端解决一切。把复杂问题拆解为可独立优化的子问题。
重建 → 表征 → 决策：每个模块都可以替换为最新的技术（如 3DGS 换 Gaussian Opacity Fields，ResNet 换 ViT，PPO 换 Dreamer），而不影响其他模块。这种模块化设计是工程上最可持续的方案。

10. 学习路线 & 改进方向

学习路线

  1. 理解 3DGS —— 阅读 Kerbl et al. (SIGGRAPH 2023) 原始论文，理解高斯泼溅的渲染原理
  2. 理解对比学习 —— 阅读 SimCLR (Chen et al. 2020) 和 MoCo (He et al. 2020)，搞懂 InfoNCE 的数学
  3. 复现 3DGS 重建 —— 用手机拍一个房间，用开源 3DGS 代码重建场景
  4. 训练对比编码器 —— 在 3DGS 场景中渲染训练数据，用 SimCLR 训练 ResNet-50
  5. 接入 PPO 训练 —— 用 Stable-Baselines3 在 Isaac Sim（或简化版 Gym 环境）中训练策略
  6. 真机部署 —— 如果有 Tello 或类似平台，尝试零样本迁移
