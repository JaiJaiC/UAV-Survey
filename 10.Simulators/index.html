Detailed Profiles

Isaac Sim + Isaac Gym

NVIDIA Omniverse-based. RTX photo-real rendering with PhysX 5. GPU-parallel RL training (10K+ envs). Domain randomization pipeline. Pre-built UAV assets.

Photo-realGPU RLROS2Domain Rand
OfficialGitHub

Flightmare

UZH RPG. Unity-based, decoupling rendering (GPU) and physics (CPU). Supports quadrotor + fixed-wing. Flexible sensor suite. RL toolkit integration.

UnityRL ToolkitFast RenderMulti-vehicle
DocsGitHub

RflySim

BUAA Reliable Flight Control Group. Unreal Engine rendering + PX4 SITL + Pixhawk HITL. CopterSim for rapid iteration. Fault injection capability.

Unreal EnginePX4 SITL/HITLFault Injection
OfficialGitHub

AirSim (deprecated)

Microsoft Research. Pioneered photo-real UAV simulation. Unreal Engine. Excellent for computer vision research. No longer actively maintained (2023).

XTDrone

Xiangke Wang @ NUDT. Multi-rotor + fixed-wing + VTOL simulation platform. PX4 + Gazebo integration. Pre-built models for common vehicles. Motion planning + computer vision modules.

Aerial Gym

Edinburgh. Isaac Gym-based GPU-parallel simulation. Training 10K+ quadrotors simultaneously. RL-focused. Full tensor-based physics.

How to Choose

For RL Training

Isaac Sim/Gym (massive parallel) or Flightmare (fast rendering with RL toolkit). Aerial Gym for GPU-scale training.

For PX4 / Hardware Testing

RflySim (HITL, fault injection, PX ecosystem) or Gazebo + RotorS (ROS integration). XTDrone for multi-platform support.

For Computer Vision

AirSim (best photorealism, but deprecated) or Isaac Sim (active dev, RTX rendering). Flightmare for fast data generation.

For Multi-UAV Swarms

XTDrone (built-in multi-vehicle), RflySim (PX4 swarm support), or Gazebo Ignition (native multi-robot).
