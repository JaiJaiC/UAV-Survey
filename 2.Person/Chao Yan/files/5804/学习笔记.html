1. 系统架构全景图

GDM-MADRL 是首次将生成扩散模型引入分布式固定翼无人机集群控制的论文。系统构建在 Leader-Follower 架构 之上，每架无人机通过 扩散策略网络 + GAT 邻居信息聚合 + GAT 值网络 实现分布式集群飞行。

1.1 整体框架：Dec-POMDP + Leader-Follower

系统将固定翼集群建模为去中心化部分可观测马尔可夫决策过程 (Dec-POMDP)。一架 Leader 沿预设轨迹飞行，N 架 Follower 通过分布式策略保持集群形态——每架 Follower 只依赖局部观测（自身状态 + Leader + 邻居），不需全局通信或中心协调。

系统三大核心模块
模块功能核心创新
扩散策略网络 基于当前观测生成动作——通过迭代去噪产生高质量动作 首次将扩散模型用于集群控制策略——捕获多模态动作分布
GAT 邻居信息聚合 用图注意力网络 (GAT) 处理邻居无人机的状态信息 自动学习哪些邻居的信息更重要——而非等权平均
GAT 值网络 将 GAT 聚合的观测也输入 Critic——准确估计 Q 值 Critic 和 Actor 共享 GAT 编码器——信息一致性

1.2 可扩展的 Leader-Follower 架构

Leader-Follower 是集群控制的经典范式，但传统方法面临扩展性瓶颈——规则数量随集群规模急剧增长。GDM-MADRL 的方案：

  • Leader：按预设轨迹飞行（或由独立策略控制），提供全局参考方向
  • Follower：每架运行相同的 GDM-MADRL 策略（参数共享），从局部观测做决策
  • 可扩展性：新增 Follower 只需复制策略网络——GAT 自动适应邻居数量变化
核心设计思想：扩散模型探索多模态 + GAT 图感知 + 固定翼动力学约束
固定翼集群的独特性在于——有非完整约束（不能悬停、不能侧飞）、高速（不能急刹）、机动能力有限。确定性策略只能学到一个"平均"行为，扩散模型能学到多个可行方案（如绕过障碍物从左边或右边走），GAT 让每架无人机自动关注"最重要的邻居"而非所有邻居。

2. 核心方法深度解析

2.1 为什么用扩散模型做策略？—— 多模态动作分布

这是本文最具颠覆性的思想。传统 RL 策略输出高斯分布（单峰）——只能表示"一个最佳动作周围的小范围随机"。但在集群避障场景中，可能有多个同样合理的动作（从左边绕过 vs 从右边绕过）。高斯分布只能选一个，扩散模型可以同时覆盖多个。

高斯策略 (传统 RL)

π(a|s) = N(μ_θ(s), σ²)

# 问题：遇到障碍物时
# 左绕：roll_cmd = -0.5
# 右绕：roll_cmd = +0.5
# 高斯分布：μ = 0（直飞撞墙）
# —— 学到的"平均"是灾难

单峰分布无法同时表示两个相反的动作方向

扩散策略 (GDM-MADRL)

π(a|s) = p_θ(a^0|s)
where a^0 = denoise(a^T, s, T steps)

# 遇到障碍物时：
# 采样1 → 去噪 → roll=-0.52 (左绕)
# 采样2 → 去噪 → roll=+0.48 (右绕)
# —— 两个方案都在分布中

多峰分布自然覆盖多个可行方案

2.2 GAT (图注意力网络) —— 邻居信息的选择性聚合

固定翼集群中，每架无人机通常只关心最近的几个邻居——远处的邻居对当前决策影响极小。GAT 通过注意力机制自动学习邻居权重：

GAT
αij = softmaxj( LeakyReLU( aT [W·hi || W·hj] ) )
h'i = σ( Σj∈N(i) αij · W·hj )
αij：无人机 i 对邻居 j 的注意力权重。|| 表示向量拼接，a 是可学习的注意力向量。
符号含义在集群中的物理意义
hi无人机 i 的自身状态特征位置 + 速度 + 航向
hj邻居 j 的状态特征邻居的位置 + 速度 + 航向
N(i)无人机 i 的邻居集合通信/感知范围内的其他 Follower + Leader
αij注意力权重最近的邻居 → 权重高；远处的邻居 → 权重低

2.3 扩散模型的策略网络 —— 从噪声到动作

扩散策略的核心是一个条件去噪过程。在推理时，从一个纯噪声向量开始，以当前观测（GAT 处理后）为条件，经 T 步迭代去噪，逐步生成最终动作。

初始化：从高斯噪声中采样 aT ~ N(0, I)——完全随机的动作候选

迭代去噪 (t = T → 1)：at-1 = εθ(at, t, c) —— 条件去噪网络 εθ 以观测 c 为条件，逐步去除噪声

最终动作：a0 = [ω, u] —— 去噪后的动作向量：滚转角速率指令 + 速度指令

执行与评估：a0 作用于固定翼动力学 → 新状态 → 计算奖励和 Q 值

2.4 条件去噪网络

去噪网络 εθ 的输入包含三部分：

  1. 噪声动作 at：当前去噪步骤的动作（仍含噪声）
  2. 去噪步数 t：位置编码——告诉网络"现在去噪到了第几步"
  3. 条件 c = GAT(o)：GAT 处理后的观测特征——这是扩散模型知道"该生成什么动作"的根据

网络结构：MLP 编码器——简单高效，不需要 U-Net 等图像扩散模型常用的复杂结构（因为动作空间是低维连续向量，不是高维图像）。

3. 论文精读

3.1 核心贡献

  1. 首次将生成扩散模型应用于分布式固定翼无人机集群控制——开创性工作
  2. 设计扩散策略网络：GAT 处理观测作为条件 → 迭代去噪 → 生成高质量多模态动作
  3. GAT 同时用于 Actor 和 Critic：邻居信息在图结构中自然聚合，Q 值估计更准确
  4. 可扩展架构：Follower 数量可任意增减——参数共享 + GAT 自适应
  5. HITL (硬件在环) 验证：不仅在仿真中——论文还进行了半物理仿真验证

3.2 固定翼动力学与约束

固定翼无人机与四旋翼有本质区别，这决定了集群控制策略的设计：

固定翼约束
Nonholonomic: ẋ·sin(ψ) − ẏ·cos(ψ) = 0（不能侧滑）
Speed: vmin ≤ v ≤ vmax（必须保持空速，不能悬停）
Turn rate: |ω| ≤ ωmax（转弯速率受限）
这些约束使得"停下来等队友"或"急转弯避障"在固定翼上难以实现——策略必须考虑前向速度和转弯半径
特性四旋翼固定翼对集群策略的影响
悬停能力✅ 可以❌ 不能固定翼不能"等人"——必须持续前飞
侧向移动✅ 全向❌ 非完整约束固定翼只能向前飞——调整方向必须转弯
速度范围0 ~ v_maxv_stall ~ v_max有最低空速——低于失速速度会坠毁
转弯半径0（原地转）v²/(g·tan(φ))速度越快转弯半径越大——速度和转弯耦合

3.3 观测空间设计

每架 Follower 的观测由四部分组成：

观测分量含义维度
自身状态位置、速度、航向角、滚转角~6 维
Leader 信息Leader 的相对位置和速度~4 维
邻居状态 (经 GAT)各邻居的相对位置和速度N×4 维（经 GAT 聚合为固定维度）
9 光束距离传感器前方 180° FOV 的 9 条射线距离9 维
观测设计要点：9 条射线覆盖前方 180° —— 固定翼不需要后方传感器（只能前飞）。每条射线返回最近的障碍物距离，归一化后作为网络输入。

3.4 动作空间

动作向量
a = [ω, u]T
ω: 滚转角速率指令 (roll command) —— 控制转弯。u: 速度指令 (speed command) —— 控制前进速度。

这是一个 2 维连续动作空间。注意：固定翼通过滚转来转弯——roll → bank → 产生侧向力 → 改变航向。策略必须理解这个间接控制链。

3.5 对比方法

方法类型核心特点
Reynolds Rules规则基础三条经典规则：分离、对齐、凝聚——简单但缺乏自适应
Potential Field势场法虚拟力驱动——吸引力(目标)+排斥力(障碍/友机)——容易陷入局部极小
MADDPGMADRL (确定策略)输出确定性动作——高斯噪声探索——单峰策略
MAPPOMADRL (高斯策略)输出 Gaussian 均值和方差——比 MADDPG 更好的探索——但仍单峰
GDM-MADRLMADRL (扩散策略)输出多模态动作分布——扩散模型 + GAT 聚合

3.6 HITL 硬件在环实验

这是 GDM-MADRL 的另一个亮点——不仅有仿真，还进行了硬件在环 (Hardware-in-the-Loop) 半物理仿真。将训练好的策略部署到真实的飞控硬件（如 Pixhawk），在仿真环境中运行——验证策略能在真实硬件约束下工作。

HITL 验证的关键优势：

  • 验证了推理速度：扩散模型的多步去噪是否能在嵌入式设备上实时运行
  • 验证了通信延迟：GAT 所需的邻居信息交换在真实通信条件下的可行性
  • 验证了控制兼容性：扩散策略输出的 [ω, u] 指令能否被飞控正确解析和执行
HITL 不等于实机飞行：HITL 中飞行器仍然是仿真的（只是飞控硬件是真实的）。从 HITL 到真实试飞还有一段路——但已比纯仿真可信度高很多。

4. 算法公式全解

4.1 Dec-POMDP 形式化

Dec-POMDP
Dec-POMDP = ⟨ N, S, {Ai}, T, R, {Ωi}, {Oi}, γ ⟩
与 POMDP 的关键区别：每架无人机有独立的观测函数 Oi和动作空间 Ai——真正的去中心化决策

4.2 扩散模型 —— 前向加噪过程 (Forward Diffusion)

Forward Process
q(at | at−1) = N( at ; √(1−βt)·at−1 , βt·I )
q(at | a0) = N( at ; √ᾱt·a0 , (1−ᾱt)·I )
βt ∈ (0,1)：噪声调度参数。ᾱt = ∏s=1t (1−βs)：累积信号保留率。训练时从数据 a0 直接采样 at。
符号含义在 GDM-MADRL 中的设置
a0真实动作（去噪后）[ω, u] ∈ R²，来自离线专家数据或在线交互
at加噪 t 步后的动作噪声逐渐掩盖原始动作信息
T总去噪步数通常 100~1000 步；推理时可用 DDIM 加速到 10~50 步
βt噪声调度线性调度 (β₁=1e-4, β_T=0.02) 或余弦调度

4.3 扩散模型 —— 逆向去噪过程 (Reverse Denoising) ⭐ 核心

Reverse Process
pθ(at−1 | at, c) = N( at−1 ; μθ(at, t, c) , σt²·I )
c = GAT(o) 是条件——GAT 编码后的观测。μθ 由去噪网络预测，σt 由噪声调度确定。

去噪网络的训练目标

Diffusion Loss
Ldiff = Ea⁰, ε, t[ || ε − εθ( at, t, c ) ||² ]
ε ~ N(0,I)：真实注入的噪声。εθ：网络预测的噪声。训练目标：让网络学会从加噪动作中识别并分离出噪声。

推理时的完整去噪流程

# 推理：给定观测 o → 生成动作 a
a_T = randn(2)  # 纯噪声： [ω_noise, u_noise] ~ N(0,I)
c = GAT(obs)   # 条件：GAT 聚合邻居 + 自身状态

for t in range(T, 0, -1):   # 从 T 步去噪到第 1 步
    ε_pred = denoise_net(a_t, t, c)  # 预测当前噪声
    a_{t-1} = remove_noise(a_t, ε_pred, t)  # 减去预测的噪声
    if t > 1:
        a_{t-1} += σ_t * randn(2)  # 加少量随机噪声（随机性来源）

a_0 = clamp(a_0, -1, 1)  # 最终动作 [ω, u]

4.4 GAT —— 图注意力网络

GAT Layer
h'i = ||k=1K σ( Σj∈N(i) αkij · Wk·hj )
K 头注意力拼接——每个头学习不同的邻居关系模式。多头结合后经过线性投影得到最终特征。

GAT 在 Actor 和 Critic 中的双重使用

Actor 中的 GAT

o_raw = [self_state, leader, neighbors]
c_actor = GAT_actor(o_raw)
a = diffusion_policy(c_actor)

GAT 编码邻居 → 条件向量 → 扩散去噪 → 动作

Critic 中的 GAT

o_raw = [self_state, leader, neighbors]
c_critic = GAT_critic(o_raw)
Q = value_net(c_critic, a)

GAT 编码邻居 → Q 值评估——确保 Critic 的信息视角与 Actor 一致

4.5 集群奖励函数 —— Reynolds 三规则的 RL 版本

Flocking Reward
R = wcoh·Rcohesion + wsep·Rseparation + walign·Ralignment + wobs·Robstacle
奖励项对应 Reynolds 规则计算方式
Rcohesion凝聚 (Cohesion)−∥pi − p̄neighbors∥ —— 与邻居平均位置的距离
Rseparation分离 (Separation)−Σ exp(−dij/σ) —— 与邻居距离太近的惩罚
Ralignment对齐 (Alignment)−(vi − v̄neighbors)² —— 与邻居速度差
Robstacle避障 (扩展)−Σ exp(−dobs/σ) —— 接近障碍物的惩罚
Rleader追随 (扩展)−∥pi − pleader − Δdesired∥² —— 相对 Leader 保持编队位置
从规则到奖励：Reynolds 的 Boids 模型用三条确定性规则驱动集群行为。GDM-MADRL 将这三条规则转化为奖励信号——RL 学习何时以及如何权衡各规则，而非硬编码优先级。

6. RL 训练详解 —— 扩散模型 + MADRL 的训练挑战

6.1 训练的两阶段策略

扩散模型的训练需要大量数据。GDM-MADRL 采用两阶段训练：

预训练扩散模型 (Behavior Cloning)：用传统控制器（如 Reynolds + Potential Field）生成大量 (观测, 动作) 对，训练扩散策略学习"基本的飞行行为"

在线微调 (RL Fine-tuning)：将预训练的扩散策略作为初始策略，用 RL（如 Q-learning / Policy Gradient）在线微调——提升在复杂场景中的表现

多智能体协调训练：多架 Follower 同时运行，共享扩散策略参数——CTDE 范式

两阶段训练的关键：纯 RL 从零开始训练扩散策略极其困难（去噪步数 × 高方差梯度）。先用 BC 获得"不会坠毁"的初始策略，再用 RL 提升——大幅降低训练难度。

6.2 扩散策略的 RL 更新

如何在 RL 框架中更新扩散模型的参数？

# 扩散策略的 Actor 损失
# 两种选择：

# 方案 A: 直接用 Q 值引导去噪过程（类似 Classifier Guidance）
L_actor = -Q(s, a^0)  where a^0 = denoise(a^T, s)

# 方案 B: 用去噪损失（类似 Diffusion Policy / Diffusion-QL）
L_actor = E[ ||ε − ε_θ(a^t, t, s)||² − α·Q(s, a^0) ]
# 两项：去噪精度 + 动作价值最大化

6.4 推理加速 —— DDIM 采样

完整的 T=100 步去噪在实时控制中太慢（每步 30Hz 控制意味着 3ms 内必须输出动作）。解决方案：

标准 DDPM 采样

T = 100 步
每一步: 网络前向 + 噪声去除
总网络前向: 100 次
时间: ~50ms (不可接受)

DDIM 加速采样

T_eff = 10~20 步
跳步采样: 从 {1,2,...,100} 中选子序列
总网络前向: 10~20 次
时间: ~5-10ms (可行)

8. 科研 Tricks —— 扩散模型 + 多智能体的工程智慧

1

扩散模型做策略 — 天然多模态探索

核心洞见：传统高斯策略只有一个峰值——无法同时表达"左绕"和"右绕"。扩散模型通过随机噪声采样 + 条件去噪自然地产生多模态分布。

可迁移场景：任何具有多个等优解的决策问题——机器人导航、多目标跟踪、博弈对抗。

2

GAT 图注意力 — 邻居信息的自动优先级

问题：集群中并非所有邻居都同等重要——最近的邻居、挡路的邻居更重要。

方案：GAT 自动学习 αij 注意力权重——不需要人工设计哪些邻居重要。

GAT 还解决了输入维度可变的问题：邻居数量变化 → GAT 输出固定维度。

3

两阶段训练 — BC 预训练 + RL 微调

扩散模型从零开始用 RL 训练几乎不可能：高方差梯度 + T 步去噪 + 多智能体非平稳性。

方案：先用传统控制器生成数据做 BC 预训练 → 获得"不坠毁"的初始策略 → RL 在线微调提升性能。

这是深度学习领域的标准范式——预训练 + 微调。

4

DDIM 加速 — 推理速度的工程优化

100 步去噪在 30Hz 控制中不可接受（~50ms > 33ms 控制周期）。

DDIM 跳步采样：100 → 10~20 步，质量几乎不变。关键调参：步子序列的选择——前密后疏（初期去噪大步，后期精细调整）。

5

Reynolds 规则 → 奖励函数

将经典的 Boids 三规则（分离、对齐、凝聚）转化为可微的奖励函数。

RL 学习的是这些规则的动态优先级调度——而非硬编码的线性组合。在狭窄通道中，分离的权重自动提高；在开阔区域，对齐和凝聚更重要。

6

GAT Actor + GAT Critic — 信息一致性

Actor 和 Critic 都使用 GAT 编码邻居信息——确保两者对"当前环境状态"的理解一致。

如果 Actor 用的编码方式和 Critic 不同，Critic 评估的 Q 值可能与 Actor 实际面临的状态不对齐——导致训练信号误导。

7

HITL 验证 — 提升论文的工程可信度

纯仿真 RL 论文容易被质疑"实际能用吗？"。GDM-MADRL 做了 HITL（硬件在环）实验——训练好的策略部署到真实飞控硬件，在仿真环境中验证。

虽然还不是真机试飞，但比纯仿真论文的可信度高一个档次。

8

可扩展性 — 参数共享 + GAT 自适应

所有 Follower 共享同一策略网络（参数共享）+ GAT 天然处理变长邻居输入。

新增 Follower：复制策略网络 → 自动参与集群，无需重新训练。这是真正的"即插即用"可扩展性。

9. 设计哲学 —— 扩散模型 = 更好的探索，GAT = 更好的感知

传统 RL 策略的局限

  • 单峰高斯：只能表示一个"最佳动作"附近的小范围随机
  • 探索靠噪声：在动作上加高斯噪声——低效的随机游走
  • 多解冲突：面对多个等优方案，学到的"平均"可能最差
  • 固定邻居权重：平均池化或最大池化——丢失空间结构

GDM-MADRL 的突破

  • 扩散多模态：自然地覆盖多个等优解——采样多样性
  • 结构化探索：去噪过程本身就是一种条件引导的探索
  • 多解共存：扩散分布中有多个模式——采样即可得到不同方案
  • 自适应邻居：GAT 注意力——最近的邻居权重最高，远处忽略

扩散模型为什么适合集群控制？

集群控制的特性扩散模型的匹配优势
多解性（障碍物有多条绕行路径）多模态分布天然覆盖多条路径
需要多样性（无人机不能全部选同一条路）独立采样 + 不同随机种子 = 不同路径
安全性关键（碰撞 = 灾难）去噪过程逐步精炼——动作从模糊到精确，不会突变
局部感知（只知道周围邻居）GAT 条件引导——感知范围内信息充分融合
GDM-MADRL 的核心方法论 —— 扩散 + 图 = 分布式集群智能
扩散模型解决"动作多样性"（多模态探索），GAT 解决"信息选择性"（哪些邻居重要）。
两者结合产生了 1+1>2 的效果：GAT 提供精确的环境条件 → 扩散模型生成高质量且多样的动作 → 集群中每架无人机做出不同但都合理的决策 → 集群整体涌现出分布式智能行为。

GDM-MADRL 的实操流程

  1. 定义 Dec-POMDP：Leader + N Follower，每架独立观测和决策
  2. 构建 GAT 图：Follower 为节点，邻居关系为边——通信/感知范围定义邻域
  3. 设计观测：自身状态 + Leader + 邻居 + 9 光线传感器——覆盖前方 180° FOV
  4. 预训练扩散策略：用传统控制器生成 (obs, action) 数据 → BC 学习基本飞行
  5. RL 在线微调：扩散策略作为 Actor + GAT 编码作为 Critic → 在线提升
  6. DDIM 推理加速：T=100 → T_eff=10~20 → 满足实时控制需求
  7. HITL 验证：部署到真实飞控硬件 → 半物理仿真验证

10. 学习路线 & 改进方向

学习路线

  1. 理解扩散模型基础：DDPM (Ho et al. 2020) — 前向加噪 + 逆向去噪的基本框架
  2. 学习扩散策略 (Diffusion Policy)：Chi et al. (2023) — 扩散模型用于机器人控制的范式
  3. 理解 GAT：Velickovic et al. (2018) — 图注意力网络的原理和实现
  4. 理解 Dec-POMDP：去中心化多智能体决策的数学框架
  5. 实现固定翼动力学：非完整约束 + 速度-转弯耦合 → 简化模型 vs 6-DOF
  6. 搭建训练管道：BC 预训练 → RL 微调 → DDIM 加速 → HITL 验证
  7. 在仿真中重现：从 3 架 Follower 开始 → 扩展到 5/10 架 → 添加障碍物
