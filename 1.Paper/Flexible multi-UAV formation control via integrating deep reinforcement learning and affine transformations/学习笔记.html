1. 系统架构全景图

2. 两层控制框架 —— 模型-数据混合驱动的核心设计

💡 核心设计智慧：将编队问题拆解为"结构保持"和"安全导航"两个子问题
论文将编队约束提炼为两个基本原则：编队构型的灵活保持 + 安全导航。然后分别用模型驱动（仿射变换）和数据驱动（DRL）的方法来解决——这就是"混合驱动"的精髓。

决策层 — 仿射变换如何保证编队结构？

仿射变换的核心能力：旋转、缩放、剪切、平移——正好对应编队在复杂环境中需要的四种调整方式：

  • 旋转：编队整体转向以对准狭窄通道方向
  • 缩放：缩小编队间距以通过狭窄通道，通过后恢复
  • 剪切：适应不规则障碍物形状
  • 平移：整体偏移避开障碍物区域

关键数学保证：仿射变换保持共线性——如果三个 UAV 在标称构型中排成一条直线，在任何仿射变换下它们仍然共线。这就是"结构完整性"的数学基础。

执行层 — DRL 为什么放在执行层？

如果 DRL 放在决策层？

DRL 直接输出编队参数（A, b），然后每个 UAV 用传统控制器追踪目标位置。

问题：DRL 需要处理全局信息（所有 UAV 状态 + 所有障碍物），状态空间巨大，且对未知动态障碍物泛化差。

论文的做法：DRL 在执行层 ✅

决策层用数学模型保证编队结构 → 只需输出目标位置。执行层每个 UAV 的 DRL 只需：追踪目标 + 避障。

优势：每个 UAV 只关注局部信息（LiDAR + 自身状态 + 相对目标），维度低、泛化好、分布式执行。

3. 论文精读

3.1 核心贡献

  1. 模型-数据混合驱动框架：首次将仿射变换（模型驱动）与 DRL（数据驱动）系统性地结合用于多 UAV 编队控制
  2. 两层解耦架构：决策层保证编队结构完整性，执行层处理安全导航——各司其职、优势互补
  3. 端到端速度控制：DRL 直接输出速度指令 [v, ω]，无需中间 PID 层级
  4. 将相邻 UAV 视为动态障碍物：训练时只设静态障碍物，相邻 UAV 自然充当动态障碍物——巧妙的训练策略
  5. 两阶段课程学习：先在无障碍环境学基本导航，再在复杂障碍环境学避障——大幅加速收敛

3.2 UAV 运动学模型

本文使用独轮车模型 (Unicycle Model)，相比 OpenFlight 的 6-DOF 模型简化很多：

Eq.(1)
ẋ = v · cos ψ,   ẏ = v · sin ψ,   ψ̇ = ω
0 ≤ v ≤ v_max,   |ω| ≤ ω_max   (Eq.2)
这是本文和 OpenFlight 的重要区别：本文关注的是编队层面的协调控制，使用了简化运动学；OpenFlight 关注单机极限机动，使用了高保真 6-DOF

3.3 感知模型

每个 UAV 配备 270° FOV 机载 LiDAR，3 层扫描线，用于实时检测周围障碍物（静态 + 动态）和其他 UAV。这是 DRL 策略的主要输入来源。

4. 算法公式全解 —— 从 POMDP 到仿射变换到 PPO

论文的数学体系分为四个层次：问题建模（POMDP）→ 决策层（仿射变换+应力矩阵）→ 执行层（DRL）→ 训练优化（PPO）。

4.1 POMDP 形式化

POMDP = ⟨ S, A, T, R, Ω, O, γ ⟩
每个 UAV 面临的是一个部分可观测的序列决策问题——只能通过 LiDAR 感知局部环境
符号含义本文中的具体对应
S真实状态所有 UAV 位置 + 障碍物位置 + 目标编队构型
A动作空间2 维连续 [v, ω]，v∈[0,v_max], ω∈[-ω_max,ω_max]
T状态转移独轮车运动学 Eq.(14-15)
R奖励函数r = r_target + r_collision + r_smooth1 + r_smooth2
Ω观测空间LiDAR 扫描 (270°×3) + 速度 + 相对目标位置
O观测函数LiDAR 感知 + 状态读取（不完全观测——看不到全局）

4.2 决策层公式 —— 仿射变换与应力矩阵

Eq.(3): 应力矩阵基本关系

Eq.(3)
(Ω ⊗ I_d) · x = 0
应力矩阵 Ω 定义了编队成员之间的"平衡关系"——这是编队构型的数学约束

Eq.(4): 应力矩阵定义

Eq.(4)
[Ω]_{ij} = ω_{ij} (if i≠j and (i,j)∈E),   0 (if i≠j and (i,j)∉E),   −Σ ω_{ik} (if i=j)
ω_{ij} > 0 = 吸引力, ω_{ij} < 0 = 排斥力, ω_{ij} = 0 = 无直接关系

Eq.(5): 时变目标编队（核心公式）

Eq.(5)
x'(t) = (A(t) ⊗ I_d) · x + [1_N ⊗ I_d] · b(t)
A(t) ∈ ℝ^(d×d)仿射变换矩阵：编码旋转、缩放、剪切
b(t) ∈ ℝ^d平移向量
x ∈ ℝ^(dN)标称编队构型（预设形状）
x'(t)当前时刻的目标编队位置（所有 UAV）
⊗Kronecker 积——将 d×d 变换广播到每个 UAV

Eq.(6-7): Leader-Follower 应力矩阵分块

Eq.(6-7)
Ω · x = 0 ⇒ Ω_l·x_l + Ω_f·x_f = 0
Ω = [Ω_{ll} Ω_{lf}; Ω_{fl} Ω_{ff}]
前 n_l 行为 leader 约束，后 n_f 行为 follower 约束

Eq.(8): 跟随者目标位置计算公式（最实用的公式）

Eq.(8)
p_f*(t) = −Ω_{ff}⁻¹ · Ω_{fl} · p_l(t)
只要知道 leaders 的位置和应力矩阵，每个 follower 就能独立计算自己的目标位置——无需中心节点！这是分布式实现的关键。

4.3 执行层公式 —— DRL 策略优化

Eq.(9): 随机策略

Eq.(9)
a_i ~ π_θ(·|o_i)
每个 UAV 的动作为从以 o_i 为条件的高斯分布中采样——训练时引入探索，测试时用确定性均值

Eq.(10-15): 优化问题

Eq.(10-15)
min E(T_i | π_θ)
s.t. ‖p_i − p_j‖ ≥ r_safe  （UAV 间避碰, Eq.11）
‖p_i − o_k‖ ≥ r_safe  （障碍物避碰, Eq.12）
p_i* = 目标位置（满足编队约束, Eq.13）
p_i^{t+1} = p_i^t + Δt·v_i[cos ψ_i, sin ψ_i]ᵀ  （运动学, Eq.14-15）

4.4 奖励函数 —— Eq.(16-20)

Eq.(16)
r = r_target + r_collision + r_smooth1 + r_smooth2
奖励项公式设计意图
r_target (Eq.17)到达目标: +r_arrive (15)
过程奖励: λ₁·(|Δp_{t-1}| − |Δp_t|)		鼓励 UAV 向目标点移动
r_collision (Eq.18)碰撞: −15
无碰撞: 0		硬惩罚——碰撞 = 任务失败
r_smooth1 (Eq.19)−λ₂·|ω_i| if |ω_i| > 0.7
0 otherwise		抑制过大角速度，防止 zigzag
r_smooth2 (Eq.20)heading 偏差惩罚 / 距离权重防止过度适应避障策略（避免 zigzag）
奖励设计亮点：heading 惩罚 / 距离权重 (Eq.20)
这是一个自适应惩罚——当 UAV 离目标远时，heading 偏差的惩罚较小（允许绕路避障）；当 UAV 接近目标时，惩罚增大（强制对准目标）。这种设计自然地在"避障灵活性"和"目标导向"之间取得了平衡。

4.5 PPO 训练算法

论文使用 PPO（Proximal Policy Optimization），关键超参数见 Table 1：γ=0.99, λ=0.95, K_epoch=20, lr=5e-5(S1)/2e-5(S2), KL_target=15e-4。

特别的是使用了 自适应 KL 惩罚系数：如果 KL 散度太大就增大惩罚系数 β，太小就减小——这是 PPO 原始论文中 KL-penalty 变体的做法。

6. RL 训练详解

6.1 训练超参数（Table 1）

超参数值说明
γ (折扣因子)0.9999% 未来奖励计入当前决策
λ (GAE)0.95Bias-variance 平衡
T_horizon8000每次收集 8000 步经验才更新
K_epoch20同一批数据重复用 20 次
lr (学习率)5e-5 (S1) / 2e-5 (S2)两阶段递减——越到后面越精细
KL_target15e-4自适应 KL 的目标值
KL 惩罚系数 β自适应KL 太大 → β×2; KL 太小 → β÷2
ε (clip range)1.0 (初始)使用 KL-penalty，非 clip 变体
reward scaler_arrive=15, r_collision=−15到达和碰撞对称
safe radiusr_safe = 0.5m碰撞判定距离
注意：PPO 的两种变体
OpenFlight 使用 PPO clip 变体（ε=0.2, 限制概率比），本文使用 PPO KL-penalty 变体（自适应 β, KL_target=1.5e-3）。两种变体都是 PPO 原论文提出的，但 clip 变体更常见（实现更简单）。本文使用 KL-penalty 可能与多智能体训练的不稳定性有关——自适应 KL 比固定 clip 更灵活。

6.2 两阶段课程学习

Stage 1: 无障碍环境训练 (~70k iter)：多个 UAV 在无障碍场景中学习基本导航能力 + UAV 间避碰。保存训练好的网络参数。

Stage 2: 复杂障碍环境训练：用 Stage 1 的参数初始化网络，在复杂障碍场景中继续训练。UAV 逐步学会在避障的同时保持编队。

为什么课程学习有效？直接从复杂场景开始训练，奖励信号稀疏且噪声大——策略在学会基本导航前就"迷失"了。Stage 1 提供了一个好的初始化，让 Stage 2 的探索更有效率。这与 OpenFlight 的"从 V1 开始验证框架"思路一致。

6.3 训练环境设置

  • 模拟器：STAGE（常用的 2D 机器人模拟器）
  • 障碍物：随机生成的灰色圆柱体（静态），训练时 UAV 间互为动态障碍物
  • 每轮初始化：随机 UAV 起始位置 + 随机目标位置
  • 硬件：Ubuntu 20.04, 32GB RAM, GeForce RTX 3090
  • 训练时长：两阶段共约 24 小时

6.4 训练结果（Fig.8）

Stage 1 的奖励在 ~70,000 迭代后稳定在 ~90——说明 UAV 学会了基本导航和 UAV 间避碰。Stage 2 开始时奖励明显高于 Stage 1 的起点（因为用了 Stage 1 的预训练参数），最终快速收敛到 ~115——说明 UAV 学会了在复杂障碍环境中的安全导航。

📊 Fig.6 — Network structure

  • 为什么拼接后再 FC？将感知信息（LiDAR）与自身状态（速度、目标）融合做决策

这比 OpenFlight 的纯 MLP 更复杂——因为需要处理高维 LiDAR 数据，而 OpenFlight 的观测已经是精心设计的 6D 低维特征。

8. 科研 Tricks —— 哪些可以直接用到你的项目里

1

混合驱动：模型 + 数据各司其职

核心思想：不要把全部问题交给 DRL。用数学模型处理有明确结构约束的子问题（编队形状保持），用 DRL 处理难以建模的子问题（避障、未知环境适应）。

模型驱动的部分让 DRL 的输入更干净（只需要追踪目标点），学习更高效。

2

将邻居视为动态障碍物 —— 巧妙的训练策略

训练场景中只放静态障碍物，但其他 UAV 自然充当了动态障碍物。这避免了需要专门生成动态障碍物数据——UAV 间的相对运动自然提供了"动态"训练信号。

这招对多智能体 RL 训练特别实用：不需要专门设计动态障碍物行为模型。

3

两阶段课程学习 —— 从简单到复杂

# Stage 1: 无障碍 —— 学会基本导航
train(nav + inter-UAV collision avoidance,
      env=obstacle_free)

# Stage 2: 复杂障碍 —— 在好的基础上精进
train(complex obstacle avoidance,
      env=obstacle_rich,
      init_weights=stage1_checkpoint)

比直接从复杂场景开始训练快得多、稳得多。

4

分布式计算的数学优雅性 (Eq.8)

p_f*(t) = −Ω_ff⁻¹·Ω_fl·p_l(t) —— 每个 follower 只需要知道 leaders 的位置和本地的应力矩阵块，就能独立计算自己的目标。不需要全局通信、不需要中心节点。

这是"分布式算法"的理想形态：数学保证的分布式可计算性。

5

自适应 heading 惩罚 (Eq.20)

heading 惩罚除以距离权重：离目标远 → 惩罚小（允许绕路）；离目标近 → 惩罚大（必须对准）。

这种状态依赖的自适应奖励比固定权重更灵活——RL 策略在远距离时更自由地探索避障路径，在近距离时更精确地对准目标。

6

PPO KL-penalty 变体 —— 多智能体训练的稳定性

本文选择 PPO 的 KL-penalty 变体（而非更常见的 clip 变体）。自适应 KL 惩罚系数在多智能体场景中可能更稳定——当策略变化过大时自动加强约束。

选择算法变体要匹配任务特征：单智能体 → clip；多智能体 → KL-penalty 可能更好。

7

编队评价指标设计

不只是"碰撞次数"——设计了编队位置误差 + 编队相似性误差两个核心指标，区分了"绝对位置精度"和"形状保持能力"。

对于编队控制论文，必须同时评估安全性（碰撞）、编队质量（误差）和运动质量（Jerk/时间）。

8

对比方法的公平 setup

每个对比方法都经过参数调优，确保在各自的实验场景中能实现无碰撞飞行。ORCA 方法还设置了 Fixed 和 Varying 两种目标模式。

做比较实验的铁律：让 baseline 也充分发挥，否则赢了也没有说服力。

9. 设计哲学 —— "混合驱动" 的方法论

❌ 常见误区

  • 把所有问题都交给 DRL 端到端解决
  • 认为"纯学习"比"模型+学习"更先进
  • 忽略问题的结构特性
  • 训练时不分阶段，直接从最难的场景开始
  • 多智能体训练不考虑分布式约束

✅ 本文的做法

  • 数学模型处理结构约束（仿射变换 → 编队形状）
  • DRL 处理难以建模的部分（避障、未知环境）
  • 两层解耦：决策层全局协调 + 执行层局部自主
  • 两阶段课程学习：简单 → 复杂
  • 分布式可计算：每个 follower 独立算目标
🎯 本文的核心方法论
不要试图用一个黑箱神经网络解决所有问题。先把问题拆解——哪些有明确的数学结构？哪些必须依靠数据驱动？
模型驱动的部分让系统可解释、可保证；数据驱动的部分让系统自适应、灵活。两者结合才是工程上最优的方案。

作为多智能体 RL 新手的实操流程

  1. 分析问题结构：哪些约束有数学模型？哪些需要学习？
  2. 设计两层架构：决策层（全局协调）+ 执行层（局部自主）
  3. 选数学模型：仿射变换、应力矩阵、一致性协议……选最适合的
  4. 设计 DRL 部分：观测（传感器+模型输出）、动作、奖励
  5. 两阶段训练：先简单场景学会基本功，再复杂场景精进
  6. 多指标评估：安全（碰撞）+ 质量（误差）+ 效率（时间/Jerk）
  7. 充分的 baseline 对比：纯模型、纯学习、混合——三种都要比

10. 学习路线 & 改进方向

学习路线

  1. 理解仿射变换 → 搞懂 Eq.(5) 的数学含义：旋转、缩放、剪切、平移如何用矩阵表示
  2. 理解应力矩阵 → 搞懂 Eq.(3-4, 6-8)：为什么 follower 可以独立计算目标位置
  3. 理解 DRL 执行层 → CNN 处理 LiDAR + FC 融合状态 → PPO 训练（这是和 OpenFlight 最相似的部分）
  4. 复现实验 → 如果有开源代码，先跑通 Stage 1 的训练
  5. 修改观测 → 在观测中增删维度，观察对编队质量的影响
  6. 扩展到新场景 → 换一种编队形状、换一种障碍物分布
