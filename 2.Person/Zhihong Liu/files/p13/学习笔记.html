融合深度强化学习与仿射变换的灵活多无人机编队控制

Flexible Multi-UAV Formation Control via Integrating Deep Reinforcement Learning and Affine Transformations

Multi-UAV Affine Transform Deep RL Formation Control NUDT Chao Yan 合作
作者Yunhao Liu, Zhihong Liu, Guanzheng Wang, Chao Yan, Xiangke Wang, Zhiping Huang
期刊Aerospace Science and Technology, Vol.157, 109812, 2025
DOI10.1016/j.ast.2024.109812
领域多无人机系统 — 深度强化学习 — 编队控制  |  IF~5.5

一、论文概述

本文提出了一种将深度强化学习 (DRL) 与仿射变换 (Affine Transformations) 深度融合的多无人机编队控制方法。该方法利用仿射变换作为编队机动的数学描述框架（平移、旋转、缩放、剪切），由 DRL 学习最优控制策略，实现了比传统刚性编队更灵活的队形变换能力。

与 Chao Yan 的直接合作
本文是 Zhihong Liu 与 Chao Yan（闫超） 直接合作的论文，两人均为国防科技大学 (NUDT) 学者。Chao Yan 的研究聚焦于视觉运动策略与多智能体协同控制，Zhihong Liu 在 DRL 加速训练方面有深厚积累（ACM Computing Surveys 综述一作），两人的技术互补构成了本文的学术基础。

核心研究问题

传统多 UAV 编队控制多采用刚性几何约束（如 leader-follower、虚拟结构法），一旦队形需要变换（如避障时缩放、转向时旋转），往往需要重新规划或切换控制器，缺乏灵活性。本文的核心问题是：能否利用仿射变换统一描述编队的各种机动，并让 DRL 学习在此框架下输出平滑、鲁棒的控制指令？

二、研究背景与相关工作

2.1 多UAV编队控制的主流方法

方法核心思想优势局限
Leader-Follower一架领航机引导，其他跟随结构简单、工程成熟单点故障、灵活性差
Virtual Structure编队视为刚体，各机刚性锚定队形保持精度高无法变形、避障困难
Behavior-Based规则组合驱动行为分布式、可扩展理论分析困难、无法保证收敛
Consensus-Based基于图论的多智能体一致协议理论完善、可证明收敛队形变换需重新定义拓扑
DRL-Based数据驱动学习控制策略灵活适应、端到端训练开销大、可解释性差

2.2 仿射变换的引入动机

仿射变换 T(x) = Ax + b 是计算机视觉和计算机图形学中的标准工具（图像配准、形状匹配），但在多无人机编队中的应用尚少见。其关键优势在于：

  • 统一框架：平移、旋转、缩放、剪切——四种基本机动被一个 2x2 矩阵 A 和位移向量 b 统一刻画
  • 平滑变换：通过连续改变 A 和 b 即可实现编队的平滑形变，无需切换控制器
  • DRL 友好：A 和 b 的参数空间维度低 (2D: 6 个自由度)，适合作为 DRL 动作空间的输出

2.3 DRL 在编队控制中的角色

DRL 的优势在于处理高维、非线性、模型不确定的控制问题。在本文中，DRL 策略网络以编队状态（各 UAV 位置、速度）和环境感知（障碍物、目标点）作为输入，直接输出仿射变换参数 {A, b}，然后通过仿射映射计算各 UAV 的目标位置，最终由底层飞行控制器执行。

三、方法论：DRL + 仿射变换融合架构

3.1 系统架构

整体架构分为三层：

  1. 感知层 (Perception)：多 UAV 状态汇聚 + 环境感知（通过板载传感器或仿真环境获取）
  2. 策略层 (Policy)：DRL 策略网络输出仿射变换参数 {A, b}
  3. 执行层 (Execution)：仿射映射生成各 UAV 目标位姿 → 底层姿态/位置控制器执行
图2: 系统架构流程
    ┌─────────────────────────────────────────────────────────────┐
    │                    DRL Policy Network                        │
    │  Input:  [formation_state, env_obs, goal]                   │
    │                   ↓                                          │
    │  Output: {A (2×2 matrix), b (2×1 vector)}                  │
    └─────────────────────────┬───────────────────────────────────┘
                              ↓
    ┌─────────────────────────────────────────────────────────────┐
    │                Affine Transformation Layer                   │
    │                                                             │
    │   p_i^target = A · p_i^ref + b    (for each UAV i)          │
    │                                                             │
    │   where p_i^ref = 第 i 架 UAV 在参考编队中的坐标             │
    │         p_i^target = 第 i 架 UAV 的目标位置                  │
    └─────────────────────────┬───────────────────────────────────┘
                              ↓
    ┌─────────────────────────────────────────────────────────────┐
    │                   Low-Level Flight Controller                │
    │   各 UAV 独立执行位置/姿态控制 → 到达目标位置                  │
    └─────────────────────────────────────────────────────────────┘

3.2 状态空间与动作空间设计

空间内容维度
状态空间 (State)编队中心位置、各UAV相对位置偏差、速度、障碍物距离向量、目标方向取决于 UAV 数量和感知信息量
动作空间 (Action)仿射矩阵 A 的 4 个元素 + 平移向量 b 的 2 个分量6 (连续动作空间)

3.3 奖励函数设计

多目标奖励函数指导 DRL 学习期望的编队行为：

奖励函数结构

R = w₁·R_formation + w₂·R_goal + w₃·R_obstacle + w₄·R_smooth

R_formation编队形状保持奖励（实际队形与目标队形的相似度）
R_goal向目标点靠近的奖励
R_obstacle避障惩罚（与障碍物距离越近惩罚越大）
R_smooth动作平滑性惩罚（抑制控制指令突变）

3.4 训练策略

  • 算法选择：基于 PPO (Proximal Policy Optimization) 或 SAC (Soft Actor-Critic)，适合连续动作空间
  • 集中式训练、分布式执行 (CTDE)：训练时使用全局信息（集中式 critic），执行时各 UAV 仅使用本地观测（分布式 actor）
  • 课程学习 (Curriculum Learning)：从简单场景（无障碍、固定目标）逐步增加复杂度（动态障碍、移动目标）

四、仿射变换编队框架详解

4.1 数学形式

对于由 n 架 UAV 组成的编队，设参考编队中各 UAV 的参考坐标为 p_i^ref = (x_i^ref, y_i^ref)，则目标坐标由仿射变换给出：

p_i^target = A · p_i^ref + b

where A = [[a₁₁, a₁₂], [a₂₁, a₂₂]], b = [b₁, b₂]

4.2 四种基本仿射变换与编队机动对应关系

变换类型A 矩阵形式编队机动含义
平移A = I (单位阵)编队整体平移到新位置
均匀缩放A = s·I, s>0编队整体放大/缩小（如过狭窄通道时收缩）
旋转A = [[cosθ, -sinθ], [sinθ, cosθ]]编队整体旋转（如转向新航向）
剪切A = [[1, k], [0, 1]]非均匀变形（较少使用，但可用于特殊机动）
一般仿射A 为一般 2×2 可逆矩阵上述变换的任意组合

4.3 编队灵活性体现

与刚性编队方法相比，仿射变换框架的核心优势在于连续参数空间：

  • DRL 输出的是 6 个连续值（a₁₁, a₁₂, a₂₁, a₂₂, b₁, b₂），而非离散的编队模式选择
  • 队形可以在任意两种形状之间平滑过渡，中间不存在控制律切换
  • 编队可以同时进行缩放+旋转+平移（如绕过障碍物转角时自然缩小并旋转）

五、应用场景与学术意义

5.1 典型应用场景

场景编队需求仿射变换的具体运用
协同侦察/监视大范围覆盖 → 重点区域聚焦缩放 + 平移：广域扫描时编队拉大，发现目标后收缩聚焦
空中加油精确编队保持 + 对接高精度平移跟踪 + 微小旋转校正
协同攻击/饱和突防密集编队 → 分散突防 → 重新聚合缩放 + 剪切：收缩穿越防空区，展开分配目标
城市峡谷飞行根据街道宽窄动态调整各向异性缩放：横向收缩（适应街道宽度），纵向维持或拉长
无人机灯光秀复杂图案平滑变换一般仿射：每个图案即为一种 A 的配置

5.2 学术贡献

  • 框架创新：首次将仿射变换作为 DRL 与编队控制之间的结构化接口，将无结构的端到端 DRL 转化为有数学约束的优化问题
  • 灵活性与安全的平衡：仿射变换保证了编队的基本几何约束（直线保持为直线、平行保持为平行），避免 DRL 输出"奇形怪状"的不安全编队
  • 跨合作网络：体现了 NUDT 内部 Zhihong Liu (DRL 加速) 与 Chao Yan (多智能体/UAV) 两个方向的融合

5.3 局限与改进方向

  • 仿射变换限于线性变换，无法表达弯曲或扭曲的编队形状
  • 2D 仿射变换可直接推广至 3D (Affine(3) = 12 个自由度)，但维度增加导致 DRL 训练难度急剧上升
  • 真实环境中的通信延迟、GPS 拒止等问题在仿真中难以充分建模
与 Chao Yan 研究方向的互补性
Chao Yan 在视觉运动策略与多智能体协同方面的工作（如视觉 servoing、多机协同感知）可以与本文的编队框架结合——例如，利用视觉信息直接估计目标编队的仿射变换参数，实现"看到即编队"的视觉引导编队控制。这是一个值得关注的交叉方向。
