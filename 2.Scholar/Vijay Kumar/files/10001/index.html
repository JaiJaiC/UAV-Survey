2. 核心洞察与创新点

2.1 核心洞察

核心洞察
为什么传统机器人安全方法无法处理开放世界的上下文安全需求？
传统安全方法（CBF、HJ Reachability、时序逻辑）需要专家预先指定所有安全约束——但在开放世界中，安全是上下文相关的：同一条走廊在拥挤和空旷时要求不同的导航策略。机器人无法在部署前预见到每一种可能的上下文，因此必须从视觉观测中自主推理在运行时什么行为是安全的。

本文的核心洞察是什么？
将 VLM 用作"上下文安全推理引擎"。VLM 不仅检测物体（如"小心地滑"标志牌），还理解其语义含义（"小心地滑标志牌意味着周边区域可能有危险"），并将这种理解转化为可执行的空间安全约束。关键在于三点：(1) 将 VLM 的推理结构化为安全谓词，如 ON(floor)、AROUND(wet_floor_sign)、BETWEEN(cones)；(2) 将这些谓词接地到物理空间中的像素级安全集合；(3) 通过 CBF 强制执行安全约束，并在 VLM 感知不确定性下提供概率保证。

2.2 灵感映射

灵感来源启发了什么洞察类型
VLM 的开放词汇理解能力（Gemma 3, SAM3）VLM 能够理解物体在上下文中的"安全含义"——"小心地滑"标志不仅是一个标志，而是周围区域存在危险的信号。VLM 成为一个在线安全推理模块，而非仅仅是感知模块。架构
Control Barrier Function (CBF) 前向不变性理论CBF 仅需要动力学模型和障碍函数即可保证安全，与规划器/控制器解耦。传统 CBF 使用固定的障碍函数——CORE 使其从感知中在线构建，并具有概率安全保证。方法
语义推理与接地的分离将"推理"（VLM）与"接地"（分割 + 点云投影 + 网格地图）解耦。VLM 输出语义约束；接地模块将其转换为空间安全集合。这种分离使 VLM 专注于推理，而接地模块专注于空间精度。策略
检测概率函数 m(delta x) 与 Ville 不等式借鉴随机安全分析的方法，通过检测概率函数建模 VLM 感知不确定性，再利用非负上鞅推导概率轨迹长度安全保证——以概率意义上确保机器人不进入未检测到的危险区域。数学

2.3 创新点分解

创新点 1：VLM 驱动的上下文安全推理 —— 机器人无需先验知识即可自主推理安全规则

【解决的问题】传统安全方法需要专家为每种场景手动指定安全约束，但开放世界场景无限多变。现有的语言驱动安全方法支持用自然语言描述安全规则，但仍需要用户提供——机器人无法自主推理。
【灵感来源】VLM 擅长开放词汇理解。Gemma 3 27B 足够强大，能够理解图像语义。已有工作仅将 VLM 用于可通行性估计或轨迹选择——从未用于自主安全约束推理。
【具体设计】结构化的安全谓词系统：ON（可通行表面）、NEAR（碰撞风险物体）、AROUND（危险区域周边）、BETWEEN（物体排列形成的屏障）。VLM 输出 JSON 格式的安全推理结果（safety_logic 理由 + 语义类别 + 安全/不安全区域谓词）。Chain-of-Thought 推理和安全逻辑指令将 VLM 可靠性从 61% 提升至 91%。

创新点 2：语义到空间的接地 —— 将自然语言安全约束转化为 CBF 障碍函数

【解决的问题】VLM 输出是自然语言安全约束（如"AROUND(wet_floor_sign)"），不能直接控制机器人。需要一种方法将这些语义约束转化为可执行的空间安全集合。
【具体设计】(1) 开放词汇分割（SAM3）定位 VLM 识别的语义类别；(2) 按谓词类型构建像素安全集合——ON/NEAR 选取类别像素，AROUND 施加形态学膨胀，BETWEEN 计算凸包；(3) 深度投影到 3D 点云，累积到 2D 网格地图，每格记录 n_safe/n_unsafe 计数；(4) 估计 P[x is safe] = n_safe/(n_safe+n_unsafe) 并通过阈值获得安全集合 S；(5) 通过符号距离场构建 CBF 障碍函数 h(x)。

创新点 3：感知不确定性下的概率安全保证

【解决的问题】VLM 感知并非完美——可能遗漏危险区域。当 VLM 可能产生检测错误时，如何以概率意义上提供安全保证？
【具体设计】定义检测概率函数 m(delta x)，对从相对位置 delta x 检测到不安全区域的 Bernoulli 概率进行建模。定义逆正则化距离 d^(-1)(r; c, l) = c/(r+l)。通过非负上鞅和 Ville 不等式，推导出概率轨迹长度安全保证：如果系统在每个时间步收集 k*(t) 次观测，满足 E[d^(-1)(r)] 的界，则 P[存在 t: x_t 不在 S 中] 至多为 delta——轨迹上任意时刻离开安全集合的概率被 delta 界定。

4. 任务定义 —— 解决什么问题？

4.1 问题形式化

核心问题 —— 上下文安全（Problem 1） $$ \text{Given: } \dot{x} = f(x) + g(x)u,\quad G,\quad \{I_t\} $$ $$ \text{Find } u_{\text{safe}} \text{ s.t. } (i)\; \{s_1,...,s_K\} \text{ inferred from } I_t, G;\; (ii)\; s_k \rightarrow S_k \subset \mathbb{R}^n;\; (iii)\; P\left[x \in \bigcap_k S_k\right] \ge 1-\delta $$
$x \in \mathbb{R}^n$ = robot state (2D position + heading) | $u$ = control input (linear velocity + yaw rate) | $G$ = high-level safety guide (e.g., "follow pedestrian social norms") | $\{I_t\}$ = streaming RGB-D observations | $s_k$ = contextual safety constraints (predicate form) | $S_k$ = grounded spatial safe sets | $1-\delta$ = probabilistic safety guarantee level | 核心挑战：机器人对环境没有任何先验知识（没有地图、没有预定义的安全规则），必须仅从运行时的视觉观测中推理什么是安全的。

4.2 输入 / 输出

类别内容维度
输入RGB-D 相机流 + 里程计 + 高层安全指南 G图像帧序列、深度图、机器人位姿
输出安全控制输入 u_safe，最小化与名义控制的偏差线速度、偏航角速度（3-DOF 控制）
目标在未知环境中实现上下文安全导航：理解场景语义、推理安全约束、执行安全控制仿真成功率 95%（不安全任务 93.3%）

4.3 任务类型

这是一个感知-推理-控制融合的系统级安全框架，核心任务类型为上下文安全导航。具体场景包括：(1) 上下文缓冲区——识别隐式危险物体（如仓库中的叉车）并在其周围保持安全距离；(2) 上下文屏障——识别由物体排列形成的禁行区域（如交通锥排列成一条线），将其理解为禁行区而非独立障碍物；(3) 上下文自适应通行——区分可通行与不可通行表面（人行道 vs. 草地），尽管两者物理上均可通行。平台：移动机器人（仿真中为差速驱动，真实世界为 Boston Dynamics Spot）；场景：仓库、医院、户外住宅区。

5. 核心挑战 —— 为什么难？

5.1 本质（第一性原理）

第一性原理诊断
核心矛盾：安全是上下文相关的，但传统安全方法是上下文无关的。

(1) "上下文鸿沟"：传统障碍物检测只能识别"物理障碍"——但一排交通锥并非障碍物，它们排列成的"线"才代表禁行区域。"小心地滑"标志不是障碍物，它暗示周边区域可能湿滑。理解这种语义含义需要传统流水线所缺乏的世界知识。

(2) "推理-接地鸿沟"：即使 VLM 理解了"'小心地滑'标志意味着周边危险"，将其转化为可执行的空间约束（如"距离标志至少 1 米"）是一个语言到空间的映射问题。语言理解给出抽象结果，空间约束必须精确——桥接两者需要精心设计的接地机制。

(3) "不确定性鸿沟"：VLM 可能出错——遗漏危险区域或误分类物体。如果在 VLM 漏检的瞬间机器人进入真正危险的区域，安全就受到威胁。传统 CBF 理论假设障碍函数是确定性的，但 CORE 的障碍函数是从噪声感知中在线构建的——需要在不确定性下提供安全保证。

5.2 前人方法及其局限

前人方法核心思想失败原因
传统 CBF / HJ Reachability 使用已知障碍函数强制执行前向不变性，确保系统永远不会离开安全集合 需要预先已知且固定的障碍函数。在开放世界中，安全约束随上下文变化——传统方法无法在线推理或修改障碍函数。
语言条件安全 用户用自然语言描述安全规则（如"远离厨房"），系统将语言指令接地到代价地图或安全集合 仍然需要用户预先提供环境特定的自然语言安全规则。不适用于未知环境——机器人无法在无用户输入的情况下自主推理安全。
LLM 规划 + 验证 使用 LLM 生成任务规划，然后形式化验证规划安全性 验证检查的是离散动作序列，而非连续状态的动态安全约束。无法处理感知层面的安全——如果危险未被预编码到世界模型中，验证对其视而不见。
纯几何安全（避障） 基于深度/LiDAR 的障碍物检测与避让 无法区分"需要避开的障碍物"和"不需要避开的物体"——花盆和"小心地滑"标志在几何流水线中看起来完全一样。完全缺乏语义理解，无法执行上下文敏感的安全行为。

5.3 关键挑战

  • 挑战 1 —— 在线上下文推理：VLM 必须在数秒内从单帧图像中识别所有安全相关的语义（CORE 中约 4.1 秒），并正确应用空间算子（何时用 AROUND、BETWEEN、NEAR）。VLM 选择至关重要——Gemma 3 27B 达到 91.0%/85.0% 的安全/不安全谓词准确率，而 Llava 34B 仅为 49.0%/56.0%。
  • 挑战 2 —— 鲁棒语义接地：VLM 输出必须精确映射到物理空间。分割噪声产生"幽灵障碍物"，错误地阻塞安全路径。深度投影误差将安全/不安全分类错误分配到网格单元。接地误差是真实世界实验中的主要失败模式。
  • 挑战 3 —— 感知不确定性下的安全执行：构建基于网格计数的概率安全集合，通过阈值获得 S，构建 SDF 障碍函数，并在 VLM 出错时提供概率保证。将理论（定理 1）与经验安全性能（93.3%）匹配，需要仔细标定检测概率函数 m(delta x)。

6. 关键方法与公式

6.1 核心算法

CORE 是一个三阶段感知-推理-控制系统。阶段 1（上下文安全推理）使用 VLM 从图像中推理安全谓词；阶段 2（语义接地）将谓词转化为基于网格计数的概率安全集合和符号距离场障碍函数；阶段 3（CBF 安全执行）通过 QP 优化合成最小干预安全控制，并提供理论概率安全保证。

6.2 关键公式

Group A: 图像空间安全集合构建

Eq.(2) -- Image-Space Safe Set $$ I_{\text{safe}} = \left( \bigcup_j I_j^{\text{safe}} \right) \setminus \left( \bigcup_k I_k^{\text{unsafe}} \right) $$
$I_j^{\text{safe}}$ = 第 j 个安全谓词对应的像素集合（如 ON(floor) 的所有地面像素） | $I_k^{\text{unsafe}}$ = 第 k 个不安全谓词的像素集合（如 AROUND(wet_floor_sign) 的膨胀像素） | 逻辑含义：安全区域 = 所有安全区域的并集 减去 所有不安全区域的并集。不安全优先于安全——如果一个像素同时属于安全和不安全区域，最终归类为不安全。

Group B: 概率安全估计与障碍函数构建

Eq.(3)(4) -- Probabilistic Safety and SDF Barrier Function $$ P[x \text{ is safe}] = \frac{n_{ij}^{\text{safe}}}{n_{ij}^{\text{safe}} + n_{ij}^{\text{unsafe}}} $$ $$ h(x) = \begin{cases} \min_{y \in \partial S} \|x - y\| & \text{if } x \in S \text{ (safe)} \\ -\min_{y \in \partial S} \|x - y\| & \text{if } x \notin S \text{ (unsafe)} \end{cases} $$
$n_{ij}^{\text{safe}}, n_{ij}^{\text{unsafe}}$ = 网格单元 (i,j) 中安全/不安全点的计数（从深度投影累积） | $S = \{x \mid P[x \text{ is safe}] \ge \tau\}$ = 阈值化安全集合（tau = 0.5） | $h(x) \ge 0$ = 机器人在安全区域内；$h(x) < 0$ = 机器人已离开安全区域 | SDF 梯度：$\nabla \hat{h}(x) = \frac{x - y^*(x)}{\|x - y^*(x)\|}$，其中 $y^*(x)$ 是安全集合边界上的最近点——可通过网格查找高效获得。

Group C: CBF 安全控制合成

Eq.(5)(6)(7) -- CBF Forward Invariance and Safe Control $$ x(0) \in S \implies x(t) \in S \quad \forall t \ge 0 \quad \text{(Forward Invariance)} $$ $$ \langle \nabla h(x), f(x) + g(x)u \rangle \ge -\alpha(h(x)) \quad \text{(CBF Condition)} $$ $$ u_{\text{safe}} = \arg\min_u \|u - u_{\text{nom}}\|_2^2 \quad \text{s.t. CBF Condition} $$
$f(x), g(x)$ = 控制仿射动力学 | $\alpha(\cdot)$ = 扩展 class-K 函数（CORE 使用 alpha(x)=0.25x） | $u_{\text{nom}}$ = 名义控制器输出（航点跟踪 + PID） | CBF 条件含义：障碍函数 h 沿系统轨迹的导数必须至少为 -alpha(h(x))。当 h(x) 趋近于 0（机器人接近安全边界）时，此条件强制控制增大 h（远离边界），保持前向不变性。

Group D: 感知不确定性下的概率保证

Eq.(9)(10)(11) -- Detection Probability and Safety Guarantee $$ d^{-1}(r; c, \ell) = \frac{c}{r + \ell} \quad \text{(Inverse Regularized Distance)} $$ $$ \mathbb{E}[d^{-1}(r)]_{\{k\}} \le \int_0^\infty \frac{c(1 - m(r))^k}{(r + \ell)Z} dr \quad \text{(Expected Upper Bound After k Observations)} $$ $$ \text{Theorem 1: If } \mathbb{E}[d^{-1}(r)]_{\{k^*\}} \le \frac{c(\delta - \gamma)}{\ell} \text{, then } P[\exists t: x_t \notin S] \le \delta $$
$m(\delta x)$ = Bernoulli 检测概率函数（从相对位置检测到不安全区域的概率） | $r$ = 到最近未检测到的不安全区域的距离 | $c, \ell$ = 正超参数（控制函数形状） | $k^*$ = 满足安全条件所需的最小观测次数 | $\delta$ = 总允许风险（CORE 设 delta=0.1，即 90% 轨迹安全） | $\gamma$ = 分配给最坏情况上下文的风险份额 | 关键设计：测量函数 m(delta x) 处理观测之间的时间相关性——如果一个区域在 t 时刻难以检测，在 t+1 时刻可能同样难以检测。两阶段分解（最坏情况风险分离 + 保守 m）产生实用的概率保证。

7. 基本原理

7.1 Control Barrier Functions (CBFs)

CBF 根植于集合不变性的Nagumo 定理。对于控制仿射系统 $\dot{x} = f(x) + g(x)u$，如果存在扩展 class-$\mathcal{K}$ 函数 $\alpha$，使得对所有 $x$ 满足 $\sup_u [\nabla h(x)^T (f(x)+g(x)u)] \ge -\alpha(h(x))$，则连续可微函数 $h: \mathbb{R}^n \to \mathbb{R}$ 是安全集合 $S = \{x \mid h(x) \ge 0\}$ 的 CBF。该不等式确保当 $h(x)$ 趋近零时，控制总能将系统推回内部，保证 $S$ 的前向不变性。CORE 通过从 VLM 推导的安全语义在线构建 $h(x)$，而非假设预设计的障碍函数，从而适配了这一原理。关键洞察在于CBF 条件可以作为二次规划中的线性约束执行，计算高效（通过 CVXPY 在毫秒级求解），同时在 $h$ 正确时保持形式化前向不变性。

7.2 符号距离场（SDF）作为障碍函数

符号距离场 $h(x)$ 将每个点映射到其到安全集合 $S$ 边界的带符号欧几里得距离：$S$ 内部为正，外部为负。SDF 具有Eikonal 性质 $\|\nabla h(x)\| = 1$（几乎处处），确保 CBF 梯度约束具有均匀缩放，对 QP 求解器条件良好。在 CORE 中，SDF 并非解析计算，而是通过 2D 占用网格上的网格查找近似——对每个单元，通过欧几里得距离变换找到最近的边界单元，产生高效的 $O(N)$ 计算（$N$ 为网格单元数）。梯度 $\nabla h(x)$ 近似为从当前位置到最近边界点 $y^*(x)$ 的归一化向量，当安全集合边界可微时是精确的。

7.3 开放词汇视觉接地

接地流水线依赖两个支柱。第一，SAM3（Segment Anything Model 3）提供类别无关的分割，当用 VLM 识别的语义标签提示时，无需重新训练即可定位任意物体类别。这一点至关重要，因为 CORE 无法预知开放世界中会出现哪些语义类别。第二，深度投影 + 网格投票机制将 2D 像素分类转化为 3D 概率占用表示。每个具有深度 $d$ 的像素被反投影到 3D 点并分配到对应的 2D 网格单元。跨多帧，每个单元累积计数 $n_{ij}^{\text{safe}}$ 和 $n_{ij}^{\text{unsafe}}$，实现了一种贝叶斯占用滤波：$P[\text{单元 } (i,j) \text{ 安全}] = n_{ij}^{\text{safe}} / (n_{ij}^{\text{safe}} + n_{ij}^{\text{unsafe}})$。该投票机制通过要求多个观测的一致共识来自然处理分割噪声。

7.4 基于上鞅理论的概率安全

CORE 的定理 1 提供了根植于非负上鞅Ville 不等式的概率安全保证。随机过程 $M_t$ 是上鞅，如果 $\mathbb{E}[M_{t+1} \mid \mathcal{F}_t] \le M_t$，其中 $\mathcal{F}_t$ 是滤波（历史信息）。关键构造是基于 $d^{-1}(r_t; c, \ell) = c/(r_t + \ell)$ 定义的过程，其中 $r_t$ 是到最近未检测到的不安全区域的距离。在检测概率函数 $m(\delta x)$ 的适当条件下，该过程是非负上鞅。Ville 不等式随即界定该过程的上确界超过阈值的概率，转化为：机器人曾经进入未检测到的不安全区域的概率至多为 $\delta$。两阶段分解（最坏情况风险分配 $\gamma$ + 保守 $m(\cdot)$）使该界在实践中可用，尽管鞅不等式本身具有保守性。检测函数 $m(\delta x)$ 捕获了 VLM 感知质量的空间依赖性——更远或倾斜角度下的物体更难检测，这一点被显式建模而非简单假设掉。

7.5 Chain-of-Thought 推理实现结构化安全输出

VLM 被提示以安全逻辑指令，强制采用结构化推理流水线：(1) 枚举所有可观测实体，(2) 将每个实体分类为安全相关或中性，(3) 分配适当的空间算子（ON/NEAR/AROUND/BETWEEN），(4) 用自然语言为每个分类提供理由，(5) 输出带有安全和不安全谓词列表的最终 JSON。这种结构化 CoT 方法将 VLM 从"黑盒分类器"转变为透明的安全推理器，其中间步骤可被审查。消融研究（移除 CoT 使安全谓词准确率从 91% 降至 61%）经验性验证了显式推理步骤是必要的——VLM 无法在无中间推理的情况下可靠地从原始像素跳到安全谓词。这一原则可推广到 CORE 之外：任何安全关键的 LLM/VLM 应用都应强制执行结构化、可审计的推理轨迹，而非信任端到端黑盒输出。

7.6 关注点分离：推理 vs. 接地 vs. 执行

CORE 的架构体现了跨三个独立模块的关注点分离系统原则。VLM 推理模块处理语义理解（物体对安全意味着什么），但不处理空间精度。接地模块处理空间准确性（安全/不安全区域在度量空间中的位置），但不处理语义解释。CBF 执行模块处理动态安全（给定机器人动力学如何保持安全），但不处理感知。每个模块可独立改进、测试和替换：更好的 VLM 改善推理而不触及接地；更好的深度传感器改善接地而无需重新训练 VLM；不同的 CBF 公式改善执行而不影响感知。这种模块化对于安全关键系统至关重要，因为故障必须可追溯到特定组件。

9. 缺陷与扩展

9.1 局限性

  • 假设约束：(1) 假设 1（安全初始化）要求机器人在任何危险被感知为不安全之前处于安全位置——在实践中这意味着 VLM 必须在机器人进入危险区域之前识别危险。如果危险物体太远或被遮挡，该假设可能失效。(2) 感知不确定性模型假设平均错误率，未考虑逐帧预测不确定性——VLM 可能在单帧中"自信地出错"，而 CORE 的模型平等对待所有帧。
  • 适用场景：目前仅处理空间安全约束（通过空间安全集合表达），不包括时序安全约束（如"在完成 X 之前不得做 Y"）。对于动态变化的环境（移动的人、开关门），时序安全可能比空间安全更重要。
  • 可扩展性：VLM 推理延迟（4.1s）限制了机器人的最大安全速度——在高速场景中，VLM 推理与控制执行之间 4 秒的延迟可能使机器人在检测前就进入危险区域。概率保证定理中的观测数量 k* 与机器人实际速度之间存在隐式张力。

9.2 扩展方向

  • 维度扩展：从空间安全约束扩展到时空安全约束——使用 Signal Temporal Logic (STL) 编码带时间窗口的安全规则。例如"紧急情况下不得在出口处停留超过 5 秒"。这将使 CORE 能够处理更多动态安全场景。
  • 条件放宽：(1) 开发不确定性感知 VLM——让 VLM 不仅输出安全约束，还输出每个约束的置信度。高置信度约束可积极执行；低置信度约束需要更保守的处理。(2) 通过蒸馏减少 VLM 推理延迟，使 CORE 适用于更高速度的机器人平台。
  • 约束增加：(1) 将 CORE 的上下文安全推理与语言驱动规划器耦合——让安全推理不仅阻止不安全规划，还主动引导规划器生成上下文安全的任务规划。(2) 考虑高阶动力学场景——如在高速驾驶中同时考虑速度和加速度约束与上下文安全规则。

9.4 值得深入研究的课题

值得深入研究
上下文安全推理 + 多机器人协作——论文目前仅考虑单机器人上下文安全。在多机器人场景中，安全规则变得更加复杂：(1) 一个机器人的安全行为可能危及另一个（如一个机器人为行人让路却阻塞了另一个机器人的逃生路线）；(2) 机器人间通信可以共享上下文推理结果，形成分布式上下文安全地图；(3) 团队级上下文安全可能需要集中式协调——类似于 ROBOGUARD 的架构但面向多机器人团队。这将把 Vijay Kumar 课题组的 CORE 和 ROBOGUARD 两条线融合为更强大的统一安全系统。

10. 动机推导 —— 第一性原理推导

10.1 想法推导链

  1. 问题的本质是什么？
    机器人安全不仅仅是"别撞到东西"——在真实世界中，"安全"与上下文紧密耦合。同一个动作在不同场景下可能安全或危险。但传统安全方法将安全定义为固定、上下文无关的约束集合——它们无法区分"拥挤走廊"和"空旷走廊"、"紧急情况"和"正常运行"。
  2. 前人如何处理的？为什么不够？
    三条路线：(i) 基于用户指令的语言条件安全——用户描述安全规则，系统接地并执行；(ii) 基于形式验证的规划安全——验证规划动作序列的逻辑一致性；(iii) 基于感知的安全——检测物理障碍并避开。路线 (i) 仍需要用户提供指令；路线 (ii) 无法处理感知层面的上下文；路线 (iii) 完全缺乏语义理解。
  3. 如果机器人能自己"看"和"想"呢？
    VLM 具备两种关键能力：(i) 开放词汇物体识别——能识别超越训练数据的物体类别；(ii) 世界知识推理——理解"'小心地滑'标志不仅是标志，还意味着'周边区域可能湿滑'"。如果用 VLM 做"在线安全推理"，机器人就能自主理解场景中的安全含义。
  4. 还需要什么才能让推理结果可执行？
    VLM 推理在语言层面——需要接地到物理空间（通过分割 + 深度投影 + 网格地图）。接地后的安全集合需要执行——CBF 是最合适的选择，因为它与规划器解耦且提供形式化保证。但 CBF 理论传统上假设确定性障碍函数——需要概率扩展以处理 VLM 感知不确定性。
  5. 最简单的完整方案是什么？
    VLM 观察一帧，输出安全谓词，接地到空间安全集合，CBF 执行，继续下一帧。这个流水线的本质是将一个常识推理引擎（VLM）与安全控制理论（CBF）连接，以感知接地作为桥梁。

10.2 一句话动机

一句话动机
如果人类在进入一个完全陌生的房间后几秒内就能判断哪里安全、哪里危险——为什么机器人做不到？CORE 用 VLM 做安全推理，分割+投影做物理空间接地，CBF 做安全执行——第一次赋予机器人"进入陌生环境时自主评估安全风险"的能力。

10.3 你如何自己想到这个想法

  • 步骤 1 —— 观察现象：在 Vijay Kumar 课题组的 SPINE 和 ROBOGUARD 工作中，安全是通过语言指令或语义地图定义的——但这些都是"人告诉机器人什么是安全的"。有没有办法让机器人自己判断？
  • 步骤 2 —— 识别能力缺口：(1) VLM 能"理解"图像中的危险（GPT-4V 能回答"这张图有什么安全隐患？"）；(2) 但 VLM 输出是文本——不是控制指令；(3) CBF 能执行安全约束——但需要一个障碍函数，不是一段文字。
  • 步骤 3 —— 设计桥接机制：将"VLM 的文本推理"桥接到"CBF 的障碍函数"——通过结构化输出（谓词）+ 语义接地（分割 + 投影 + 网格）+ SDF（符号距离场）。这座桥是 CORE 最核心的工程创新。
  • 步骤 4 —— 处理不确定性：不完美的 VLM 意味着不完美的障碍函数，需要概率安全保证。从随机安全分析中借鉴上鞅技术，将"VLM 可能漏检"建模为概率事件，提供理论保证。

11. 核心收获

11.1 学到的教训

  • 方法论层面：这是LLM/VLM 与控制理论交叉领域的范式级工作。CORE 不仅仅将 VLM 用作分类器或检测器——它将其用作在线安全推理引擎。这是 LLM/VLM 在机器人安全中角色的本质升级：从"工具"到"决策者"。对 UAV 领域：在未知环境中飞行时，UAV 同样可以用 VLM 从第一人称视角推理上下文安全（如"那片区域有盘旋的鸟，可能不安全"）。
  • 技术技巧层面：(1) 结构化提示工程——不是简单问 VLM "这安全吗？"，而是通过安全逻辑 + CoT + JSON 格式将推理过程标准化——可靠性提升是决定性的（61% 到 91%）；(2) 接地流水线设计——分割到像素集合到点云到网格投票到 SDF——每步精心选择平衡精度和效率的算法；(3) 概率安全的理论框架——将 VLM 感知错误纳入安全分析而非简单忽略。
  • 实验/写作层面：论文的对比实验设计层次分明——Oracle（上界）、CORE（本文）、No Context（无推理）、Geometric（纯几何）——四个层次展示上下文推理的阶梯式贡献。消融研究也很有说服力：移除 CoT 使安全检测从 91% 降至 61%，直接证明了推理的必要性。

11.2 直接适用于我的研究

  • 对 UAV 安全导航：CORE 的"VLM 推理 + 接地 + CBF 执行"架构可直接适配到 UAV 场景。低空飞行时，UAV 需要理解上下文危险（电线、树枝、人群聚集区）——这些都超出传统障碍物检测。关键适配点：CBF 需要考虑 UAV 的 6-DOF 动力学和空中特有的安全约束（如最小飞行高度）。
  • 对多机器人协作：CORE 目前是单机器人——结合 ROBOGUARD 或 SPINE-HT 的多机器人框架，可构建上下文感知的多机器人安全系统。核心挑战：不同机器人的 VLM 推理结果可能不一致——如何在冲突的上下文安全推理中做团队最优决策？
  • 对人机交互：CORE 的高层安全指南 G（如"遵循行人社交规范"）本质上是一个人机交互接口——人类操作员可以用自然语言定义安全价值观，无需写代码。这为非技术用户操作机器人提供了安全基础。
