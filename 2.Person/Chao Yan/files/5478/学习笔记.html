2. 核心方法 —— 三大创新支柱

2.1 可学习视觉提示 (Learnable Visual Prompts)

定义 K=10 个可学习提示，每个提示由 L=8 个维度为 D=768 的 token 组成。这些提示被插入到 ViT 的输入序列中，与图像 patch 一起通过冻结的 Transformer 层。

P = {pk}k=1..K,   pk ∈ RL×D = R8×768
总参数量：K × L × D = 10 × 8 × 768 = 61,440 个可学习参数

2.2 混合对比学习 (Hybrid Contrastive Learning)

三类对比目标共同优化表征空间：

对比类型损失函数目标
Visual ContrastiveInfoNCE + MSE同一域内不同视角的图像特征保持一致
Temporal Action ContrastiveBYOL-style 自蒸馏相邻时间步的动作特征平滑过渡，捕捉时序一致性
Text ContrastiveInfoNCE + MSE视觉特征与任务文本描述对齐（语义锚定）

InfoNCE Loss

L_InfoNCE = -log(
    exp(sim(z_i, z_j) / τ)
    ──────────────────────
    Σ_k exp(sim(z_i, z_k) / τ)
)

温度参数 τ 控制正负样本的区分度。

BYOL-style Loss

L_BYOL = ||q_θ(z_θ) - z'_ξ||²₂

其中 z'_ξ 来自 target network
(指数移动平均更新)

无需负样本，通过自蒸馏学习时序不变特征。

2.3 自适应提示编排 (Adaptive Prompt Orchestration)

双分支注意力机制动态组合多个提示，适应不同领域和任务：

核心公式
Branch 1 (Learnable Projection): Wproj ∈ RD×D → 线性投影后与 domain embedding 交互
Branch 2 (Cosine Similarity): cos(prompti, domain_emb) → 直接语义匹配
聚合: α = softmax( Wproj·[prompt||domain] + cos(prompt, domain) )
输出: porchestrated = Σk=1..K αk · pk
核心设计智慧：文本提示作为语义锚点
文本提示特征 不经过注意力加权，直接与视觉特征拼接——这为每个任务提供了一个稳定的语义基准。无论领域如何变化（FOV、亮度、相机角度），文本锚点始终指向同一任务语义，防止视觉特征在跨域迁移中"漂移"。

3. 论文精读

3.1 核心贡献

  1. 首次提出基于 可学习视觉提示 + 对比学习的跨具身视觉运动策略框架
  2. 设计自适应提示编排机制，无需人工指定领域-提示对应关系
  3. 混合对比学习（Visual + Temporal Action + Text）捕获多层语义一致性
  4. 在 AI2-THOR ObjectNav 任务上实现跨 4 个源域到 10 个未知目标域的泛化
  5. 跨具身 zero-shot 转移：ManipulaTHOR → Stretch RE1 → LoCoBot

4. 算法公式全解

4.1 视觉对比损失

VCL
Lvisual = LInfoNCE(zv1, zv2) + λmse · ||zv1 − zv2||²₂
zv1, zv2 为同一场景两个增强视角经过视觉编码器后的特征向量

4.2 时序动作对比损失 (BYOL-style)

TAC
Ltemporal = ||qθ(fθ(st, at)) − fξ(st+1, at+1)||²₂
θ 为 online network (梯度更新)，ξ 为 target network (EMA: ξ ← m·ξ + (1−m)·θ, m=0.996)

4.3 文本对比损失（语义锚定）

TCL
Ltext = LInfoNCE(zvisual, ztext) + λ · ||zvisual − ztext||²₂
ztext = CLIP_Text_Encoder("navigate to the [target_object]")，冻结不更新

4.4 提示编排公式

PO
αk = softmax( Wattn · [ gproj(pk) || cos(pk, demb) ] )
porch = Σk=1K αk · pk
pk第 k 个可学习视觉提示，形状 L×D=8×768
demb8 维领域因子的嵌入向量
gproj可学习投影网络，将提示映射到注意力空间
cos(pk, demb)余弦相似度分支——直接语义匹配
αk第 k 个提示的注意力权重，Σαk=1

4.5 总损失函数

Total
Ltotal = LRL + λ1·Lvisual + λ2·Ltemporal + λ3·Ltext + λ4·Lreg
Lreg 为语义正则化项：对文本锚点特征添加高斯噪声 N(0, σ²=0.01)，增强域迁移鲁棒性

4.6 PPO 策略优化

PPO
LCLIP(θ) = E[ min( rt(θ)·At, clip(rt(θ), 1−ε, 1+ε)·At ) ]
rt(θ) = πθ(at|st) / πθ_old(at|st),   ε = 0.2

4.7 理论保证 —— Excess Risk Bound

Theorem
R(f̂) − R(f*) ≤ O( √(log(KL)/N) + √(D/Ntarget) + η·dHΔH(S, T) )
三项分别对应：提示容量惩罚、目标域样本量、源-目标域分布差异（HΔH-divergence）。CAPO 的编排机制显式缩小最后一项。

6.2 完整训练循环

64 环境并行采样：各环境独立运行——RGB 观测 → 领域因子 → 视觉编码(冻结) → 提示编排 → 文本编码 → 融合 → GRU → Actor 采样动作。每 env 独立维护自己的 AI2-THOR 实例和 GRU 状态。

收集 2048 步经验：64 envs × 32 steps/env = 2048 transitions。RolloutBuffer 存储 (obs, action, reward, done, value, log_prob, h_t)。

GAE 优势计算：At = Σ(γλ)l·δt+l。奖励函数：到达目标 +10，每步 -0.01（鼓励效率），碰撞 -1。

PPO 更新 ×10 epoch：同一批 2048 数据用 10 次。L_total = L_CLIP + c1·L_VF + c2·L_visual + c3·L_temporal + c4·L_text + c5·L_reg。loss.backward() → clip_grad_norm → Adam.step()。

评估与保存：每 5000 steps 评估当前策略在 4 个源域 + 30 个已知目标域 + 10 个未知目标域上的 Success Rate。最优模型保存 checkpoint。

6.3 奖励函数设计

rt = rgoal + rstep + rcollision + rprogress
rgoal+10 (到达目标，episode 终止)
rstep−0.01 (每步小惩罚，鼓励最短路径)
rcollision−1.0 (碰撞障碍物)
rprogress+Δd (向目标靠近的距离奖励)

8. 科研 Tricks

1

冻结大模型 + 小参数提示

核心思想：不微调 CLIP（150M 参数），只训练 ~61K 的视觉提示 + ~500K 的策略头。参数效率提升 300×。

# ❌ 全量微调
optimizer = Adam(model.parameters())  # 150M+ params

# ✅ CAPO 做法
for param in clip_vit.parameters():
    param.requires_grad = False       # 冻结主干
optimizer = Adam([prompts, orchestrator, gru, policy])

不仅节省 GPU 显存，更重要的是防止灾难性遗忘——CLIP 的通用视觉知识被完整保留。

2

混合对比学习 —— 三个目标互补

单一对比目标不够——视觉对比保表征，时序对比保平滑，文本对比保语义。三个目标各自独立、互补协同。

L_total = L_RL +         # 任务导向（完成 ObjectNav）
          L_visual +     # 域不变视觉表征
          L_temporal +   # 动作时序平滑
          L_text         # 语义锚定（防漂移）

消融实验证明文本对比最不可替代——去掉它 SR 暴跌 15.1%。

3

双分支注意力 —— 学习+匹配

单分支注意力的局限：仅靠可学习投影 or 仅靠相似度匹配都不够。双分支 = 一个"学"一个"算"。

# Branch 1: 可学习投影（数据驱动）
w_proj = Linear(dim, dim)(prompt)     # 让网络自己学会"看"

# Branch 2: 余弦相似度（先验驱动）
w_cos = cosine(prompt, domain_emb)    # 用先验知识"算"

alpha = softmax(w_proj + w_cos)       # 两者互补
4

文本特征作语义锚点——不参与注意力

文本提示特征绕过提示编排的注意力机制，直接 concat 到最终特征中。原因：

  • 文本是语义不变的——"navigate to the apple" 在任何域都一样
  • 如果文本也进入注意力，它的权重可能在不同域中剧烈变化
  • 绕过注意力 = 保护语义稳定性 = 防止跨域漂移
5

领域因子显式建模

不要把 FOV、亮度、相机角度等变化交给网络"自己猜"——显式编码为 8 维向量喂给网络。

domain_factors = [fov, rot, look, step,
                  brightness, contrast,
                  saturation, hue]
# 归一化到 [0,1] 或 [-1,1]
d_emb = MLP(domain_factors)  # 8 → 64

这让网络知道自己当前在哪个域——是"有意识的适应"而非"盲目的泛化"。

6

语义正则化 —— 加噪声防过拟合

对文本锚点特征添加高斯噪声 N(0, σ²=0.01)作为正则化手段：

z_text_reg = z_text + N(0, 0.01)
L_reg = ||z_text_reg - z_text_clean||²

这迫使策略不过度依赖文本锚点的精确值——在文本编码器因域变化产生微小偏移时仍能稳定工作。σ=0.1 是实验中找到的最优值。

7

跨具身测试 —— 零样本的极限挑战

Sim-to-Real 不是域泛化的极限——Sim-to-Real 跨不同机器人才是。

CAPO 做到 ManipulaTHOR (7-DOF arm) → Stretch RE1 (3-DOF + 不同相机) → LoCoBot (5-DOF)，三个完全不同的运动学和视觉系统，zero-shot 部署。

这是通过提示编排学到的具身无关（embodiment-agnostic）的视觉运动表征。

8

理论+实验双验证

不仅做实验，还给出了Excess Risk Bound的理论分析：

R(f̂) − R(f*) ≤ O(√(log(KL)/N) + √(D/N_target) + η·d_HΔH)

三项分别约束：模型复杂度、目标域样本量、域差异。提示编排的设计直接缩小了第三项（域差异）的影响。

9. 设计哲学 —— 参数效率与域不变表征

常见误区

  • 微调整个 CLIP 模型 → 灾难性遗忘
  • 只做域随机化 → 泛化能力有限
  • 单一对比损失 → 表征不完整
  • 注意力机制无先验引导 → 学习不稳定
  • 忽视文本模态 → 丢失语义信息

CAPO 的做法

  • 冻结 CLIP + 小参数提示 → 保留通用知识
  • 提示编排 + 领域因子 → 主动适应而非被动泛化
  • 三目标混合对比 → 多层次表征对齐
  • 双分支注意力 → 学习 + 先验互补
  • 文本语义锚定 → 防止表征漂移
CAPO 的核心方法论
冻结通用视觉知识 (CLIP) + 少量可学习提示 (61K params) + 显式域因子 + 混合对比学习 = 跨具身泛化的视觉运动策略。
核心洞察：不需要每个域都重新训练，只需要教会策略如何适应不同域——这就是"learning to adapt"vs "learning for each domain"的本质区别。

关键设计权衡

权衡维度过度的一侧CAPO 的平衡点
提示数量 KK 太小：域覆盖不足
K 太大：冗余 + 过拟合		K=10（消融证明性价比最优）
提示长度 LL 太小：表达能力弱
L 太大：参数量暴增		L=8（8×768=6144 参数/提示）
对比损失权重太大：压制 RL 信号
太小：表征学习不充分		λ₁=0.1, λ₂=0.05, λ₃=0.2
噪声正则化 σ太大：语义信息被破坏
太小：无正则化效果		σ=0.1（最优实验值）

10. 学习路线

前置知识

  1. Vision Transformer (ViT)：理解 patch embedding、self-attention、[CLS] token
  2. CLIP：理解图文对比预训练、双塔架构
  3. 对比学习：InfoNCE、BYOL、SimCLR 的基本原理
  4. Prompt Learning：Visual Prompt Tuning (VPT)、CoOp 等方法的演进
  5. PPO：策略梯度、GAE、clipped objective
  6. ObjectNav：AI2-THOR 环境、任务定义、评估指标

学习路线图

  1. 理解 CLIP + Prompt 范式 → 阅读 VPT (Mengde Xu et al. 2022) 和 CoOp (Zhou et al. 2022)
  2. 复现视觉对比学习 → 在 AI2-THOR 上实现单域 ObjectNav baseline
  3. 加入文本锚点 → 验证语义对齐的效果提升
  4. 实现提示编排 → 双分支注意力 + 领域因子编码
  5. 多域训练 → 4 个源域同时训练，观察泛化曲线
